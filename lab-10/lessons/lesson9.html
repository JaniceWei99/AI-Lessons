9.2 Prompt 工程三大原则 9.2 Three Principles of Prompt Engineering

Prompt 就是你给 AI 的指令。写得好不好，直接决定了 AI 回答的质量。好的 Prompt 就像好的问题——越具体，答案越好。 A prompt is your instruction to AI. How well it's written directly determines response quality. Good prompts are like good questions—the more specific, the better the answer.

🎭 角色设定 🎭 Role Setting

"你是一位资深 Python 导师"——告诉 AI 它应该扮演什么角色，决定回答的风格和深度。 "You are a senior Python tutor"—tell AI what role to play, determining response style and depth.

📝 示例引导 📝 Few-shot Examples

给 AI 1-3 个示例，让它理解你期望的输入输出格式。"所见即所得"。 Give AI 1-3 examples to show expected input/output format. "What you show is what you get."

📋 格式控制 📋 Format Control

明确要求输出格式——表格、列表、JSON、步骤化。结构化输出更易使用。 Explicitly request output format—table, list, JSON, step-by-step. Structured output is more usable.

9.3 实验：用 Python 构建 Prompt 模板 9.3 Experiment: Build Prompt Templates with Python

用 Python 可以系统化地构建和管理 Prompt，实现批量处理和自动化。 Python lets you systematically build and manage prompts for batch processing and automation.

Python 
# Prompt 模板系统 / Prompt template system

def build_prompt(role, task, context="", format_req="", examples=""):
    """构建结构化 Prompt / Build a structured prompt"""
    prompt = f"## 角色 / Role\n{role}\n\n"
    if context:
        prompt += f"## 背景 / Context\n{context}\n\n"
    prompt += f"## 任务 / Task\n{task}\n\n"
    if examples:
        prompt += f"## 示例 / Examples\n{examples}\n\n"
    if format_req:
        prompt += f"## 输出格式 / Output Format\n{format_req}\n"
    return prompt

# 使用示例 / Usage example
prompt = build_prompt(
    role="你是一位经验丰富的 Python 编程导师",
    task="解释列表推导式（list comprehension）的用法",
    context="学生是初学者，刚学完基本循环",
    format_req="先给出简单解释，再给3个由易到难的代码示例",
    examples="输入: 解释for循环\n输出: for循环用于遍历序列..."
)
print(prompt)
输出：Output: 
## 角色 / Role
你是一位经验丰富的 Python 编程导师

## 背景 / Context
学生是初学者，刚学完基本循环

## 任务 / Task
解释列表推导式（list comprehension）的用法

## 示例 / Examples
输入: 解释for循环
输出: for循环用于遍历序列...

## 输出格式 / Output Format
先给出简单解释，再给3个由易到难的代码示例

# Prompt 质量评估工具 / Prompt quality checker
def check_prompt_quality(prompt):
    score = 0
    feedback = []

    if len(prompt) > 50:
        score += 2
        feedback.append("✅ 足够详细")
    else:
        feedback.append("❌ 太简短，需要更多细节")

    role_words = ["作为", "你是", "扮演", "as a", "you are"]
    if any(w in prompt.lower() for w in role_words):
        score += 2
        feedback.append("✅ 包含角色设定")
    else:
        feedback.append("💡 建议添加角色设定")

    format_words = ["格式", "表格", "列表", "步骤", "format", "table", "list", "step"]
    if any(w in prompt.lower() for w in format_words):
        score += 2
        feedback.append("✅ 指定了输出格式")
    else:
        feedback.append("💡 建议指定输出格式")

    print(f"Prompt 质量评分: {score}/6")
    for f in feedback:
        print(f"  {f}")

# 测试 / Test
check_prompt_quality("帮我写个程序")
print()
check_prompt_quality("你是一位Python专家，请用列表和代码示例的格式，解释Python中的装饰器，适合初学者理解")

9.5 实战挑战：3 个 Prompt 场景 9.5 Hands-on: 3 Prompt Scenarios

用你学到的技巧，为以下场景设计最佳 Prompt。先自己写，再看参考答案！ Use what you've learned to design the best prompt for each scenario. Write yours first, then check the reference!

场景 1：代码调试助手 Scenario 1: Code Debugging Assistant

你写了一段 Python 代码但有 bug，想让 AI 帮你找出并修复问题。 You wrote Python code with a bug and want AI to find and fix it.

参考 Prompt： Reference Prompt:

"你是一位经验丰富的 Python 调试专家。请检查以下代码，找出所有 bug，并按以下格式回答：1) 问题描述 2) 错误原因 3) 修复后的完整代码 4) 修改说明。代码如下：[粘贴代码]" "You are an experienced Python debugging expert. Review the following code, find all bugs, and respond in this format: 1) Problem description 2) Root cause 3) Fixed complete code 4) Explanation of changes. Code: [paste code]"

为什么更好：明确角色（调试专家）、任务（找 bug）、输出格式（4 步结构化）。 Why better: Clear role (debug expert), task (find bugs), output format (4-step structured).

场景 2：学习笔记生成器 Scenario 2: Study Notes Generator

你想让 AI 帮你把一篇长文章整理成结构化的学习笔记。 You want AI to turn a long article into structured study notes.

参考 Prompt： Reference Prompt:

"你是一位学习方法专家。请将以下文章整理成结构化笔记，包含：① 核心概念（用粗体标记） ② 关键要点（每点不超过2句话） ③ 知识点之间的关系图（用文字描述） ④ 3个帮助记忆的问题。文章内容：[粘贴文章]" "You are a study methods expert. Turn this article into structured notes with: ① Core concepts (bold) ② Key points (max 2 sentences each) ③ Concept relationships (text description) ④ 3 memory-aid questions. Article: [paste article]"

为什么更好：角色明确、输出结构清晰（4 个部分）、每部分有具体要求。 Why better: Clear role, structured output (4 parts), specific requirements for each.

场景 3：数据分析报告 Scenario 3: Data Analysis Report

你有一份班级考试成绩数据，想让 AI 帮你分析并生成报告。 You have class exam data and want AI to analyze it and generate a report.

参考 Prompt： Reference Prompt:

"你是一位数据分析师。请分析以下考试成绩数据，生成一份报告，包含：1) 整体统计（平均分、最高分、最低分、标准差） 2) 各分数段人数分布 3) 需要关注的学生（低于60分） 4) 提升建议。请用表格和要点列表格式呈现。数据：[粘贴数据]" "You are a data analyst. Analyze this exam data and generate a report with: 1) Overall stats (mean, max, min, std) 2) Score distribution by range 3) Students needing attention (below 60) 4) Improvement suggestions. Use tables and bullet points. Data: [paste data]"

为什么更好：角色（数据分析师）、结构化任务（4 部分）、具体格式要求（表格+列表）。 Why better: Role (data analyst), structured task (4 parts), specific format (tables + bullet points).

9.6 小测验 9.6 Quiz

Q1. 以下哪个不是好 Prompt 的特征？ Q1. Which is NOT a characteristic of a good prompt?

好的 Prompt 需要足够的信息量——角色、背景、任务、格式。简短但缺乏关键信息的 Prompt 会导致回答不理想。 Good prompts need sufficient info—role, context, task, format. Short prompts lacking key details lead to poor responses.

Q2. "Few-shot" Prompt 是什么意思？ Q2. What does "few-shot" prompting mean?

Few-shot 是在 Prompt 中提供几个输入→输出的示例，让 AI 理解你期望的回答格式和风格。 Few-shot means providing a few input→output examples in the prompt so AI understands your expected format and style.

Q3. 以下哪种场景最适合使用 Prompt 工程？ Q3. Which scenario benefits most from prompt engineering?

Prompt 工程在需要 AI 生成复杂、结构化内容时最有价值——明确的指令能显著提升输出质量。 Prompt engineering is most valuable for complex, structured AI outputs—clear instructions significantly improve quality.
