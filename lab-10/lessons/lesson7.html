7.1 实验目标7.1 Lab Goal

在这个实验中，你将构建一个音乐推荐系统。你的系统会根据用户的喜好来推荐类似的歌曲——就像 Spotify、网易云音乐和 QQ 音乐的推荐功能一样。你会学习"协同过滤"和"基于内容的推荐"两种核心算法。 In this lab, you'll build a music recommendation system. Your system will suggest similar songs based on user preferences — just like Spotify, NetEase Music, and QQ Music's recommendation features. You'll learn two core algorithms: "Collaborative Filtering" and "Content-Based Recommendation."

A

协同过滤Collaborative Filtering

"喜欢这首歌的人也喜欢那首歌""People who liked this song also liked that song"

B

基于内容Content-Based

"这首歌和你喜欢的歌风格相似""This song has a similar style to songs you like"

7.2 背景知识7.2 Background

推荐系统无处不在：你在 B站 看到的"推荐视频"、淘宝上的"猜你喜欢"、抖音的"为你推荐" —— 这些背后都是推荐系统在工作。据统计，Netflix 上 80% 的观看内容来自推荐系统，而不是用户自己搜索。 Recommendation systems are everywhere: The "Recommended" videos on YouTube, "You might also like" on Amazon, and TikTok's "For You" feed — these all run on recommendation systems. Netflix reports that 80% of content watched comes from their recommendation system, not user searches.

相似度是推荐系统的核心概念。我们通过计算两首歌（或两个用户）之间的"距离"来判断它们有多相似。常用的方法叫做"余弦相似度"——它衡量两个向量之间的角度，角度越小越相似。 Similarity is the core concept in recommendation systems. We calculate the "distance" between two songs (or two users) to determine how similar they are. A common method is "cosine similarity" — it measures the angle between two vectors; the smaller the angle, the more similar they are.

协同过滤Collaborative Filtering

优点：能发现意想不到的推荐
原理：基于用户行为相似度Advantage: Discovers unexpected recommendations
How: Based on user behavior similarity

基于内容Content-Based

优点：不需要其他用户数据
原理：基于物品属性相似度Advantage: Doesn't need other users' data
How: Based on item attribute similarity

步骤 2：基于内容的推荐Step 2: Content-Based Recommendation

Python 
from sklearn.metrics.pairwise import cosine_similarity

# 安装 sklearn / Install sklearn (usually pre-installed)
# !pip install scikit-learn

# 用音乐特征计算相似度
# Calculate similarity using music features
features = songs[["energy", "danceability", "mood"]].values
similarity_matrix = cosine_similarity(features)

def recommend_by_content(song_title, n=3):
    """Recommend songs based on musical features"""
    idx = songs[songs["title"] == song_title].index[0]
    scores = list(enumerate(similarity_matrix[idx]))
    scores = sorted(scores, key=lambda x: x[1], reverse=True)

    print(f"\nBecause you liked '{song_title}', you might also like:\n")
    for i, (song_idx, score) in enumerate(scores[1:n+1], 1):
        song = songs.iloc[song_idx]
        print(f"  {i}. {song['title']} by {song['artist']}")
        print(f"     Genre: {song['genre']} | Similarity: {score:.0%}")

# 测试推荐 / Test recommendations
recommend_by_content("Shape of You")
recommend_by_content("Bohemian Rhapsody")
recommend_by_content("Someone Like You")

步骤 3：协同过滤推荐Step 3: Collaborative Filtering Recommendation

Python 
# 创建用户评分数据 / Create user rating data
ratings = pd.DataFrame({
    "User": ["Alice", "Alice", "Alice", "Bob", "Bob", "Bob",
             "Carol", "Carol", "Carol", "Dave", "Dave", "Dave"],
    "Song": ["Shape of You", "Levitating", "Happy",
             "Shape of You", "Uptown Funk", "Happy",
             "Bohemian Rhapsody", "Hotel California", "Smells Like Teen Spirit",
             "Bohemian Rhapsody", "Blinding Lights", "Smells Like Teen Spirit"],
    "Rating": [5, 4, 5, 4, 5, 4, 5, 4, 5, 5, 3, 4]
})

# 创建用户-歌曲评分矩阵
# Create a user-song rating matrix
rating_matrix = ratings.pivot_table(index="User", columns="Song", values="Rating").fillna(0)
print("User-Song Rating Matrix:")
print(rating_matrix)

# 计算用户之间的相似度
# Calculate similarity between users
user_similarity = cosine_similarity(rating_matrix)
user_sim_df = pd.DataFrame(user_similarity,
                           index=rating_matrix.index,
                           columns=rating_matrix.index)

def recommend_collaborative(user, n=3):
    """Recommend songs based on similar users' tastes"""
    similar_users = user_sim_df[user].sort_values(ascending=False).index[1:]
    user_rated = set(ratings[ratings["User"] == user]["Song"])

    recommendations = []
    for sim_user in similar_users:
        sim_user_songs = ratings[ratings["User"] == sim_user]
        for _, row in sim_user_songs.iterrows():
            if row["Song"] not in user_rated and row["Rating"] >= 4:
                recommendations.append((row["Song"], row["Rating"], sim_user))

    print(f"\nRecommendations for {user} (based on similar users):\n")
    seen = set()
    count = 0
    for song, rating, from_user in recommendations:
        if song not in seen and count < n:
            print(f"  {count+1}. {song} (liked by {from_user})")
            seen.add(song)
            count += 1

recommend_collaborative("Alice")
recommend_collaborative("Carol")

思考：Alice 喜欢流行和舞曲，Bob 和她品味相似。所以系统会推荐 Bob 喜欢但 Alice 还没听过的歌。这就是"协同过滤"的魔力！ Think: Alice likes pop and dance music, and Bob has similar taste. So the system recommends songs Bob likes that Alice hasn't heard yet. That's the magic of collaborative filtering!

实验检查Lab Check

1. Netflix 上大约多少比例的观看内容来自推荐系统？1. What percentage of Netflix viewing comes from recommendations?

Netflix 报告说约 80% 的观看内容是通过推荐系统发现的，而不是用户主动搜索。Netflix reports that about 80% of content watched is discovered through their recommendation system, not user searches.

2. 协同过滤推荐的基本思路是什么？2. What is the basic idea behind collaborative filtering?

协同过滤的核心思路是：如果两个用户在过去喜欢了类似的东西，那么他们未来也可能喜欢类似的东西。Collaborative filtering's core idea: if two users liked similar things in the past, they'll likely enjoy similar things in the future.

3. 余弦相似度衡量的是什么？3. What does cosine similarity measure?

余弦相似度衡量两个向量之间的角度。角度越小（值越接近 1），两个向量越相似。Cosine similarity measures the angle between two vectors. The smaller the angle (closer to 1), the more similar they are.

4. 基于内容的推荐不需要什么？4. What does content-based recommendation NOT need?

基于内容的推荐只分析物品本身的属性特征，不需要其他用户的行为数据。Content-based recommendation only analyzes items' attribute features — it doesn't need other users' behavioral data.

5. 混合推荐系统的优点是什么？5. What is the advantage of a hybrid recommendation system?

混合推荐系统结合了多种推荐方法（如协同过滤和基于内容）的优势，通常能获得比单一方法更好的推荐效果。Hybrid systems combine strengths of multiple methods (like collaborative filtering and content-based), typically producing better recommendations than any single approach.
