## 导入环节（8 分钟）

> 同学们，从今天开始，你们会学到一项在 AI 时代最值钱的技能——Prompt Engineering（提示词工程）。同样是使用 ChatGPT，为什么有的人能得到精准有用的回答，有的人却总是得到泛泛而谈的废话？区别就在于提示词的质量。

**现场对比演示（可用 ChatGPT 或模拟输出）：**

| 差提示词 | 好提示词 |
|----------|----------|
| "帮我写一篇作文" | "你是一位高中语文老师，请以'科技改变生活'为题，写一篇800字的议论文。要求：开头引用名言，正文包含3个具体案例（手机支付、远程医疗、自动驾驶），结尾呼应开头。" |

> 大家看到区别了吗？好的提示词像一份详细的工作说明书，差的提示词像对着空气喊一句话。今天我们就来系统学习怎么写好提示词。

## 核心概念讲解（15 分钟）

### 原则一：角色设定（Role-Setting）

> 第一个技巧：告诉 AI "你是谁"。就像给演员一个角色剧本。

```
你是一位有10年经验的Python程序员，擅长代码审查和性能优化。
```

> 设定角色后，AI 会用那个角色的专业知识和说话方式来回答。

### 原则二：少样本学习（Few-Shot Learning）

> 第二个技巧：给 AI 看几个例子，让它模仿。

```
请按以下格式分析句子的情感：
输入："今天天气真好" → 情感：积极，关键词：好
输入："作业太难了" → 情感：消极，关键词：难
输入："这个电影还行吧" → 情感：？
```

> 给了两个例子后，AI 就知道你想要什么格式的输出了。这就像教小朋友做题——先做两道示范题，再让他自己做。

### 原则三：格式控制（Format Control）

> 第三个技巧：明确规定输出的格式。

```
请用以下JSON格式返回结果：
{
  "情感": "积极/消极/中性",
  "置信度": 0.0-1.0,
  "关键词": ["词1", "词2"]
}
```

> 格式控制非常重要，特别是当你需要用程序解析 AI 输出的时候。

**板书总结公式：** `好的 Prompt = 角色 + 背景 + 任务 + 示例 + 格式要求`

## 实验指导

### 构建 Prompt 模板系统（15 分钟）

> 我们先用 Python 建一个可复用的提示词模板。

**教师演示，学生跟做：**

```python
def create_prompt(role, task, examples=None, format_spec=None):
    prompt = f"角色：{role}\n\n"
    prompt += f"任务：{task}\n\n"
    
    if examples:
        prompt += "参考示例：\n"
        for ex in examples:
            prompt += f"  输入：{ex['input']} → 输出：{ex['output']}\n"
        prompt += "\n"
    
    if format_spec:
        prompt += f"输出格式要求：{format_spec}\n"
    
    return prompt

# 测试模板
prompt = create_prompt(
    role="资深Python程序员",
    task="检查以下代码的bug并给出修复建议",
    examples=[
        {"input": "for i in range(10) print(i)", "output": "缺少冒号：for i in range(10):"}
    ],
    format_spec="先指出错误位置，再给出修复代码，最后解释原因"
)
print(prompt)
```

> 有了这个模板函数，我们可以快速生成各种场景的高质量提示词。

### 质量检查函数（5 分钟）

**教师快速演示：**

```python
def check_prompt_quality(prompt):
    score = 0
    checks = {
        "包含角色设定": "角色" in prompt or "你是" in prompt,
        "任务描述清晰": len(prompt) > 50,
        "包含示例": "示例" in prompt or "例如" in prompt,
        "有格式要求": "格式" in prompt or "要求" in prompt,
        "足够详细": len(prompt) > 100
    }
    
    print("Prompt 质量检查：")
    for check, passed in checks.items():
        status = "✓" if passed else "✗"
        print(f"  {status} {check}")
        if passed:
            score += 1
    
    print(f"\n质量评分：{score}/5")
    return score

check_prompt_quality(prompt)
```

### 实战场景一：代码调试助手（12 分钟）

> 我们用模板构建一个代码调试场景的提示词。

**教师展示场景，让学生补充完善：**

```python
debug_prompt = create_prompt(
    role="你是一位耐心的编程导师，擅长向初学者解释代码错误",
    task="分析以下Python代码中的错误，给出修复方案",
    examples=[
        {"input": "print('hello'", "output": "错误：缺少右括号。修复：print('hello')"}
    ],
    format_spec="1. 错误类型 2. 错误位置 3. 修复代码 4. 错误原因（用简单语言）"
)

# 模拟要调试的代码
buggy_code = """
def calculate_average(numbers):
    total = 0
    for num in numbers
        total += num
    average = total / len(numbers)
    return average
"""

full_prompt = debug_prompt + f"\n待调试代码：\n```python\n{buggy_code}\n```"
print(full_prompt)
```

**让学生尝试用这个提示词去 ChatGPT 上测试效果。** 或者让学生自己写有 bug 的代码，构造调试提示词。

### 实战场景二：学习笔记生成器（12 分钟）

**此环节让学生更多独立完成：**

```python
notes_prompt = create_prompt(
```

> 给大家 8 分钟，选择自己正在学的一门课，构造一个学习笔记生成提示词。

### 实战场景三：数据分析报告（12 分钟）

```python
analysis_prompt = create_prompt(
    role="你是一位数据分析师，擅长从数据中发现洞察并撰写报告",
    task="根据给定数据生成分析报告",
    format_spec="""报告格式：
    1. 数据概览（样本量、时间范围）
    2. 关键发现（3-5条，每条配数据支撑）
    3. 趋势分析（上升/下降/稳定）
    4. 建议措施（基于数据的可操作建议）

print(analysis_prompt)
```

> 这个模板可以用于分析任何数据——销售数据、考试成绩、网站流量等等。

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 提示词太长导致 AI 忽略部分要求 | 优先级排序：最重要的要求放在开头和结尾 |
| AI 没有遵循格式要求 | 在提示词末尾再次强调格式，使用"必须""严格按照" |
| 少样本示例选得不好 | 示例要多样化，覆盖典型场景和边界情况 |
| 角色设定太模糊 | 加入具体经验年限、专长领域、性格特点 |

## 总结与回顾（6 分钟）

> 记住 Prompt 工程三大原则：**角色设定、少样本学习、格式控制**。好的提示词就像好的沟通——说清楚你是谁、你想要什么、你希望怎么得到。
