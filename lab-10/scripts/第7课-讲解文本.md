## 核心概念讲解（12 分钟）

### 余弦相似度

> 这是推荐系统最核心的数学工具。别怕，原理很简单！

**板书类比：**

> 想象两首歌各有三个属性：能量（0-1）、节奏感（0-1）、情绪（0-1）。我们可以把每首歌想象成空间中的一个"箭头"（向量）。两个箭头方向越接近，两首歌越相似。

> 余弦相似度的结果：
> - **1** = 完全相同方向（非常相似）
> - **0** = 完全垂直（毫无关系）
> - **-1** = 完全相反（完全不同）

**用简单数字举例：**

> 歌曲A = [0.9, 0.8, 0.7]（高能量、高节奏、快乐）
> 歌曲B = [0.8, 0.9, 0.6]（高能量、高节奏、快乐）→ 相似度 ≈ 0.99
> 歌曲C = [0.1, 0.2, 0.3]（低能量、慢节奏、忧郁）→ 与A的相似度 ≈ 0.98（方向类似但数值小）

> 实际计算我们交给 Python 来做，大家只需要理解概念就好。

## 实验指导

### 构建音乐数据集（10 分钟）

> 首先我们需要创造一个"小型音乐数据库"。

**教师演示代码，学生跟做：**

```python
import pandas as pd
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# 12首歌的音乐数据集
songs = pd.DataFrame({
    '歌曲': ['摇滚风暴', '爵士之夜', '电子脉冲', '民谣小调', '嘻哈节拍',
             '古典月光', '流行甜心', '金属闪电', '蓝调忧伤', '拉丁热情',
             '乡村吉他', '朋克反叛'],
    'energy': [0.9, 0.3, 0.8, 0.2, 0.7, 0.1, 0.6, 0.95, 0.3, 0.8, 0.4, 0.85],
    'danceability': [0.6, 0.5, 0.9, 0.3, 0.85, 0.1, 0.8, 0.5, 0.2, 0.9, 0.4, 0.6],
    'mood': [0.7, 0.6, 0.8, 0.4, 0.7, 0.5, 0.9, 0.6, 0.2, 0.85, 0.7, 0.5]
})

print(songs)
```

> 每首歌有三个数值属性：energy（能量）、danceability（舞蹈性）、mood（情绪正面度）。数值越接近 1 表示该属性越强。

**让学生花 3 分钟观察数据：** 哪些歌曲看起来比较相似？先用直觉判断，一会儿用算法验证。

### 基于内容的推荐（20 分钟）

> 第一种方法：根据歌曲本身的属性来推荐。如果你喜欢"摇滚风暴"，我们就找属性最接近的歌。

**教师先演示完整流程，然后让学生修改参数探索：**

```python
# 提取特征列
features = songs[['energy', 'danceability', 'mood']]

# 计算所有歌曲之间的相似度矩阵
sim_matrix = cosine_similarity(features)
sim_df = pd.DataFrame(sim_matrix, index=songs['歌曲'], columns=songs['歌曲'])

print("相似度矩阵：")
print(sim_df.round(3))
```

> 这个矩阵展示了每首歌和其他所有歌之间的相似度。对角线上都是 1.0（自己和自己当然完全相似）。

**推荐函数：**

```python
def recommend_by_content(song_name, top_n=3):
    if song_name not in sim_df.index:
        return "未找到该歌曲"
    
    similarities = sim_df[song_name].sort_values(ascending=False)
    # 排除自身
    recommendations = similarities[1:top_n+1]
    
    print(f"\n因为你喜欢《{song_name}》，推荐给你：")
    for song, score in recommendations.items():
        print(f"  《{song}》 - 相似度: {score:.3f}")

recommend_by_content("摇滚风暴")
```

**此时让学生尝试不同的歌曲名称作为输入，观察推荐结果是否合理。** 给 8 分钟探索。

> 大家觉得推荐结果准吗？跟你们刚才的直觉判断一致吗？

### 协同过滤推荐（25 分钟）

> 第二种方法更有趣——不看歌曲本身的属性，而是看"跟你品味相似的人在听什么"。

**教师解释原理后，展示代码：**

```python
# 用户-歌曲评分矩阵（1-5分，0表示未听过）
user_ratings = pd.DataFrame({
    '摇滚风暴': [5, 2, 4, 1, 5],
    '爵士之夜': [2, 5, 1, 4, 2],
    '电子脉冲': [4, 1, 5, 2, 4],
    '民谣小调': [1, 4, 2, 5, 1],
    '嘻哈节拍': [3, 2, 4, 2, 4],
    '流行甜心': [3, 3, 3, 3, 3],
}, index=['小明', '小红', '小华', '小丽', '小强'])

print("用户评分矩阵：")
print(user_ratings)
```

> 看这个矩阵：小明给"摇滚风暴"打了 5 分，给"民谣小调"只打了 1 分。小红正好相反。说明他们的音乐品味很不同。

```python
# 计算用户之间的相似度
user_sim = cosine_similarity(user_ratings)
user_sim_df = pd.DataFrame(user_sim, index=user_ratings.index, columns=user_ratings.index)

print("\n用户相似度：")
print(user_sim_df.round(3))
```

> 数据告诉我们，小明和小强最相似（都喜欢摇滚和电子），小明和小丽最不同（品味完全相反）。

**让学生讨论 3 分钟：** 如果小强还没听过"民谣小调"，根据协同过滤，我们应该推荐还是不推荐？为什么？

## 总结与回顾（10 分钟）

> 今天我们学了两种推荐算法：

**板书对比表：**

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 基于内容 | 看物品属性 | 不需要其他用户数据 | 推荐同质化，缺乏惊喜 |
| 协同过滤 | 看相似用户行为 | 能发现新类型 | 冷启动问题（新用户没数据） |

> 真正的 Netflix 和抖音会把这两种方法结合起来使用，这叫做"混合推荐"。
