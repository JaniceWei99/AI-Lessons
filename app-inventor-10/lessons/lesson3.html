3.2 API 入门3.2 API Basics

API 是什么？ API 的全称是 Application Programming Interface（应用程序编程接口）。听起来很复杂，但其实你可以把它想象成餐厅里的服务员！ What is an API? API stands for Application Programming Interface. It sounds complicated, but think of it like a waiter in a restaurant!

想象你在一家餐厅吃饭。你不会自己跑到厨房去做菜，而是告诉服务员你想吃什么，然后服务员把你的要求带给厨房，厨房做好了菜，服务员再把菜端给你。 Imagine you're eating at a restaurant. You don't run into the kitchen to cook — you tell the waiter what you want, the waiter takes your order to the kitchen, the kitchen prepares the food, and the waiter brings it back to you.

1

你点菜You Order

你（App）告诉服务员想要什么数据You (App) tell the waiter what data you need

2

服务员传达Waiter Delivers

API 把你的请求发送给服务器The API sends your request to the server

3

厨房做菜Kitchen Cooks

服务器处理请求，准备好数据The server processes the request and prepares data

4

你拿到菜You Get Food

API 把数据返回给你的 AppThe API returns data back to your App

URL、请求和响应URLs, Requests, and Responses

每个 API 都有一个网址（URL），你的 App 向这个网址发送一个请求（Request），服务器收到请求后处理数据，再返回一个响应（Response）。 Every API has a URL (web address). Your App sends a Request to this URL. The server receives the request, processes it, and sends back a Response.

举个例子：假设你想查北京的天气。你的 App 会向天气 API 的 URL 发送一个请求："请告诉我北京现在的天气"。服务器收到后，查一下北京的天气数据，然后把结果（温度、天气描述等）用 JSON 格式返回给你的 App。 Example: You want to check Beijing's weather. Your App sends a request to the weather API URL: "Tell me Beijing's current weather." The server checks the data and returns the results (temperature, weather description, etc.) in JSON format to your App.

一个典型的天气 API 请求 URL 长这样： A typical weather API request URL looks like this:

URL 
https://api.openweathermap.org/data/2.5/weather?q=Beijing&appid=YOUR_API_KEY&units=metric&lang=zh_cn

让我们拆解一下这个 URL： Let's break down this URL:

基础地址Base URL

https://api.openweathermap.org/data/2.5/weather — 这是天气 API 的"地址"，就像餐厅的门牌号。https://api.openweathermap.org/data/2.5/weather — This is the API's address, like a restaurant's street address.

参数Parameters

?q=Beijing — 查询城市名。&appid=YOUR_API_KEY — 你的身份证明。&units=metric — 使用摄氏度。这些是你告诉服务员的具体要求。?q=Beijing — City name to query. &appid=YOUR_API_KEY — Your identity pass. &units=metric — Use Celsius. These are the specific details of your order.

响应是什么样的？What Does a Response Look Like?

当服务器返回数据时，数据是JSON 格式的。JSON 就像一个"数据盒子"，里面整齐地装着各种信息： When the server returns data, it comes in JSON format. JSON is like an organized "data box" that neatly holds different pieces of information:

JSON 
{
  "name": "Beijing",
  "main": {
    "temp": 22.5,
    "humidity": 45
  },
  "weather": [
    {
      "description": "晴",
      "icon": "01d"
    }
  ],
  "wind": {
    "speed": 3.2
  }
}

3.3 动手实践：天气查询 App3.3 Hands-On: Weather Query App

现在我们要动手做一个真正的天气查询 App！用户输入城市名称，点击按钮，App 就会显示该城市的实时天气信息。 Now let's build a real weather query App! The user types a city name, taps a button, and the App shows that city's live weather information.

步骤 1：注册免费天气 APIStep 1: Sign Up for a Free Weather API

获取 API Key：访问 openweathermap.org，点击 "Sign Up"，创建一个免费账号。注册完成后，在 "API Keys" 页面可以找到你的专属 API Key。这个 Key 就是你的"身份证"，API 用它来识别是谁在发送请求。免费账号每分钟可以调用 60 次，足够我们使用了！ Get your API Key: Visit openweathermap.org, click "Sign Up", and create a free account. After signing up, find your API Key on the "API Keys" page. This Key is your "ID card" — the API uses it to know who is making requests. The free plan allows 60 calls per minute, more than enough for us!

步骤 2：设计 App 界面Step 2: Design the App UI

打开 App Inventor，创建一个新项目，命名为 WeatherApp。在 Designer 视图中，添加以下组件： Open App Inventor, create a new project named WeatherApp. In the Designer view, add these components:

可见组件（界面上看得到的）Visible Components (shown on screen)

1. HorizontalArrangement — 水平排列容器
    TextBox（命名为 txtCity）— 让用户输入城市名，Hint 设为"输入城市名 / Enter city"
    Button（命名为 btnGetWeather）— 文字设为"查询天气"
2. VerticalArrangement — 垂直排列容器，用来显示结果
    Label（命名为 lblCityName）— 显示城市名
    Label（命名为 lblTemperature）— 显示温度
    Label（命名为 lblDescription）— 显示天气描述（晴、多云等）
    Image（命名为 imgWeatherIcon）— 显示天气图标 1. HorizontalArrangement — Horizontal layout container
    TextBox (named txtCity) — For user to type city name, set Hint to "Enter city"
    Button (named btnGetWeather) — Text set to "Get Weather"
2. VerticalArrangement — Vertical layout container for results
    Label (named lblCityName) — Displays city name
    Label (named lblTemperature) — Displays temperature
    Label (named lblDescription) — Displays weather description (sunny, cloudy, etc.)
    Image (named imgWeatherIcon) — Displays weather icon

不可见组件（幕后工作者）Non-Visible Components (behind the scenes)

Web 组件（命名为 Web1）— 这是最关键的组件！它负责向天气 API 发送请求并接收响应。你可以在 Palette 面板的 Connectivity 分类中找到它。

把 Web 组件拖到界面中，它会自动出现在屏幕下方的"不可见组件"区域。 Web component (named Web1) — This is the key component! It sends requests to the weather API and receives responses. Find it in the Connectivity category of the Palette panel.

Drag the Web component onto the screen — it will appear in the "Non-visible components" area at the bottom.

步骤 3：在 Blocks 中发送 GET 请求Step 3: Send a GET Request in Blocks

切换到 Blocks 视图。我们需要在用户点击"查询天气"按钮时，向天气 API 发送请求。按以下步骤拼接积木块： Switch to the Blocks view. We need to send a request to the weather API when the user taps the "Get Weather" button. Connect these blocks:

积木块 1：按钮点击事件Block 1: Button Click Event

从 btnGetWeather 中拖出 when btnGetWeather.Click do 事件块。在这个块里面，我们要做两件事：

a) 拼接 URL：使用 join（文本拼接）块，把 API 地址和用户输入的城市名拼在一起：
join("https://api.openweathermap.org/data/2.5/weather?q=", txtCity.Text, "&appid=你的API_KEY&units=metric&lang=zh_cn")

b) 设置 Web1 的 URL：使用 set Web1.Url to 块，把拼好的 URL 放进去。

c) 发起请求：使用 call Web1.Get 块，发送 GET 请求。 Drag out the when btnGetWeather.Click do event block from btnGetWeather. Inside this block, we'll do two things:

a) Build the URL: Use a join (text join) block to combine the API address with the user's city input:
join("https://api.openweathermap.org/data/2.5/weather?q=", txtCity.Text, "&appid=YOUR_API_KEY&units=metric&lang=zh_cn")

b) Set Web1's URL: Use the set Web1.Url to block and plug in the joined URL.

c) Make the request: Use the call Web1.Get block to send the GET request.

步骤 4：解析 JSON 响应Step 4: Parse the JSON Response

当 API 返回数据后，我们需要从 JSON 里提取需要的信息。使用 when Web1.GotText 事件块来处理响应： When the API returns data, we need to extract the information we need from the JSON. Use the when Web1.GotText event block to handle the response:

积木块 2：处理 API 响应Block 2: Handle API Response

从 Web1 中拖出 when Web1.GotText (url, responseCode, responseType, responseContent) 事件块。

在里面依次执行以下操作：

a) 解析 JSON：使用内置块 call Web1.JsonTextDecodeWithDictionaries，传入 responseContent。把结果存储到一个变量中（比如 jsonData）。

b) 提取城市名：使用 call look up in pairs 块，在 jsonData 中查找 key 为 "name" 的值，设置到 lblCityName.Text。

c) 提取温度：先用 look up in pairs 查找 "main"，得到一个子字典；再从子字典中查找 "temp"，设置到 lblTemperature.Text（拼接上 "°C"）。

d) 提取天气描述：用 look up in pairs 查找 "weather"，得到一个列表；从列表第 1 项中查找 "description"，设置到 lblDescription.Text。 Drag out the when Web1.GotText (url, responseCode, responseType, responseContent) event block from Web1.

Inside, do the following:

a) Parse JSON: Use the built-in block call Web1.JsonTextDecodeWithDictionaries, passing in responseContent. Store the result in a variable (e.g. jsonData).

b) Extract city name: Use a look up in pairs block to find the key "name" in jsonData, and set lblCityName.Text to that value.

c) Extract temperature: First use look up in pairs to find "main" and get a sub-dictionary. Then from that sub-dictionary, look up "temp" and set lblTemperature.Text (join it with "°C").

d) Extract weather description: Use look up in pairs to find "weather" and get a list. From the 1st item in that list, look up "description" and set lblDescription.Text.

步骤 5：显示天气图标Step 5: Display the Weather Icon

OpenWeatherMap 提供天气图标。我们可以用图标代码拼出图片的 URL，然后显示在 Image 组件中： OpenWeatherMap provides weather icons. We can build the icon's image URL from the icon code, then display it in the Image component:

URL 
https://openweathermap.org/img/wn/01d@2x.png

提示：在 JSON 响应的 weather 列表第 1 项中，有一个 "icon" 字段（比如 "01d"）。用 join 块把它拼成完整的图标 URL：join("https://openweathermap.org/img/wn/", iconCode, "@2x.png")，然后设置 imgWeatherIcon.Picture 为这个 URL。 Tip: In the JSON response, the 1st item of the weather list has an "icon" field (e.g. "01d"). Use a join block to build the full icon URL: join("https://openweathermap.org/img/wn/", iconCode, "@2x.png"), then set imgWeatherIcon.Picture to this URL.

3.4 理解 JSON3.4 Understanding JSON

JSON（JavaScript Object Notation）是互联网上最常用的数据格式之一。几乎所有 API 都用 JSON 来传递数据。它的特点是结构清晰、易于阅读。 JSON (JavaScript Object Notation) is one of the most popular data formats on the internet. Nearly all APIs use JSON to transmit data. It's structured, clean, and easy to read.

键值对（Key-Value Pairs）Key-Value Pairs

JSON 的基本结构是键值对。"键"是数据的名字，"值"是数据的内容，中间用冒号隔开。就像一个标签和它对应的内容： The basic structure of JSON is key-value pairs. The "key" is the data's name, the "value" is the data's content, separated by a colon. Like a label and its content:

JSON 
{
  "name": "Alice",
  "age": 12,
  "city": "Shanghai"
}

这里有 3 个键值对："name" 的值是 "Alice"，"age" 的值是 12，"city" 的值是 "Shanghai"。 Here there are 3 key-value pairs: "name" has value "Alice", "age" has value 12, "city" has value "Shanghai".

嵌套对象（Nested Objects）Nested Objects

有时候，一个值本身又是一个 JSON 对象（用花括号 {} 包裹），就像一个盒子里面套着另一个盒子： Sometimes a value is itself another JSON object (wrapped in curly braces {}), like a box inside a box:

JSON 
{
  "student": {
    "name": "Bob",
    "scores": {
      "math": 95,
      "english": 88
    }
  }
}

要获取 Bob 的数学成绩，你需要先找到 "student"，再找到 "scores"，最后找到 "math" — 就像打开盒子一样，一层一层地找。在 App Inventor 中，就是连续用多个 look up in pairs 块。 To get Bob's math score, you first find "student", then "scores", then "math" — like opening boxes one layer at a time. In App Inventor, you chain multiple look up in pairs blocks.

列表（Arrays）Arrays (Lists)

JSON 中用方括号 [] 表示列表。天气 API 返回的 "weather" 字段就是一个列表。在 App Inventor 中，你可以用 select list item 来获取列表中的第 N 项。 In JSON, square brackets [] indicate a list. The "weather" field returned by the weather API is a list. In App Inventor, use select list item to get the Nth item from a list.

小贴士：JSON 中的值可以是文字（用引号括起来）、数字（没有引号）、true/false、列表（方括号）或者另一个对象（花括号）。在天气 API 的响应中，你会同时遇到所有这些类型！ Quick tip: JSON values can be text (in quotes), numbers (no quotes), true/false, lists (square brackets), or another object (curly braces). In the weather API response, you'll encounter all of these types!

课堂测验Lesson Quiz

1. API 最像下面哪个比喻？1. Which analogy best describes an API?

API 就像服务员，在你的 App（顾客）和服务器（厨房）之间传递请求和数据。你不需要直接进厨房，服务员帮你搞定一切！An API is like a waiter who passes requests and data between your App (customer) and the server (kitchen). You don't need to go into the kitchen — the waiter handles everything!

2. 在天气 API 的 URL 中，?q=Beijing 是什么意思？2. In the weather API URL, what does ?q=Beijing mean?

?q=Beijing 是一个查询参数（query parameter）。q 代表 query（查询），Beijing 是你要查询的城市名。这样 API 就知道你想要北京的天气数据。?q=Beijing is a query parameter. q stands for query, and Beijing is the city name you're looking for. This tells the API you want weather data for Beijing.

3. JSON 中的花括号 {} 表示什么？3. What do curly braces {} represent in JSON?

在 JSON 中，花括号 {} 表示一个"对象"，里面包含若干键值对。方括号 [] 才是表示列表。In JSON, curly braces {} represent an "object" containing key-value pairs. Square brackets [] represent a list.

4. 在 App Inventor 中，哪个组件用来发送网络请求？4. In App Inventor, which component is used to send web requests?

Web 组件是 App Inventor 中专门用来发送网络请求的组件。你可以在 Connectivity 分类中找到它。用它来调用 API、获取网页数据等。The Web component is used in App Inventor to send network requests. Find it in the Connectivity category. Use it to call APIs, fetch web data, and more.

5. API Key 的作用是什么？5. What is the purpose of an API Key?

API Key 是你的专属"身份证"。API 服务器用它来识别是谁在发送请求，同时也用来跟踪使用量和防止滥用。记得保管好你的 API Key，不要分享给别人！An API Key is your personal "ID card." The API server uses it to identify who is making requests, track usage, and prevent abuse. Remember to keep your API Key safe and never share it!
