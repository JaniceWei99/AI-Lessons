5.2 Prompt 工程入门 5.2 Prompt Engineering Basics

Prompt 工程（Prompt Engineering）是指精心设计发给 AI 的提示词，让 AI 给出更好、更准确的回答。在 ChatGPT 的 API 中，有一种特殊的消息叫做系统提示词（System Prompt），它就像给 AI 一份"工作说明书"，告诉 AI 应该扮演什么角色、用什么风格回答。 Prompt Engineering means carefully designing the prompts you send to AI to get better, more accurate responses. In the ChatGPT API, there's a special message called the System Prompt, which is like giving AI a "job description" — telling it what role to play and how to respond.

什么是系统提示词？系统提示词是发给 AI 的第一条隐藏消息，用户看不到它，但 AI 会一直记住它。它决定了 AI 的"性格"和行为方式。就像给演员一个剧本——AI 会按照你设定的角色来表演！ What is a System Prompt? The system prompt is the first hidden message sent to AI. Users don't see it, but AI always remembers it. It determines AI's "personality" and behavior. Think of it like giving an actor a script — AI will perform according to the role you set!

让我们来看看好的系统提示词和坏的系统提示词有什么区别： Let's compare the difference between good and bad system prompts:

不够好的 PromptWeak Prompt

"你是一个助手。""You are an assistant."

太模糊了！AI 不知道该怎么回答，可能给出很长、很无聊的回复。Too vague! AI doesn't know how to respond and might give long, boring replies.

优秀的 PromptGreat Prompt

"你是一个友好的少儿科学老师，叫'小博士'。用简单有趣的语言解释科学知识，多举生活中的例子。每次回答不超过100字。如果学生答对了，给予鼓励！""You are a friendly science tutor for kids called 'Dr. Fun.' Explain science in simple, fun language with everyday examples. Keep answers under 100 words. Encourage students when they get things right!"

明确了角色、语气、长度和互动方式！Clear role, tone, length, and interaction style!

步骤 1：添加对话历史列表变量Step 1: Add a Conversation History List Variable

为了让 AI 记住之前的对话，我们需要创建一个列表变量来保存所有消息。每条消息包含两个信息：谁说的（role）和说了什么（content）。 To make AI remember the conversation, we need to create a list variable to store all messages. Each message contains two pieces of info: who said it (role) and what was said (content).

操作方法：在 App Inventor 的 Blocks 编辑器中，创建一个全局变量 conversationHistory，设为一个空列表（create empty list）。这个列表会存储所有对话记录。 How to do it: In App Inventor's Blocks editor, create a global variable conversationHistory set to an empty list (create empty list). This list will store all conversation records.

ChatGPT API 需要的消息格式是这样的（JSON）： The ChatGPT API expects messages in this format (JSON):

JSON 
{
  "messages": [
    {"role": "system", "content": "You are a friendly science tutor for kids."},
    {"role": "user",   "content": "What is gravity?"},
    {"role": "assistant", "content": "Gravity is like an invisible hand..."},
    {"role": "user",   "content": "Why do things fall down?"}
  ]
}

注意三种角色：system（系统提示词）、user（用户消息）、assistant（AI 回复）。每次发请求时，都要把所有历史消息一起发过去，这样 AI 才能记住上下文。 Notice the three roles: system (system prompt), user (user message), assistant (AI reply). Every time you send a request, you must send all history messages together so AI can remember the context.

界面设计：在 Designer 中，将之前的单个 Label 替换为一个 VerticalScrollArrangement（垂直滚动布局）。我们会在代码中动态添加 Label 来显示每条消息。设置宽度为 Fill parent，高度为 80%。 UI Design: In the Designer, replace the single Label with a VerticalScrollArrangement (vertical scrolling layout). We'll dynamically add Labels in code to show each message. Set width to Fill parent and height to 80%.

在 Blocks 中，创建一个过程（procedure）叫 addChatBubble，它接收两个参数：message（消息内容）和 isUser（是否是用户消息）。 In Blocks, create a procedure called addChatBubble that takes two parameters: message (message content) and isUser (whether it's a user message).

这是最关键的一步！每次用户发送新消息时，我们需要把系统提示词 + 所有历史消息 + 新消息一起发给 ChatGPT API。 This is the most critical step! Every time the user sends a new message, we need to send the system prompt + all history messages + new message together to the ChatGPT API.

步骤 5：添加"清除聊天"按钮Step 5: Add a "Clear Chat" Button

当用户想开始新话题时，需要清除之前的对话记录。添加一个"清除聊天"按钮来重置一切。 When users want to start a new topic, they need to clear previous conversation records. Add a "Clear Chat" button to reset everything.

操作方法：
1. 在 Designer 中添加一个按钮，命名为 btnClear，文字设为"清除聊天"
2. 在 Blocks 中，当 btnClear 被点击时：
  • 将 conversationHistory 重置为空列表
  • 移除 VerticalScrollArrangement 中的所有动态 Label
  • （可选）显示一条提示："聊天已清除，开始新对话吧！" How to do it:
1. In Designer, add a button named btnClear with text "Clear Chat"
2. In Blocks, when btnClear is clicked:
  • Reset conversationHistory to empty list
  • Remove all dynamic Labels from VerticalScrollArrangement
  • (Optional) Show a notice: "Chat cleared, start a new conversation!"

为什么要清除？对话历史越长，发送给 API 的数据就越多，响应速度会变慢，而且 API 有 Token 数量限制（GPT-3.5-turbo 大约 4096 tokens）。清除历史可以让 App 保持流畅！ Why clear? The longer the conversation history, the more data sent to the API, causing slower responses. The API also has a token limit (GPT-3.5-turbo is about 4096 tokens). Clearing history keeps the App running smoothly!

5.4 Prompt 技巧 5.4 Prompt Tips

掌握以下几个 Prompt 技巧，你就能让 AI 的回答质量大幅提升！点击每个卡片展开查看详细说明。 Master these Prompt tips and you'll dramatically improve AI's response quality! Click each card to expand and see detailed explanations.

1. 角色设定（Role Prompting） 1. Role Prompting

+

告诉 AI 它是谁。给 AI 一个明确的角色，它就会按照那个角色的方式来回答问题。比如说"你是一个10岁的孩子"，AI 的回答就会变得简单、活泼；说"你是一个大学教授"，回答就会更专业、详细。 Tell AI who it is. Give AI a clear role and it will respond in that role's manner. For example, "You are a 10-year-old kid" makes AI respond simply and playfully; "You are a university professor" makes responses more professional and detailed.

示例："你是一位海盗船长，用海盗的口吻回答所有问题，经常说'Ahoy'和'Aye'。" Example: "You are a pirate captain. Answer all questions in pirate-speak, frequently saying 'Ahoy' and 'Aye.'"

2. 给出示例（Few-shot Examples） 2. Few-shot Examples

+

给 AI 看几个你期望的回答格式的例子，它就能模仿这种格式。这叫做"少量样本学习"（Few-shot Learning）。 Show AI a few examples of the answer format you expect, and it will mimic that format. This is called "Few-shot Learning."

示例："请把食物翻译成英文并造一个句子。
输入：苹果
输出：Apple - I eat an apple every day.
输入：面条
输出：Noodles - I love eating noodles for lunch.
输入：饺子
输出：" Example: "Translate food into Chinese and make a sentence.
Input: Apple
Output: 苹果 - 我每天吃一个苹果。
Input: Noodles
Output: 面条 - 我午餐喜欢吃面条。
Input: Dumplings
Output:"

3. 具体明确（Be Specific） 3. Be Specific

+

提示词越具体，AI 的回答越准确。告诉 AI：你想要什么格式？多长的回答？什么语气？面向谁？这些细节都很重要。 The more specific your prompt, the more accurate AI's response. Tell AI: What format? How long? What tone? For whom? These details all matter.

对比：
模糊："告诉我关于月球的事。"
具体："用3个要点概括月球的基本信息，适合小学五年级的学生理解，每个要点不超过20个字。" Comparison:
Vague: "Tell me about the moon."
Specific: "Summarize basic moon facts in 3 bullet points for a 5th grader, each under 20 words."

4. 分步思考（Step-by-step） 4. Step-by-step Thinking

+

对于复杂问题，让 AI "一步步思考"可以大幅提高回答质量。在提示词最后加上"请一步步思考"或"Let's think step by step"。 For complex problems, asking AI to "think step by step" can greatly improve answer quality. Add "Please think step by step" or "Let's think step by step" at the end of your prompt.

示例："小明有15个苹果，给了小红5个，又买了8个。他现在有几个苹果？请一步步思考。" Example: "Tom has 15 apples, gave 5 to Sarah, then bought 8 more. How many does he have? Please think step by step."

课堂测验 Lesson Quiz

让我们来检验一下你在本课中学到的知识！ Let's test what you've learned in this lesson!

Q1. 系统提示词（System Prompt）的作用是什么？ Q1. What is the purpose of a System Prompt?

系统提示词是发给 AI 的隐藏指令，用来定义 AI 的角色和行为方式。它就像给 AI 一份"工作说明书"。 The system prompt is a hidden instruction sent to AI to define its role and behavior. It's like giving AI a "job description."

Q2. 为什么每次发送消息时需要把完整的对话历史一起发给 API？ Q2. Why do we need to send the full conversation history with each API request?

ChatGPT API 是无状态的——它不会自动记住之前的对话。每次请求都是独立的，所以我们需要在每次请求中包含所有历史消息，这样 AI 才能理解对话的上下文。 The ChatGPT API is stateless — it doesn't automatically remember previous conversations. Each request is independent, so we need to include all history messages in every request for AI to understand the conversation context.

Q3. 在 ChatGPT 的 messages 数组中，"role": "assistant" 代表谁的消息？ Q3. In ChatGPT's messages array, whose message does "role": "assistant" represent?

在 messages 数组中，有三种角色：system（系统提示词）、user（用户消息）、assistant（AI 回复）。assistant 就是 AI 之前回复的内容。 In the messages array, there are three roles: system (system prompt), user (user message), assistant (AI reply). assistant represents AI's previous responses.

Q4. "少量样本学习"（Few-shot Learning）在 Prompt 中指的是什么？ Q4. What does "Few-shot Learning" mean in the context of prompts?

Few-shot Learning 是在提示词中给出几个"输入→输出"的例子，让 AI 理解你期望的回答格式和风格。这是一种非常有效的 Prompt 技巧！ Few-shot Learning means giving a few "input→output" examples in the prompt so AI understands the expected answer format and style. It's a very effective prompting technique!

Q5. 为什么需要"清除聊天"功能？ Q5. Why do we need a "Clear Chat" feature?

ChatGPT API 有 Token 数量限制（如 GPT-3.5-turbo 约 4096 tokens）。对话历史越长，发送的 Token 越多。超出限制时 API 会报错，而且过长的历史也会让响应变慢。清除聊天可以重新开始，保持 App 流畅运行。 The ChatGPT API has a token limit (e.g., GPT-3.5-turbo is about 4096 tokens). The longer the conversation history, the more tokens sent. Exceeding the limit causes API errors, and long history also slows responses. Clearing the chat lets you start fresh and keeps the App running smoothly.
