4.2 AI API 入门4.2 AI API Basics

ChatGPT API 是什么？What is the ChatGPT API?

你可能已经在网页上用过 ChatGPT 聊天了。ChatGPT API 就是让程序和 App也能和 ChatGPT 对话的通道。网页版 ChatGPT 是给人用的界面，而 API 是给程序用的接口——它们背后使用的是同一个 AI 大脑！ You may have already chatted with ChatGPT on the website. The ChatGPT API is a channel that lets programs and Apps talk to ChatGPT too. The ChatGPT website is a human-facing interface, while the API is a program-facing interface — they both use the same AI brain!

ChatGPT 网站ChatGPT Website

你在浏览器里打字聊天，看到 AI 的回复。操作简单，适合个人使用。You type and chat in the browser and see AI's replies. Simple to use, great for personal use.

ChatGPT APIChatGPT API

你的 App 向 API 发送消息，AI 返回回复数据。你可以控制 AI 的"性格"、回复风格，嵌入到任何 App 中！Your App sends messages to the API and AI returns reply data. You can control AI's "personality", reply style, and embed it in any App!

什么是 API Key？What is an API Key?

API Key 就像一张"入场券"或"通行密码"。就像进入游乐园需要门票一样，你的 App 要调用 ChatGPT API 也需要一把专属的"钥匙"。这个 Key 是一串长长的字符，看起来像这样： An API Key is like an "entry ticket" or "access password." Just as you need a ticket to enter an amusement park, your App needs a special "key" to use the ChatGPT API. The Key is a long string of characters, looking something like this:

API Key 
sk-proj-aBcDeFgHiJkLmNoPqRsTuVwXyZ1234567890...

重要安全提示！API Key 就像你的银行密码一样重要！绝对不要把它分享给别人、发到聊天群里、或者上传到网上。如果别人拿到了你的 Key，他们可以用你的账号调用 API，花掉你的钱！如果你怀疑 Key 泄露了，立刻去 OpenAI 网站重新生成一个新的 Key。 Important Security Warning! An API Key is as important as your bank password! Never share it with others, post it in chat groups, or upload it online. If someone gets your Key, they can use your account to call the API and spend your money! If you suspect your Key is leaked, immediately go to the OpenAI website and generate a new one.

费用说明Pricing

OpenAI 提供免费的试用额度，新注册账号通常会有几美元的免费额度。ChatGPT API 按使用量收费——每次对话消耗一点点"token"（可以理解为文字量）。对于我们的课程练习来说，免费额度完全够用！如果老师提供了共用的 API Key，直接使用老师的 Key 即可。 OpenAI offers free trial credits — new accounts usually get a few dollars of free credits. The ChatGPT API charges by usage — each conversation consumes a few "tokens" (think of them as word counts). For our class exercises, the free credits are more than enough! If your teacher provides a shared API Key, just use the teacher's Key.

什么是 Token？Token 是 AI 处理文字的最小单位。一个英文单词大约是 1 个 token，一个中文汉字大约是 1-2 个 token。发送 "Hello, how are you?" 大约消耗 6 个 token。GPT-4o-mini 模型每 100 万个 token 只需大约 0.15 美元，非常便宜！ What is a Token? A token is the smallest unit AI uses to process text. One English word is roughly 1 token, and one Chinese character is roughly 1-2 tokens. Sending "Hello, how are you?" costs about 6 tokens. The GPT-4o-mini model costs only about $0.15 per million tokens — super affordable!

This time we need a POST request — because we're sending data (our message) to ChatGPT, not just fetching data. Switch to Blocks view and build these blocks:

GET vs POST：GET 就像去图书馆借书——你只是"拿"数据回来。POST 就像寄一封信——你把数据"送"到服务器，服务器处理后再给你回信。跟 AI 聊天需要 POST，因为你要把你的消息"送"给 AI。 GET vs POST: GET is like borrowing a book from the library — you just "take" data. POST is like mailing a letter — you "send" data to the server, and the server processes it and replies. Chatting with AI needs POST because you're "sending" your message to AI.

步骤 4：设置请求头（Headers）Step 4: Set Request Headers

ChatGPT API 需要两个特殊的请求头信息，就像寄信时要在信封上写清楚寄件人和信的格式： The ChatGPT API requires two special request headers, like writing the sender info and letter format on an envelope:

Authorization（身份验证）Authorization

告诉 API "我是谁"。格式为：
Bearer YOUR_API_KEY

在 Blocks 中，使用 set Web1.RequestHeaders to 块，构建一个列表（list），其中包含一对键值：["Authorization", "Bearer sk-proj-你的Key..."] Tells the API "who I am." Format:
Bearer YOUR_API_KEY

In Blocks, use the set Web1.RequestHeaders to block and build a list containing a key-value pair: ["Authorization", "Bearer sk-proj-yourKey..."]

Content-Type（内容类型）Content-Type

告诉 API "我发送的数据是什么格式"。我们发送的是 JSON，所以设置为：
application/json

在同一个列表中添加第二对键值：["Content-Type", "application/json"] Tells the API "what format my data is in." Since we're sending JSON:
application/json

Add a second key-value pair in the same list: ["Content-Type", "application/json"]

提示：在 App Inventor 中，RequestHeaders 需要设置成一个"列表的列表"格式。最外层是一个 list，里面每一项也是一个包含两个元素的 list（key 和 value）。用 make a list 块嵌套构建：make a list → [make a list("Authorization", "Bearer …"), make a list("Content-Type", "application/json")] Tip: In App Inventor, RequestHeaders need to be a "list of lists." The outer layer is a list, and each item is also a list with two elements (key and value). Build it using nested make a list blocks: make a list → [make a list("Authorization", "Bearer …"), make a list("Content-Type", "application/json")]

步骤 5：构建 JSON 请求体（Request Body）Step 5: Build the JSON Request Body

ChatGPT API 需要你发送一个 JSON 格式的"信件内容"，告诉 AI 使用哪个模型，以及你要说什么。格式如下： The ChatGPT API needs you to send a JSON-formatted "letter body" telling AI which model to use and what you want to say. The format looks like this:

JSON 
{
  "model": "gpt-4o-mini",
  "messages": [
    {
      "role": "system",
      "content": "You are a friendly assistant for kids. Reply in short, simple sentences."
    },
    {
      "role": "user",
      "content": "What is the tallest mountain in the world?"
    }
  ]
}

让我们理解每个部分： Let's understand each part:

model（模型）model

"gpt-4o-mini" 是我们使用的 AI 模型。它既聪明又便宜，非常适合学习使用。"gpt-4o-mini" is the AI model we're using. It's smart and affordable — perfect for learning.

system（系统角色）system role

这是给 AI 的"人设"指令。你可以告诉 AI 它应该扮演什么角色、用什么风格说话。AI 会按照这个指令回复。This is the AI's "persona" instruction. Tell AI what role to play and what style to use. AI will follow this instruction in its replies.

user（用户消息）user message

这是用户说的话。在我们的 App 中，这就是 txtUserMessage.Text 的内容——用户在文本框里输入的消息。This is what the user says. In our App, this is the content of txtUserMessage.Text — whatever the user types in the text box.

4.4 安全提示4.4 Security Tips

在使用 AI API 时，有一些重要的安全和成本注意事项： When using AI APIs, there are important security and cost considerations:

保护 API KeyProtect Your API Key

在我们的课堂练习中，API Key 直接写在积木块里是可以的。但如果你要把 App 发布给别人用，千万不要把 Key 直接写在 App 里！别人可以反编译你的 App 找到 Key。正式项目应该用后端服务器来保管 Key。 For our class exercises, putting the API Key directly in blocks is fine. But if you publish your App for others, never put the Key directly in the App! Others could decompile your App and find it. Real projects should use a backend server to manage Keys.

注意使用量Watch Your Usage

每次调用 API 都会消耗 token 并产生费用。虽然 GPT-4o-mini 很便宜，但如果你写了一个无限循环不停调用 API，费用会迅速累积！在 OpenAI 的控制台可以设置使用量上限（Usage Limit），建议设为每月 $5 来保护自己。 Every API call consumes tokens and costs money. While GPT-4o-mini is cheap, if you write an infinite loop that keeps calling the API, costs will pile up fast! In the OpenAI dashboard you can set a Usage Limit — we recommend setting it to $5/month to protect yourself.

速率限制Rate Limiting

API 有速率限制（Rate Limit），就像排队买奶茶——同一时间只能服务这么多人。如果你太频繁地发送请求，API 会返回错误。正常聊天完全没问题，但不要写程序疯狂地循环发送请求。 APIs have Rate Limits — like a milk tea shop that can only serve so many customers at once. If you send requests too frequently, the API will return an error. Normal chatting is perfectly fine, but don't write code that sends requests in a crazy loop.

课堂安全规则：1) 不要把 API Key 截图或分享到任何地方；2) 如果使用老师的共用 Key，不要恶意大量调用；3) 不要让 AI 生成不当内容；4) 发现 Key 泄露立即通知老师。遵守这些规则，大家才能一起愉快地学习！ Classroom Safety Rules: 1) Don't screenshot or share the API Key anywhere; 2) If using the teacher's shared Key, don't abuse it with excessive calls; 3) Don't ask AI to generate inappropriate content; 4) If a Key leak is discovered, notify the teacher immediately. Follow these rules so everyone can learn happily!

课堂测验Lesson Quiz

1. API Key 最像下面哪样东西？1. An API Key is most like which of the following?

API Key 就像一张入场券或通行证。API 服务器用它来确认"你是谁"，记录你的使用量，并防止未授权的访问。An API Key is like an entry ticket or pass. The API server uses it to confirm "who you are," track your usage, and prevent unauthorized access.

2. 为什么跟 ChatGPT API 对话要用 POST 请求而不是 GET 请求？2. Why do we use a POST request instead of GET when talking to the ChatGPT API?

GET 请求用来"获取"数据（比如查天气），POST 请求用来"发送"数据给服务器。和 AI 聊天时，你要把你的消息发给 AI 处理，所以用 POST。GET requests are for "getting" data (like checking weather). POST requests are for "sending" data to the server. When chatting with AI, you send your message for AI to process, so you use POST.

3. 在 ChatGPT API 的 JSON 请求中，"system" 角色的作用是什么？3. What is the purpose of the "system" role in the ChatGPT API's JSON request?

"system" 角色用来给 AI 设定背景和规则（也叫"人设"）。比如你可以让 AI 用简短的句子回答，扮演老师角色，或者只用中文回复等。The "system" role sets the AI's background and rules (also called "persona"). For example, you can tell AI to reply in short sentences, play a teacher role, or only reply in Chinese.

4. 如果你怀疑 API Key 被泄露了，应该怎么做？4. What should you do if you suspect your API Key has been leaked?

API Key 泄露可能会导致别人用你的账号调用 API 产生费用。正确做法是立刻去 OpenAI 控制台删除（撤销）旧的 Key，然后生成一个新的 Key。同时通知老师。A leaked API Key could let others use your account and rack up charges. The right move is to immediately go to the OpenAI dashboard, delete (revoke) the old Key, and generate a new one. Also notify your teacher.

5. ChatGPT API 返回的 JSON 中，AI 的回复文字在哪里？5. In the JSON returned by ChatGPT API, where is the AI's reply text?

ChatGPT API 的回复嵌套了好几层：先找 "choices" 列表，取第 1 项，再找 "message" 对象，最后取 "content" 字段——这就是 AI 的回复文字。The ChatGPT API response is nested several layers deep: find the "choices" list, take the 1st item, find the "message" object, then get the "content" field — that's the AI's reply text.
