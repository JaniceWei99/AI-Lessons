7.1 课程目标7.1 Course Goal

在这节课中，你将学习如何使用 App Inventor 内置的 SpeechRecognizer（语音识别）和 TextToSpeech（文字转语音）组件，并结合 ChatGPT API，打造一个能"听懂"你说话、"思考"问题、然后"说出"答案的智能语音助手 App。就像你口袋里的 Siri 或小爱同学一样！ In this lesson, you'll learn to use App Inventor's built-in SpeechRecognizer (speech-to-text) and TextToSpeech (text-to-speech) components, combined with the ChatGPT API, to build a smart voice assistant App that can "hear" your speech, "think" about your question, and "speak" the answer back. Just like having your own Siri or Alexa in your pocket!

7.2 语音技术入门7.2 Voice Technology Basics

你知道吗？全球每天有超过 10 亿次语音搜索！从 Siri、Alexa、小爱同学到 Google Assistant，语音助手已经成为我们日常生活的一部分。而这些助手的核心技术，就是 STT（语音转文字）和 TTS（文字转语音）。 Did you know? Over 1 billion voice searches happen every day worldwide! From Siri, Alexa, and XiaoAi to Google Assistant, voice assistants are now part of our daily lives. The core technologies powering them are STT (Speech-to-Text) and TTS (Text-to-Speech).

语音助手是怎么工作的？想象一下你和 Siri 说"今天天气怎么样"。Siri 的工作过程其实就像一条流水线： How do voice assistants work? Imagine you ask Siri "What's the weather today?" Siri's process works just like an assembly line:

1

听 (Listen)Listen

麦克风录下你的声音，STT 技术把声音波形转换成文字："今天天气怎么样"The mic records your voice, STT converts the sound waves into text: "What's the weather today"

2

理解 (Understand)Understand

AI 分析文字的意思，理解你在问天气信息AI analyzes the text meaning and understands you're asking for weather info

3

回答 (Respond)Respond

AI 生成回答文字："今天晴天，最高温度 25 度"AI generates a text answer: "It's sunny today, high of 25°C"

4

说 (Speak)Speak

TTS 技术把文字转成自然的语音播放出来TTS converts the text into natural speech and plays it out loud

STT — 语音转文字STT — Speech-to-Text

作用：把人说的话转换为文字
App Inventor 组件：SpeechRecognizer
原理：分析声音频率和模式，与语言模型匹配
例子：你对手机说"你好"→ 屏幕显示文字"你好"Purpose: Converts human speech into text
App Inventor Component: SpeechRecognizer
How it works: Analyzes sound frequencies and patterns, matches with language models
Example: You say "Hello" → screen shows text "Hello"

TTS — 文字转语音TTS — Text-to-Speech

作用：把文字转换为自然的语音
App Inventor 组件：TextToSpeech
原理：将文字拆分成音素，合成自然语音
例子：文字"今天天气很好"→ 手机用语音读出来Purpose: Converts text into natural speech
App Inventor Component: TextToSpeech
How it works: Breaks text into phonemes and synthesizes natural speech
Example: Text "The weather is nice today" → phone reads it aloud

7.3 动手实践：搭建语音助手7.3 Hands-On: Build the Voice Assistant

步骤 1：添加 SpeechRecognizer 组件Step 1: Add the SpeechRecognizer Component

在 App Inventor 的 Designer 界面中，从左侧组件面板找到 Media（媒体）分类，拖拽 SpeechRecognizer 到屏幕上。它是一个"不可见组件"（non-visible），会出现在屏幕下方的区域。 In the App Inventor Designer view, find the Media category in the left component panel and drag SpeechRecognizer onto the screen. It's a "non-visible component" and will appear in the area below the screen.

小提示：SpeechRecognizer 不会在屏幕上显示任何东西，但它在后台默默工作。当你调用它时，手机会弹出一个语音输入窗口，等你说完就自动返回识别结果。 Tip: SpeechRecognizer won't display anything on screen, but it works quietly in the background. When you call it, your phone shows a voice input popup, and after you finish speaking, it automatically returns the recognized text.

步骤 2：添加 TextToSpeech 组件Step 2: Add the TextToSpeech Component

同样在 Media 分类中，拖拽 TextToSpeech 到屏幕上。这也是一个不可见组件。在它的属性面板中，你可以设置： Also in the Media category, drag TextToSpeech onto the screen. This is also a non-visible component. In its properties panel, you can set:

Language（语言）Language

设置语音播报的语言。例如 zh-CN 代表中文，en-US 代表英文。Set the language for speech output. For example, zh-CN for Chinese, en-US for English.

Pitch & SpeechRate（音调和语速）Pitch & Speech Rate

Pitch 控制声音高低（1.0 正常），SpeechRate 控制说话快慢（1.0 正常）。试试调到 2.0 会变成什么样？Pitch controls how high/low the voice is (1.0 = normal). SpeechRate controls how fast it talks (1.0 = normal). Try setting it to 2.0 — what happens?

步骤 3：设计用户界面Step 3: Design the User Interface

现在来设计 App 的界面。我们需要以下可见组件： Now let's design the App's interface. We need these visible components:

A

麦克风按钮Mic Button

一个大大的圆形按钮，用来启动语音识别。可以放一个麦克风图标或文字"点击说话"。设置背景色为亮色（如蓝色），让用户一眼就能看到。A big round button to start voice recognition. Add a mic icon or text "Tap to Speak." Set a bright background color (like blue) so users notice it immediately.

B

状态标签Status Label

显示当前状态的文字标签。比如"准备就绪"、"正在听..."、"思考中..."、"正在说..."。帮助用户了解 App 正在做什么。A text label showing current status, like "Ready," "Listening...," "Thinking...," or "Speaking..." This helps users understand what the App is doing.

C

用户语音文字User Speech Text

一个 Label，用来显示语音识别的结果——你说了什么。让用户确认 App 是否听对了。A Label to display what the speech recognizer heard — what you said. This lets users confirm the App heard them correctly.

D

AI 回答区域AI Response Area

一个 Label 或 TextBox，用来显示 AI 回答的文字内容。可以设置稍大的字体，方便阅读。A Label or TextBox to display the AI's text response. Use a slightly larger font for easy reading.

步骤 4：点击麦克风 → 开始语音识别Step 4: Click Mic → Start Speech Recognition

切换到 Blocks（积木）编辑器，创建以下逻辑： Switch to the Blocks editor and create this logic:

积木逻辑：
当 MicButton.Click 被触发
  → 设置 StatusLabel.Text = "正在听..."
  → 调用 SpeechRecognizer1.GetText

这段逻辑的意思是：当用户按下麦克风按钮时，更新状态文字为"正在听..."，然后启动语音识别器，等待用户说话。 Block Logic:
when MicButton.Click
  → set StatusLabel.Text = "Listening..."
  → call SpeechRecognizer1.GetText

This means: when the user presses the mic button, update the status to "Listening..." and start the speech recognizer, waiting for the user to speak.

步骤 5：语音识别完成 → 发送到 ChatGPTStep 5: Speech Recognized → Send to ChatGPT

当语音识别完成后，SpeechRecognizer 会触发 AfterGettingText 事件，返回识别出的文字。我们把这段文字发送给 ChatGPT API： When speech recognition finishes, SpeechRecognizer triggers the AfterGettingText event and returns the recognized text. We send this text to the ChatGPT API:

积木逻辑：
当 SpeechRecognizer1.AfterGettingText (result)
  → 设置 UserTextLabel.Text = result
  → 设置 StatusLabel.Text = "思考中..."
  → 设置 Web1.Url = "https://api.openai.com/v1/chat/completions"
  → 设置 Web1.RequestHeaders = [Authorization: Bearer YOUR_API_KEY]
  → 调用 Web1.PostText，发送 JSON 请求体（包含 model、messages 等参数，message 内容为 result）

关键：记得把请求体中的 content 设为 result（用户说的话），这样 ChatGPT 就会根据用户的语音内容来回答。 Block Logic:
when SpeechRecognizer1.AfterGettingText (result)
  → set UserTextLabel.Text = result
  → set StatusLabel.Text = "Thinking..."
  → set Web1.Url = "https://api.openai.com/v1/chat/completions"
  → set Web1.RequestHeaders = [Authorization: Bearer YOUR_API_KEY]
  → call Web1.PostText with the JSON request body (including model, messages parameters, with message content = result)

Key point: Remember to set the content in the request body to result (what the user said), so ChatGPT will respond based on the user's spoken question.

步骤 6：收到 AI 回复 → 显示并朗读Step 6: AI Response Received → Display and Speak

当 ChatGPT 返回回答后，我们把回答文字显示到屏幕上，同时让 TextToSpeech 朗读出来： When ChatGPT returns a response, we display the answer text on screen and have TextToSpeech read it aloud:

积木逻辑：
当 Web1.GotText (responseContent)
  → 用 JSON 解析从 responseContent 中提取 AI 回复文字
    （路径：choices → 第一项 → message → content）
  → 设置 AIResponseLabel.Text = 解析出的回复文字
  → 设置 StatusLabel.Text = "正在说..."
  → 调用 TextToSpeech1.Speak (message = 解析出的回复文字)

魔法时刻：你会听到手机用语音把 AI 的回答"念"出来，就像真人在和你对话！ Block Logic:
when Web1.GotText (responseContent)
  → Use JSON parsing to extract the AI reply text from responseContent
    (path: choices → first item → message → content)
  → set AIResponseLabel.Text = parsed reply text
  → set StatusLabel.Text = "Speaking..."
  → call TextToSpeech1.Speak (message = parsed reply text)

Magic moment: You'll hear your phone "read" the AI's answer out loud — just like a real person talking to you!

步骤 7：添加语言选择功能Step 7: Add Language Selection

让你的语音助手支持多种语言！添加一个 ListPicker 或 Spinner（下拉选择器），让用户选择 TTS 的朗读语言： Make your voice assistant support multiple languages! Add a ListPicker or Spinner (dropdown selector) so users can choose the TTS language:

中文模式Chinese Mode

设置 TextToSpeech1.Language = "zh-CN"，AI 回复会用中文朗读。同时可以设置 System Prompt 让 ChatGPT 用中文回答。Set TextToSpeech1.Language = "zh-CN", and the AI reply will be spoken in Chinese. You can also set the System Prompt to have ChatGPT respond in Chinese.

英文模式English Mode

设置 TextToSpeech1.Language = "en-US"，AI 回复会用英文朗读。让 ChatGPT 用英文回答。Set TextToSpeech1.Language = "en-US", and the AI reply will be spoken in English. Have ChatGPT respond in English.

进阶技巧：在积木中使用 if/else 逻辑块来根据用户的语言选择动态切换。当用户选择"中文"时，把 System Prompt 设为"请用中文回答"，并把 TTS 语言设为 zh-CN；选择"English"时则相反。 Advanced tip: Use if/else logic blocks to dynamically switch based on the user's language choice. When the user selects "Chinese," set the System Prompt to "Please respond in Chinese" and TTS language to zh-CN; when "English" is selected, do the opposite.

7.4 改进体验7.4 Improving UX

一个好的语音助手不仅要功能正确，还要让用户用起来舒服。以下是一些改进用户体验的技巧： A great voice assistant needs to feel good to use, not just work correctly. Here are some tips for improving the user experience:

A

视觉反馈：麦克风动画Visual Feedback: Mic Animation

当用户按下麦克风按钮时，改变按钮颜色（例如变成红色）并更新文字为"正在听..."。当语音识别完成后恢复原来的颜色。这样用户就知道 App 正在认真"听"他说话。可以用 MicButton.BackgroundColor 积木来实现。When the user presses the mic button, change its color (e.g., to red) and update text to "Listening..." When recognition is done, restore the original color. This tells users the App is actively "listening." Use the MicButton.BackgroundColor block to implement this.

B

错误处理：没听到怎么办Error Handling: What If No Speech?

有时语音识别可能没有听到任何内容（用户没说话或环境太吵）。用 if result = "" 来判断，如果结果为空，显示友好的提示"抱歉没听清，请再说一次"，而不是让 App 崩溃或发送空消息给 AI。Sometimes speech recognition might not hear anything (user didn't speak or too noisy). Use if result = "" to check — if the result is empty, show a friendly message like "Sorry, I didn't catch that. Please try again" instead of crashing or sending an empty message to the AI.

C

状态管理State Management

在 AI 思考时禁用麦克风按钮（设置 MicButton.Enabled = false），防止用户重复点击。AI 回答播报完成后再重新启用按钮，准备下一轮对话。Disable the mic button while AI is thinking (set MicButton.Enabled = false) to prevent repeated taps. Re-enable it after the AI finishes speaking, ready for the next conversation round.

D

对话历史Conversation History

用一个列表保存之前的对话内容，这样用户可以向上滚动查看之前问过什么。可以用 ListView 组件来展示对话记录，就像微信聊天界面一样。Use a list to save previous conversation content so users can scroll up to see what they asked before. You can use the ListView component to display chat history, just like a WeChat chat interface.

7.5 挑战任务7.5 Challenge

挑战 A：语音翻译 AppChallenge A: Voice Translation App

制作一个语音翻译 App：用户说英文 → SpeechRecognizer 转成英文文字 → 发送给 ChatGPT 并要求它翻译成中文 → 把中文翻译结果用 TextToSpeech 朗读出来（设置语言为 zh-CN）。反过来也可以，说中文翻译成英文！这就是一个口袋里的翻译官。Build a voice translation App: user speaks English → SpeechRecognizer converts to text → send to ChatGPT asking it to translate to Chinese → TextToSpeech reads the Chinese translation aloud (language set to zh-CN). It works the other way too — speak Chinese and get English! It's like having a translator in your pocket.

挑战 B：添加"唤醒词"检测Challenge B: Add "Wake Word" Detection

让你的助手像真正的 Siri 一样，支持"唤醒词"功能！使用 Clock（时钟）组件每隔几秒自动调用一次 SpeechRecognizer，监听用户是否说了特定的唤醒词（比如"你好助手"）。如果检测到唤醒词，才开始正式的语音问答流程。提示：用 contains 积木来检查识别文字中是否包含唤醒词。Make your assistant work like real Siri with "wake word" support! Use the Clock component to automatically call SpeechRecognizer every few seconds, listening for a specific wake word (like "Hey Assistant"). Only start the full voice Q&A flow when the wake word is detected. Hint: Use the contains block to check if the recognized text includes the wake word.

课堂测验Lesson Quiz

1. STT 技术的全称是什么？1. What does STT stand for?

STT 是 Speech To Text 的缩写，意思是"语音转文字"。它是语音助手能"听懂"你说话的核心技术。STT stands for Speech To Text. It's the core technology that allows voice assistants to "understand" what you say by converting spoken words into text.

2. 在 App Inventor 中，TextToSpeech 组件属于哪个分类？2. In App Inventor, which category does the TextToSpeech component belong to?

TextToSpeech 和 SpeechRecognizer 都属于 Media（媒体）分类。在 App Inventor 的组件面板中，找到 Media 就能看到它们。Both TextToSpeech and SpeechRecognizer belong to the Media category. In App Inventor's component panel, look under Media to find them.

3. 语音助手的工作流程顺序是什么？3. What is the correct workflow order for a voice assistant?

语音助手的正确流程是：先"听"用户说话（STT），然后"理解"语义（AI 分析），接着"回答"问题（生成文字），最后"说"出来（TTS）。The correct flow is: "Listen" to the user (STT), then "Understand" the meaning (AI analysis), then "Respond" to the question (generate text), and finally "Speak" the answer (TTS).

4. 当 SpeechRecognizer 识别完成后，会触发哪个事件？4. When SpeechRecognizer finishes recognizing, which event is triggered?

SpeechRecognizer 识别完成后会触发 AfterGettingText 事件，并把识别出的文字作为参数 result 传递给你。After SpeechRecognizer finishes, it triggers the AfterGettingText event and passes the recognized text as the result parameter.

5. 如果语音识别没有听到任何内容，最佳的处理方式是什么？5. What is the best way to handle it when speech recognition doesn't hear anything?

好的用户体验需要优雅地处理错误。当没有识别到语音时，应该显示友好的提示信息（如"抱歉没听清，请再说一次"），而不是忽略错误或崩溃。Good UX means handling errors gracefully. When no speech is detected, show a friendly message (like "Sorry, I didn't catch that. Please try again") instead of ignoring the error or crashing.
