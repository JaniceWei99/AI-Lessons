2.2 核心概念2.2 Core Concepts

变量（Variables）Variables

变量就像一个有标签的盒子，可以在里面存放数据。比如你可以创建一个变量叫 score，把数字 100 存进去。之后你可以读取它、修改它、用它来做计算。在 App Inventor 中，你可以在 Blocks 视图的 Variables 类别中找到 initialize global name to 积木块来创建变量。 A variable is like a labeled box where you can store data. For example, you can create a variable called score and put the number 100 in it. Later you can read it, change it, or use it in calculations. In App Inventor, go to the Variables category in the Blocks view and use the initialize global name to block to create a variable.

积木块逻辑Block Logic 
// 创建变量 / Creating variables

initialize global num1 to 0
initialize global num2 to 0
initialize global result to 0

条件判断（Conditionals）Conditionals

条件判断让你的 App 能够"做决定"。就像生活中的选择一样：如果下雨 → 带伞；否则 → 戴墨镜。在 App Inventor 中，使用 Control 类别中的 if ... then ... else 积木块。你可以判断一个条件是否成立，然后执行不同的操作。 Conditionals let your App "make decisions." Just like choices in real life: if it's raining → bring an umbrella; otherwise → wear sunglasses. In App Inventor, use the if ... then ... else block from the Control category. You can check whether a condition is true and then perform different actions.

条件判断积木块可以嵌套和扩展。点击 if 积木块上的蓝色齿轮图标，你可以添加 else if 和 else 分支。这样就能处理多种不同的情况。比如：如果运算符是"+"就做加法，如果是"-"就做减法，以此类推。 Conditional blocks can be nested and extended. Click the blue gear icon on the if block to add else if and else branches. This lets you handle multiple different cases. For example: if the operator is "+" do addition, if it's "-" do subtraction, and so on.

列表（Lists）Lists

列表是一个可以存放多个数据的"容器"。想象一个购物清单 —— 上面可以有很多条内容。在 App Inventor 中，使用 Lists 类别中的积木块来创建和操作列表。你可以往列表中添加项目、读取某一项、或者获取列表的长度。列表在后面制作计算历史记录功能时会非常有用！ A list is a "container" that can hold multiple pieces of data. Think of a shopping list — it can have many items on it. In App Inventor, use blocks from the Lists category to create and manipulate lists. You can add items to a list, read a specific item, or get the list's length. Lists will be very useful later when we build a calculation history feature!

2.3 动手实践2.3 Hands-On

现在让我们一步步制作一个四则运算计算器 App！用户输入两个数字，点击运算按钮，App 会显示计算结果。 Now let's build a four-operation calculator App step by step! The user enters two numbers, taps an operation button, and the App displays the result.

步骤 1：设计 UI 界面Step 1: Design the UI

创建一个新项目，命名为 SimpleCalc。在 Designer 视图中，按照以下顺序添加组件： Create a new project named SimpleCalc. In the Designer view, add components in this order:

输入区域Input Area

① 添加一个 Label，Text 设为"第一个数字："
② 添加一个 TextBox（命名为 TextBox1），设置 NumbersOnly 为 true，Hint 设为"请输入数字"
③ 再添加一个 Label，Text 设为"第二个数字："
④ 再添加一个 TextBox（命名为 TextBox2），同样设置 NumbersOnly 为 true ① Add a Label with Text set to "First number:"
② Add a TextBox (named TextBox1), set NumbersOnly to true, Hint to "Enter a number"
③ Add another Label with Text "Second number:"
④ Add another TextBox (named TextBox2), also with NumbersOnly set to true

按钮和结果区域Buttons & Result Area

⑤ 添加一个 HorizontalArrangement（水平布局），在里面放 4 个 Button，Text 分别为 + - × ÷
⑥ 在按钮下方添加一个 Label（命名为 LabelResult），Text 设为"结果："，字体大小设为 20，FontBold 勾选 ⑤ Add a HorizontalArrangement, and place 4 Buttons inside it with Text: + - × ÷
⑥ Below the buttons, add a Label (named LabelResult), set Text to "Result:", font size to 20, and check FontBold

界面美化提示：你可以给每个 Button 设置不同的 BackgroundColor（比如加法用蓝色、减法用绿色、乘法用橙色、除法用红色），这样用户更容易分辨不同的运算。还可以设置 Button 的 Width 为 25%，让 4 个按钮均匀排列。 UI Design Tip: Give each Button a different BackgroundColor (e.g., blue for add, green for subtract, orange for multiply, red for divide) to help users distinguish operations. You can also set each Button's Width to 25% so all four buttons are evenly spaced.

步骤 2：创建变量Step 2: Create Variables

切换到 Blocks 视图。首先，我们需要创建 3 个全局变量来存储两个输入数字和计算结果： Switch to the Blocks view. First, we need 3 global variables to store the two input numbers and the result:

积木块逻辑Block Logic 
// 初始化变量 / Initialize variables

initialize global num1 to 0
initialize global num2 to 0
initialize global result to 0

步骤 3：编写运算逻辑Step 3: Program the Math Logic

为每个运算按钮添加点击事件。下面以加法按钮为例： Add a click event for each operation button. Here's the addition button as an example:

积木块逻辑 — 加法Block Logic — Addition 
// 加法按钮逻辑 / Addition button logic

When BtnAdd.Click
  do set global num1 to TextBox1.Text
     set global num2 to TextBox2.Text
     set global result to (get global num1) + (get global num2)
     set LabelResult.Text to join("结果 / Result: ", get global result)

减法、乘法按钮的逻辑类似，只需要将 + 替换为 - 或 ×。操作步骤：
① 从 Button 拖出 when BtnAdd.Click
② 从 Variables 拖出 set global num1 to，连接 TextBox1.Text
③ 同理设置 num2
④ 从 Math 拖出加法积木块 +，两侧分别连接 get global num1 和 get global num2
⑤ 将加法结果存入 result，然后用 set LabelResult.Text 显示出来 Subtraction and multiplication buttons follow the same pattern — just replace + with - or ×. Steps:
① Drag when BtnAdd.Click from Button
② Drag set global num1 to from Variables and connect TextBox1.Text
③ Do the same for num2
④ Drag the + math block from Math and connect get global num1 and get global num2 on each side
⑤ Store the result in the result variable, then display it with set LabelResult.Text

步骤 4：添加除零错误处理Step 4: Add Division-by-Zero Error Handling

除法需要特别注意 —— 不能除以零！我们要在除法按钮中加入条件判断： Division needs special attention — you can't divide by zero! We need to add a conditional check in the division button:

积木块逻辑 — 除法Block Logic — Division 
// 除法按钮（含错误处理） / Division button (with error handling)

When BtnDivide.Click
  do set global num1 to TextBox1.Text
     set global num2 to TextBox2.Text
     if (get global num2) = 0
       then set LabelResult.Text to "错误：不能除以零！/ Error: Cannot divide by zero!"
       else set global result to (get global num1) / (get global num2)
            set LabelResult.Text to join("结果 / Result: ", get global result)

为什么要做错误处理？在编程中，"错误处理"非常重要。如果用户输入了 0 作为除数，程序不做检查就会崩溃或显示错误结果。好的 App 应该提前预判可能出错的情况，给用户友好的提示，而不是让 App 直接崩溃。这是专业程序员的基本功！ Why error handling matters: In programming, "error handling" is critical. If a user enters 0 as the divisor and the program doesn't check, it will crash or show incorrect results. A good App anticipates possible errors and gives friendly messages instead of crashing. This is a fundamental skill for professional programmers!

2.4 进阶：多屏幕 App2.4 Advanced: Multi-Screen App

到目前为止，我们的 App 只有一个屏幕。但真正的 App 通常有多个屏幕 —— 比如一个"主页"和一个"设置页"。App Inventor 支持创建多个 Screen。 So far, our App only has one screen. But real apps usually have multiple screens — like a "home page" and a "settings page." App Inventor supports creating multiple Screens.

如何添加新屏幕：

1. 在 Designer 视图上方，点击 "Add Screen..." 按钮
2. 给新屏幕起个名字，比如 Screen2
3. 在新屏幕上设计不同的界面

屏幕之间传递数据：
使用 open another screen with start value 积木块可以打开新屏幕并传递数据。在新屏幕中用 get start value 获取传递过来的数据。例如，你可以把计算器的结果传到第二个屏幕来展示。

注意：App Inventor 建议不要创建太多屏幕（通常不超过 10 个），过多屏幕会影响 App 性能。 How to add a new screen:

1. In the Designer view, click the "Add Screen..." button at the top
2. Name the new screen, e.g., Screen2
3. Design a different interface on the new screen

Passing data between screens:
Use the open another screen with start value block to open a new screen and pass data. In the new screen, use get start value to retrieve the data. For example, you could pass the calculator's result to a second screen for display.

Note: App Inventor recommends not creating too many screens (usually no more than 10), as too many screens can affect app performance.

课堂测验Lesson Quiz

1. 在 App Inventor 中，"变量"（Variable）的作用是什么？1. In App Inventor, what is a "variable" used for?

变量就像一个有标签的盒子，用来存储数据（如数字、文字）。你可以创建、读取和修改变量的值。A variable is like a labeled box used to store data (like numbers or text). You can create, read, and modify the value stored in a variable.

2. 条件判断 if ... then ... else 积木块的作用是什么？2. What does the if ... then ... else conditional block do?

if ... then ... else 让程序可以"做决定"：如果条件成立（true），执行 then 部分的操作；否则执行 else 部分的操作。if ... then ... else lets the program "make decisions": if the condition is true, it runs the "then" actions; otherwise it runs the "else" actions.

3. TextBox 组件的 NumbersOnly 属性设为 true 后，会发生什么？3. What happens when a TextBox's NumbersOnly property is set to true?

当 NumbersOnly 设为 true 时，TextBox 只接受数字输入。在手机上会自动弹出数字键盘，方便用户输入。When NumbersOnly is set to true, the TextBox only accepts numeric input. On a phone, the numeric keyboard pops up automatically for easier input.

4. 为什么在计算器中需要对除法做特别的错误处理？4. Why does division need special error handling in a calculator?

在数学中，除以零是未定义的。如果不做检查，App 可能会崩溃或显示错误结果。好的做法是在执行除法之前检查除数是否为零，如果是则显示友好的错误提示。In math, division by zero is undefined. Without checking, the App could crash or show incorrect results. Best practice is to check if the divisor is zero before performing division, and show a friendly error message if it is.

5. 如何在 App Inventor 中将数据从 Screen1 传递到 Screen2？5. How do you pass data from Screen1 to Screen2 in App Inventor?

使用 open another screen with start value 打开新屏幕时可以附带数据。在新屏幕中用 get start value 来获取传递过来的数据。Use open another screen with start value to open a new screen with data attached. In the new screen, use get start value to retrieve the passed data.
