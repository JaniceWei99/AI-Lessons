6.2 AI 如何"看"图片 6.2 How AI "Sees" Images

计算机视觉简介
我们人类看到一朵花时，大脑会立刻识别出"这是一朵花"。但计算机看到的其实只是一堆数字！每张图片都是由成千上万个像素（pixel）组成的，每个像素有红、绿、蓝三个颜色值（0-255）。AI 要做的就是从这些数字中找出规律。 Introduction to Computer Vision
When we humans see a flower, our brain instantly recognizes "this is a flower." But what a computer actually sees is just a bunch of numbers! Every image is made up of thousands of pixels, and each pixel has red, green, and blue color values (0-255). AI's job is to find patterns in these numbers.

AI 识别图片的过程可以简单理解为三个步骤： The process of AI recognizing images can be simply understood in three steps:

1

像素输入Pixel Input

AI 接收图片的所有像素数据——比如一张 224×224 的图片就有超过 15 万个像素点AI receives all pixel data — for example, a 224×224 image has over 150,000 pixels

2

特征提取Feature Extraction

AI 从像素中识别出边缘、形状、纹理等特征——就像你先看到轮廓，再看到细节AI identifies edges, shapes, textures from pixels — like how you first see outlines, then details

3

分类识别Classification

AI 将这些特征与学过的知识对比，判断图片中是什么——"这是一只猫！置信度 95%"AI compares these features with its knowledge and determines what's in the image — "This is a cat! Confidence: 95%"

现在的 AI 模型，比如 GPT-4 Vision 和 Gemini，不仅能识别物体是什么，还能描述图片中的场景、阅读图片中的文字，甚至回答关于图片的问题。这些模型被称为多模态模型（Multimodal Model），因为它们同时能处理文字和图片。 Modern AI models like GPT-4 Vision and Gemini can not only identify what objects are, but also describe scenes in images, read text in pictures, and even answer questions about images. These models are called Multimodal Models because they can process both text and images simultaneously.

有趣的事实：训练一个图像识别 AI 需要让它看数百万张图片！比如 ImageNet 数据集就包含了超过 1400 万张标注好的图片。AI 就是通过这些海量数据学会"看"东西的。 Fun fact: Training an image recognition AI requires showing it millions of images! For example, the ImageNet dataset contains over 14 million labeled images. AI learns to "see" things by studying this massive amount of data.

现在让我们一步步搭建一个"拍照识物"App！拍一张照片，AI 就能告诉你照片里是什么。 Now let's build a "Photo Identifier" App step by step! Take a photo and AI will tell you what's in it.

当用户点击"拍照识别"按钮时，我们调用 Camera 组件打开手机摄像头。 When the user clicks "Take & Identify," we call the Camera component to open the phone camera.

积木逻辑Block Logic 
// 当点击拍照按钮时 / When Take Photo button is clicked:
when btnTakePhoto.Click do

  set global imagePath = image
  // 调用识别函数 / Call identification function
  call identifyImage

步骤 3：将图片转换为 Base64Step 3: Convert Image to Base64

要把图片发送给 AI API，我们需要将图片编码为 Base64 格式——这是一种把图片变成纯文本字符串的方法，这样就能放进 JSON 请求中发送了。 To send an image to the AI API, we need to encode it in Base64 format — a method of converting images into plain text strings so they can be included in JSON requests.

什么是 Base64？Base64 是一种编码方式，它把任何二进制数据（如图片、音频）转换成由字母、数字和少量符号组成的文本。一张 100KB 的图片转成 Base64 后大约是 133KB 的文本。虽然变大了，但好处是可以作为纯文本传输。 What is Base64? Base64 is an encoding method that converts any binary data (like images, audio) into text made of letters, numbers, and a few symbols. A 100KB image becomes about 133KB of Base64 text. It gets bigger, but the advantage is it can be transmitted as plain text.

在 App Inventor 中，我们可以使用扩展插件或者将图片路径发给服务器来处理。一种简单的方法是使用 Base64 编码扩展（Extension）来完成转换。 In App Inventor, we can use an extension or send the image path to a server for processing. A simple method is to use a Base64 encoding extension to handle the conversion.

步骤 4：调用 Gemini Vision APIStep 4: Call the Gemini Vision API

为什么推荐 Gemini？Google 的 Gemini API 提供免费额度，非常适合学习和实验。它支持图片理解，而且注册简单。当然，你也可以使用 GPT-4 Vision API（需要付费的 OpenAI 账户）。两者的调用方式类似。 Why Gemini? Google's Gemini API offers a free tier, making it perfect for learning and experimenting. It supports image understanding and has simple registration. Of course, you can also use the GPT-4 Vision API (requires a paid OpenAI account). Both work similarly.

发送图片给 Gemini Vision API 的请求格式如下： Here's the request format for sending an image to the Gemini Vision API:

JSON 
// Gemini Vision API 请求示例
// Gemini Vision API request example
{
  "contents": [{
    "parts": [
      {"text": "What is in this image? Describe it in detail."},
      {
        "inline_data": {
          "mime_type": "image/jpeg",
          "data": "<BASE64_ENCODED_IMAGE>"
        }
      }
    ]
  }]
}

// API 端点 / API endpoint:
// https://generativelanguage.googleapis.com/v1beta/models/
//   gemini-1.5-flash:generateContent?key=YOUR_API_KEY

在 Blocks 中的逻辑： Logic in Blocks:

积木逻辑Block Logic 
// 识别图片的过程 / Procedure to identify image:
procedure identifyImage

  // 1. 读取图片文件并转为 Base64
  // 1. Read image file and convert to Base64
  set base64Image = call Base64.Encode(global imagePath)

  // 2. 构建 API 请求 JSON
  // 2. Build the API request JSON
  set requestBody = make a dictionary
    "contents" → list [
      dictionary {
        "parts" → list [
          dictionary {"text": "Identify what is in this image. Tell me the name, a brief description, and any interesting facts."},
          dictionary {"inline_data": {"mime_type": "image/jpeg", "data": base64Image}}
        ]
      }
    ]

  // 3. 设置 URL 并发送请求
  // 3. Set URL and send request
  set Web1.Url = "https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=YOUR_API_KEY"
  set Web1.RequestHeaders = [["Content-Type", "application/json"]]
  call Web1.PostText(JsonEncode(requestBody))

步骤 5：解析识别结果Step 5: Parse the Recognition Result

当 API 返回结果后，我们需要从 JSON 响应中提取 AI 的描述文字。 When the API returns a result, we need to extract AI's description text from the JSON response.

积木逻辑Block Logic 
// 当收到 API 回复时 / When Web got text response:
when Web1.GotText(responseCode, responseContent) do

  if responseCode = 200 then
    // 解析 JSON 响应 / Parse JSON response
    set jsonResponse = call JsonTextDecode(responseContent)

    // 提取 AI 的识别描述
    // Extract AI's identification description
    set resultText = jsonResponse
      .candidates[1].content.parts[1].text

    // 显示结果 / Display the result
    set lblResult.Text = resultText
    set lblStatus.Text = "识别完成！ / Identification complete!"

  else
    // 显示错误 / Show error
    set lblStatus.Text = "出错了，请重试 / Error, please try again"

步骤 6：显示识别结果Step 6: Display the Results

为了让结果更直观，我们可以优化显示效果： To make results more visual, we can enhance the display:

结果展示Result Display

将 lblResult 设置为多行显示，字体大小设为 16，背景色设为浅色。AI 返回的描述通常包含物体名称和详细说明，直接显示即可。 Set lblResult to multi-line display, font size 16, with a light background color. AI's description typically includes the object name and detailed explanation — just display it directly.

用户体验优化UX Optimization

在等待 API 响应时禁用拍照按钮，防止重复请求。识别完成后再启用按钮。还可以添加一个小动画或进度提示让用户知道正在处理。 Disable the photo button while waiting for API response to prevent duplicate requests. Re-enable the button after identification. You can also add a small animation or progress indicator to show processing.

GPT-4 Vision 替代方案：如果你使用 OpenAI 的 GPT-4 Vision，请求格式稍有不同。把 messages 数组中的 content 改为一个数组，其中包含文本部分和图片 URL 部分：
{"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,BASE64_DATA"}} GPT-4 Vision alternative: If using OpenAI's GPT-4 Vision, the request format differs slightly. Change content in the messages array to an array with a text part and an image URL part:
{"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,BASE64_DATA"}}

6.4 植物/动物识别 6.4 Plant/Animal Identification

通过修改发送给 AI 的提示词，我们可以让 App 针对特定领域进行识别。这就是 Prompt 工程的魅力！ By changing the prompt sent to AI, we can make the App specialize in specific areas. This is the power of Prompt Engineering!

植物识别提示词Plant Identification Prompt

将发送给 API 的 text 部分改为：

"请识别这张图片中的植物。告诉我：
1. 植物的名称（中英文）
2. 它的主要特征
3. 它是否可以食用
4. 养护建议（浇水、光照需求）
请用简洁友好的语言回答。" Change the text part sent to the API to:

"Identify the plant in this image. Tell me:
1. The plant's name (in Chinese and English)
2. Its main characteristics
3. Whether it's edible
4. Care tips (watering, sunlight needs)
Please answer in concise, friendly language."

动物识别提示词Animal Identification Prompt

同样修改提示词：

"请识别这张图片中的动物。告诉我：
1. 动物名称和品种（中英文）
2. 它的生活习性
3. 它生活在哪些地区
4. 一个有趣的冷知识
请用适合小学生理解的语言回答。" Similarly modify the prompt:

"Identify the animal in this image. Tell me:
1. Animal name and breed (in Chinese and English)
2. Its behavior and habits
3. Where it lives
4. A fun interesting fact
Please answer in language suitable for elementary students."

你甚至可以在 App 中添加一个下拉菜单（Spinner）让用户选择识别模式："通用识别"、"植物识别"、"动物识别"。不同的模式使用不同的提示词，让 AI 给出更专业的回答！ You can even add a Spinner (dropdown menu) in the App for users to choose recognition mode: "General," "Plant," "Animal." Different modes use different prompts, making AI give more specialized answers!

进阶技巧：你还可以要求 AI 用 JSON 格式返回结果，这样更方便解析。例如在提示词最后加上：
"请用以下 JSON 格式返回结果：{\"name\": \"植物名\", \"edible\": true/false, \"description\": \"描述\", \"care_tips\": \"养护建议\"}" Advanced tip: You can also ask AI to return results in JSON format for easier parsing. Add this to the end of your prompt:
"Return the result in this JSON format: {\"name\": \"plant name\", \"edible\": true/false, \"description\": \"description\", \"care_tips\": \"care tips\"}"

课堂测验 Lesson Quiz

让我们来检验一下你在本课中学到的知识！ Let's test what you've learned in this lesson!

Q1. 计算机"看到"一张图片时，实际处理的是什么？ Q1. When a computer "sees" an image, what does it actually process?

计算机看到的图片实际上是由大量像素组成的数字矩阵。每个像素包含红（R）、绿（G）、蓝（B）三个颜色通道的数值（0-255）。AI 就是从这些数字中学习识别物体的。 What a computer sees is actually a matrix of numbers made up of many pixels. Each pixel contains values (0-255) for three color channels: Red (R), Green (G), and Blue (B). AI learns to recognize objects from these numbers.

Q2. Base64 编码的作用是什么？ Q2. What is the purpose of Base64 encoding?

Base64 编码将二进制数据（如图片）转换为由字母、数字和少量符号组成的纯文本字符串。这样图片就可以作为文本放进 JSON 请求中发送给 API。注意：Base64 不是压缩，转换后文件反而会稍微变大（约增加 33%）。 Base64 encoding converts binary data (like images) into plain text strings of letters, numbers, and a few symbols. This allows images to be included as text in JSON requests to APIs. Note: Base64 is not compression — the encoded result is actually slightly larger (about 33% bigger).

Q3. GPT-4 Vision 和 Gemini 被称为什么类型的模型？ Q3. What type of model are GPT-4 Vision and Gemini called?

GPT-4 Vision 和 Gemini 是多模态模型——它们能同时处理和理解文字、图片等多种类型的数据。"多模态"意味着"多种数据类型"。普通的 GPT-3.5 只能处理文字，是单模态的。 GPT-4 Vision and Gemini are multimodal models — they can process and understand multiple types of data like text and images simultaneously. "Multimodal" means "multiple data types." Regular GPT-3.5 only handles text, making it unimodal.

Q4. 在拍照识物 App 中，如果想让 AI 专门识别植物，应该怎么做？ Q4. In the Photo Identifier App, how do you make AI specifically identify plants?

通过修改文本提示词（Prompt），我们可以引导 AI 专注于特定领域。例如将提示词改为"识别这张图片中的植物，告诉我它的名字和养护建议"，AI 就会专注于植物识别。这就是 Prompt 工程的妙用！ By modifying the text prompt, we can guide AI to focus on specific areas. For example, changing the prompt to "Identify the plant in this image, tell me its name and care tips" makes AI focus on plant identification. This is the magic of Prompt Engineering!

Q5. AI 图像识别的正确流程顺序是什么？ Q5. What is the correct order of the AI image recognition process?

AI 图像识别的正确流程是：首先接收像素输入（图片数据），然后从中提取特征（边缘、形状、纹理等），最后进行分类识别（判断是什么物体）。就像人先看到整体，再注意细节，最后做出判断。 The correct AI image recognition flow is: first receive pixel input (image data), then extract features (edges, shapes, textures, etc.), and finally classify (determine what the object is). It's like how humans first see the whole picture, then notice details, and finally make a judgment.
