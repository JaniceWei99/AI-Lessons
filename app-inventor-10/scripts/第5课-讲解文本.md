## 导入环节

> 上节课我们成功让 App 和 ChatGPT 对上话了！但是大家有没有发现两个问题？第一，每次对话 AI 都不记得之前说了什么；第二，AI 的回复风格比较普通，没有什么个性。
>
> 今天我们要解决这两个问题。首先，我要教大家一个超级厉害的技能——**Prompt Engineering（提示词工程）**。这可是现在最热门的 AI 技能之一！然后我们让 AI 拥有"记忆"，记住整个对话历史。

## 核心概念讲解

### Prompt Engineering——和 AI 对话的艺术

> Prompt 就是你发给 AI 的指令。同样一个问题，不同的问法会得到完全不同的结果。我来举个例子：
>
> **弱 Prompt：** "告诉我关于太阳系的知识。"
> **强 Prompt：** "你是一位小学科学老师，请用简单有趣的语言，用5个要点向10岁小朋友介绍太阳系，每个要点附带一个有趣的类比。"
>
> 看到区别了吗？强 Prompt 指定了角色、语气、格式和目标，结果就会精准很多。

### System Prompt——给 AI 设定"人格"

> 在 ChatGPT API 中，有一种特殊的消息叫 **System Prompt**（系统提示词）。它就像给 AI 写的"角色说明书"，放在对话的最开头，告诉 AI 它是谁、应该怎么说话。
>
> 这里给大家 6 个有趣的 System Prompt 示例：
>
> 1. **科学导师：** "你是一位友好的科学老师，用简单易懂的语言解释科学概念，多用生活中的例子。"
> 2. **故事大王：** "你是一位想象力丰富的故事大王，擅长讲有趣的冒险故事，主角总是勇敢的小动物。"
> 3. **翻译官：** "你是一位专业翻译，用户输入中文你翻译成英文，输入英文你翻译成中文。"
> 4. **数学教练：** "你是一位耐心的数学教练，一步一步引导学生解题，不直接给答案。"
> 5. **宠物顾问：** "你是一位宠物养育专家，帮助小朋友了解如何正确照顾各种宠物。"
> 6. **心情日记：** "你是一位温暖的心理辅导员，帮助用户记录和理解自己的情绪。"

### 三种消息角色

> ChatGPT API 的每条消息都有一个"角色"标签：
>
> - **system**：系统设定，放在最前面，定义 AI 的行为和人格
> - **user**：用户说的话
> - **assistant**：AI 的回复
>
> 我们发给 API 的消息列表就是按这个顺序排列的：先是一条 system 消息，然后是 user 和 assistant 交替出现。

## 操作指导

### 步骤一：添加 System Prompt 输入框

> 在上节课的项目基础上继续。回到 Designer 视图：
>
> 在聊天区域的上方加一个 **TextBox**（命名为 TxtSystemPrompt），Hint 写"输入 AI 的角色设定..."，可以给默认 Text 写"你是一位友好的AI助手"。

### 步骤二：创建对话历史列表变量

> 切换到 Blocks 视图。这是今天最关键的部分：
>
> 1. 创建一个全局变量 `initialize global chatHistory to`，连接 `create empty list`。这个列表将保存所有对话记录。
> 2. 当用户点击发送时，先把 system 消息加入列表（如果是第一次发送），然后把 user 消息加入列表。
> 3. 把整个对话历史列表转换成 JSON 格式发送给 API。
> 4. 收到 AI 回复后，把 assistant 消息也加入列表。
>
> 这样每次发送，AI 都能看到之前所有的对话内容，就实现了"记忆"功能！
>
> **我在屏幕上演示具体的积木块怎么拼。大家注意看列表操作的部分——用 `add items to list` 往 chatHistory 里添加每条消息。**

### 步骤三：制作聊天气泡效果

> 为了让聊天界面更好看，我们给用户消息和 AI 消息用不同的颜色：
>
> - 用户消息：蓝色背景，靠右显示
> - AI 消息：灰色背景，靠左显示
>
> 在 LabelChat 中，用 `join` 积木块把每条消息拼接在一起，格式为：
> - `"你: " + 用户输入 + "\n"`（用户消息）
> - `"AI: " + AI回复 + "\n\n"`（AI 消息）
>
> 这样聊天记录就会一条条显示在滚动区域里。

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| AI 不记得之前的对话 | 检查是否每次都发送完整的 chatHistory 列表 |
| System Prompt 没有效果 | 确认 system 消息放在列表的第一个位置 |
| 对话越来越慢 | 历史太长了，可以限制只保留最近 10 条消息 |
| JSON 格式错误 | 检查消息角色是否只用了 system/user/assistant 三种 |
| 聊天记录显示不全 | 确认 VerticalScrollArrangement 的 Scrollable 属性已开启 |

## 总结与回顾

> 今天我们学了三个重要内容：
>
> 1. **Prompt Engineering** —— 好的提示词 = 好的 AI 回复。记住要指定角色、语气和格式。
> 2. **System Prompt** —— 给 AI 设定人格，让它扮演不同角色。
> 3. **对话历史** —— 用列表保存所有消息，每次都发送完整历史，AI 就有了"记忆"。
>
> 现在你已经能做一个功能完整的 AI 聊天 App 了！下节课我们要解锁 AI 的另一个超能力——让它"看懂"图片！

## 课后延伸

1. 试试设定不同的 System Prompt，看看 AI 的回答风格会怎么变化。
2. 设计你自己的专属 AI 角色，写一段有创意的 System Prompt。
3. 挑战：添加一个"清除对话"按钮，点击后清空 chatHistory 列表和聊天显示区域。
