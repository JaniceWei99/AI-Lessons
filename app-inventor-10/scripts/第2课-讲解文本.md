App Inventor 快速入门（下）—— 深入组件与逻辑，制作互动计算器 App

## 核心概念讲解

### 变量（Variables）

> 变量是什么？你可以把它想象成一个**贴了标签的盒子**。比如我有一个盒子，上面写着"score"，里面放着数字 100。以后我想知道分数是多少，就打开这个盒子看；我想改分数，就换掉里面的东西。
>
> 在 App Inventor 中，到 Blocks 视图的 **Variables** 类别，拖出 `initialize global name to` 这个积木块。把 name 改成你想要的名字，比如 `num1`，后面接一个数字 0，就创建好了一个变量。

### 条件判断（Conditionals）

> 条件判断就是让 App 学会"做选择"。就像你每天出门：**如果**下雨，**就**带伞；**否则**，就戴墨镜。
>
> 在 App Inventor 中，去 **Control** 类别找到 `if...then...else` 积木块。你可以点击蓝色齿轮图标，添加更多的 `else if` 分支来处理多种情况。
>
> 今天做计算器的时候，我们就要用条件判断来检查：除数是不是零？如果是零就报错，不是零才计算。

### 列表（Lists）

> 列表就是一个"数据容器"，可以装很多东西。想象一个购物清单——苹果、牛奶、面包……你可以往里面加东西、删东西、看第几个是什么。
>
> 在后面的课程中，列表会非常有用。今天先简单了解就好。

## 操作指导

### 步骤一：设计计算器 UI（教师演示，学生跟做）

> 新建项目，名字叫 `SimpleCalc`。现在跟我一起添加组件：
>
> 1. 拖入一个 **Label**，Text 写"第一个数字"。
> 2. 拖入一个 **TextBox**，这里注意——在右侧属性面板找到 **NumbersOnly**，勾选它！这样用户只能输入数字，不会乱输字母。Hint 可以写"请输入数字"。
> 3. 同样的方式再加一个 Label 写"第二个数字"和第二个 TextBox。
> 4. 接下来拖入一个 **HorizontalArrangement**（水平布局），在它里面放 4 个 Button，Text 分别是 `+`、`-`、`x`、`/`。
> 5. 最后加一个 **Label** 用来显示结果，Text 写"结果："，字号设 20，加粗。
>
> 大家先把界面搭好，做好了跟我说。

### 步骤二：创建变量

> 切换到 Blocks 视图。先创建三个变量：
> - `initialize global num1 to 0`
> - `initialize global num2 to 0`
> - `initialize global result to 0`

### 步骤三：编写加法逻辑（教师一步步带做）

> 跟我一起做加法按钮的逻辑：
>
> 1. 点左侧的 **BtnAdd**（加法按钮），拖出 `when BtnAdd.Click do`。
> 2. 从 Variables 拖出 `set global num1 to`，连上 `TextBox1.Text`。
> 3. 同样设置 `set global num2 to` 连上 `TextBox2.Text`。
> 4. 从 **Math** 类别拖出加法积木块，左右分别连上 `get global num1` 和 `get global num2`。
> 5. 把加法结果存进 `set global result to`，然后用 `set LabelResult.Text to` 显示出来。
>
> 做好加法之后，减法和乘法的逻辑完全一样，就是把 `+` 换成 `-` 或 `x`。大家自己完成减法和乘法，给你们 5 分钟时间。

### 步骤四：除法——错误处理（重点讲解）

> 除法比较特殊。问大家一个问题：5 除以 0 等于多少？（让学生思考）对！不能除以零，这是数学规定。如果我们不做检查，App 就会崩溃或者显示奇怪的结果。
>
> 所以除法按钮里要加一个 **if 判断**：如果 num2 等于 0，就在结果标签显示"错误：不能除以零！"；否则才执行除法运算。

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 计算结果不对 | 检查是否先把 TextBox 的文字存入变量再做运算 |
| 输入了字母导致报错 | 确保 TextBox 的 NumbersOnly 属性已勾选 |
| 四个按钮挤在一起 | 把它们放在 HorizontalArrangement 中，每个按钮 Width 设为 25% |
| 除零没有报错 | 检查 if 条件是否正确：`get global num2 = 0` |
| 小数点结果太长 | 可以后续学习用 Math 中的 format 积木块限制小数位数 |

## 总结与回顾

> 今天我们学了三个重要的编程概念：
> - **变量**——有标签的盒子，存数据
> - **条件判断**——让 App 做选择，if/then/else
> - **列表**——装多个数据的容器
>
> 我们还做了一个计算器 App，特别是学会了**错误处理**——好的程序员一定会考虑"用户可能会犯什么错"，然后提前做好防护。这叫做**防御式编程**。
>
> 到目前为止，我们的 App 都是在手机本地运行的。从下节课开始，我们要连接互联网，调用 API，让 App 获取外面世界的真实数据！

## 课后延伸

1. 给计算器添加一个"清除"按钮（Clear），点击后清空两个 TextBox 和结果 Label。
2. 尝试美化 UI：给每个运算按钮设不同的颜色（加法蓝色、减法绿色、乘法橙色、除法红色）。
3. 思考：如果用户什么都没输入就按了按钮，会发生什么？能不能也加一个错误处理？
