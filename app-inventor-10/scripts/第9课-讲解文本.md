# 第9课：开发 + 调试

## 课程标题

开发 + 调试 —— 把设计变成现实，学会像专业程序员一样排错

## 教学目标

1. 理解专业开发周期：计划 → 搭建 → 测试 → 修复 → 重复
2. 掌握"一次只做一件事"的开发黄金法则
3. 认识 5 种最常见的 Bug 及其解决方法
4. 学会使用 App Inventor 的 "Do It" 调试功能和 Notifier 调试消息
5. 完成个人 AI App 项目的核心功能开发

## 导入环节

> 同学们，今天是正式的开发日！上节课你们都做好了设计方案，今天就要把设计变成真正能运行的 App。
>
> 但在开始之前，我要先教大家一个专业程序员都遵守的原则，能帮你们少走很多弯路。

## 核心概念讲解

### 专业开发周期

> 专业的软件开发不是一口气从头写到尾的，而是一个循环：
>
> **计划（Plan）→ 搭建（Build）→ 测试（Test）→ 修复（Fix）→ 重复（Repeat）**
>
> 具体来说：先做一小块功能，测试通过了再做下一块。不要一次把所有积木块都堆上去再测试，那样出了问题根本不知道是哪一块的错。

### 黄金法则：一次只做一件事

> 这是今天最重要的一句话：**一次只做一件事！**
>
> 比如你的 App 有三个功能：拍照、识别、显示结果。正确的做法是：
> 1. 先做拍照功能 → 测试 → 通过
> 2. 再做识别功能 → 测试 → 通过
> 3. 最后做显示功能 → 测试 → 通过
>
> 每完成一个功能就测试一次。如果出了 Bug，你就知道是刚才加的那部分出了问题，很容易定位。

### 5 大常见 Bug

> 在你们开发过程中，最可能遇到这 5 种 Bug，我提前告诉你们怎么解决：
>
> **Bug 1：API 调用报错**
> - 原因：URL 拼写错误、请求头没设置、GET/POST 用错了
> - 解决：仔细对照课上的积木块，一个字母一个字母检查 URL
>
> **Bug 2：JSON 解析失败**
> - 原因：Key 名字拼错（区分大小写）、路径层级不对
> - 解决：先用 Notifier 显示完整的 API 返回结果，看看实际的 JSON 结构
>
> **Bug 3：App 闪退或崩溃**
> - 原因：缺少权限（相机、麦克风）、引用了不存在的组件
> - 解决：检查 Designer 视图中的组件是否都在；测试前确认手机权限已开启
>
> **Bug 4：API Key 失效或请求太频繁**
> - 原因：Key 过期、免费额度用完、短时间请求太多（429 错误）
> - 解决：重新生成 Key；等一会儿再试；检查是否有死循环在不停发请求
>
> **Bug 5：图片太大导致超时**
> - 原因：手机拍的照片分辨率太高，Base64 编码后字符串太长
> - 解决：拍照前降低分辨率；或使用图片压缩

### 调试技巧

> App Inventor 提供了两个非常好用的调试工具：
>
> **工具一："Do It" 功能**
> 在 Blocks 视图中，右键点击任何一个积木块，选择 **"Do It"**。它会立刻执行这一块积木并显示结果。比如你不确定一个变量的值是多少，右键点它的 `get` 积木块，选 "Do It"，就能看到当前的值。
>
> **注意："Do It" 只在手机连接状态下才能用！**
>
> **工具二：Notifier 调试消息**
> 从 User Interface 类别拖入一个 **Notifier** 组件。在关键位置加上 `call Notifier1.ShowAlert notice`，把你想检查的数据显示出来。比如在 API 调用之前显示 URL，在收到回复后显示原始返回文本。调试完成后记得删掉这些 Notifier 积木块。

## 操作指导

### 开发阶段指导

> 现在开始你们的开发时间！这里有一个功能检查清单，按顺序完成：
>
> **第一阶段（25 分钟）——核心功能**
> - [ ] UI 界面搭建完成（Designer 视图）
> - [ ] 基本交互逻辑可用（按钮点击有反应）
> - [ ] API 调用能成功发送
> - [ ] API 返回数据能正确解析和显示
>
> **第二阶段（25 分钟）——完善与优化**
> - [ ] 错误处理（空输入、网络失败、API 报错）
> - [ ] UI 美化（颜色、布局、字体）
> - [ ] 额外功能（如果时间允许）
> - [ ] 在手机上完整测试一遍
>
> 遇到问题的处理方式：
> 1. 先自己看看错误信息，对照 5 大常见 Bug 排查
> 2. 用 "Do It" 或 Notifier 看看关键变量的值
> 3. 实在解决不了，举手找我

### 中场进度检查

> （25 分钟后暂停）大家停一下，我想了解一下进度。
>
> - 已经完成 UI 搭建的请举手。
> - API 调用已经成功的请举手。
> - 遇到困难卡住的请举手。
>
> 我来挑几个常见问题统一讲一下……

### 交叉测试

> 最后 5 分钟，跟旁边的同学交换手机，互相测试对方的 App。试试以下操作：
> - 正常使用：按照预期流程操作一遍
> - 异常使用：故意输入空内容、乱按按钮，看看会不会崩溃
> - 给对方提一个改进建议

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 不知道从哪里开始 | 先把 Designer 的 UI 搭好，这是最容易的第一步 |
| 积木块太多找不到 | 用搜索功能（Blocks 视图左上角的搜索框） |
| 之前课上的积木逻辑忘了 | 翻看之前课的课件或打开之前的项目参考 |
| 两个功能冲突 | 确保事件处理不会互相覆盖；用变量标记当前状态 |
| 进度太慢做不完 | 先完成核心功能（MVP），额外功能放到"未来计划"里 |
