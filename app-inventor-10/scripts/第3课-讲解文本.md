## 导入环节

> 同学们，前两节课我们做的 App 都是"单机版"——所有数据都在手机里面。但你们想想，微信能发消息、地图能导航、天气 App 能告诉你明天下不下雨——这些都是怎么做到的？
>
> 答案就是：它们都连接了互联网，通过一个叫 **API** 的东西去获取外面的数据。今天我们就来搞明白 API 到底是什么，然后动手调用一个真实的天气 API！

## 核心概念讲解

### API 是什么？——餐厅服务员类比

> 想象你去一家餐厅吃饭。你坐在桌子前（这就是你的 App），厨房在后面做菜（这就是服务器上的数据）。你不能直接跑进厨房拿菜，对吧？你需要一个**服务员**来帮你——你告诉服务员"我要一份宫保鸡丁"，服务员跑到厨房，把你的要求告诉厨师，厨师做好之后，服务员再把菜端给你。
>
> **API（Application Programming Interface）就是那个服务员。** 你的 App 发出一个请求，API 把请求传给服务器，服务器处理好之后，API 再把结果返回给你的 App。
>
> 流程就是：**你（App）→ 服务员（API）→ 厨房（服务器）→ 菜（数据）→ 你**

### URL 的结构

> 调用 API 就像填写一个网址。这个网址叫 URL，它有两部分：
>
> - **基础地址**：`https://api.openweathermap.org/data/2.5/weather` —— 这就像餐厅的地址
> - **参数**：`?q=Beijing&appid=你的密钥&units=metric` —— 这就像你的点菜单
>
> 问号后面的部分告诉 API 你具体要什么：`q=Beijing` 表示查北京的天气，`appid` 是你的身份证明，`units=metric` 表示用摄氏度。

### JSON 格式

> API 返回的数据是一种叫 **JSON** 的格式。它长这样：
>
> ```json
> {
>   "name": "Beijing",
>   "main": { "temp": 25.5 },
>   "weather": [{ "description": "晴" }]
> }
> ```
>
> 就像一个嵌套的盒子：大盒子里有小盒子，小盒子里有更小的盒子。我们需要一层一层"拆开"来获取想要的数据。

## 操作指导

### 准备工作：注册 OpenWeatherMap

> 现在请大家打开浏览器，访问 `openweathermap.org`，注册一个免费账号。注册之后到 API Keys 页面，你会看到一个默认的 API Key，把它复制下来保存好。
>
> **注意：新注册的 Key 可能需要等几分钟才能生效。**

### 步骤一：搭建天气 App UI

> 新建项目 `WeatherApp`。在 Designer 视图添加：
>
> 1. 一个 **TextBox**——让用户输入城市名，Hint 写"请输入城市名（英文）"。
> 2. 一个 **Button**——Text 写"查询天气"。
> 3. 三个 **Label**——分别用来显示城市名、温度、天气描述。可以先把 Text 设为"城市：""温度：""天气："。
> 4. 一个 **Image** 组件——用来显示天气图标。
> 5. **关键组件**：从左侧面板的 **Connectivity** 类别中拖入一个 **Web** 组件（它是不可见组件，会出现在屏幕下方的不可见区域）。

### 步骤二：编写 API 调用逻辑（教师逐步演示）

> 切换到 Blocks 视图，跟我做：
>
> 1. 拖出 `when BtnSearch.Click do`。
> 2. 在里面设置 Web 组件的 URL：用 `set Web1.Url to` 连接一个 `join` 文本块，拼接基础地址和参数。
>    - 基础地址：`https://api.openweathermap.org/data/2.5/weather?q=`
>    - 拼接 `TextBox1.Text`（用户输入的城市名）
>    - 再拼 `&appid=你的API_KEY&units=metric&lang=zh_cn`
> 3. 然后拖一个 `call Web1.Get` 积木块——这就是在发送请求。
>
> 接下来处理返回的数据：
>
> 4. 从 Web1 拖出 `when Web1.GotText do` 积木块。
> 5. 在里面使用 `JsonTextDecodeWithDictionaries` 来解析返回的文本。
> 6. 一层层提取数据：用 `get value for key` 积木块获取 "name"（城市名）、"main" 下的 "temp"（温度）、"weather" 列表的第一项的 "description"（天气描述）。
> 7. 把提取到的数据分别显示到对应的 Label 上。

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| API Key 不起作用 | 新注册的 Key 可能需要等 10-30 分钟激活；检查是否复制完整 |
| 城市名查不到 | 必须用英文城市名（如 Beijing、Shanghai）；注意拼写 |
| JSON 解析报错 | 检查 key 名是否拼写正确（区分大小写）；确认 JSON 路径层级 |
| Web 组件找不到 | Web 组件在 Connectivity 类别中，不在 User Interface 中 |
| 显示乱码 | URL 参数中加上 `lang=zh_cn` 让 API 返回中文描述 |

## 总结与回顾

> 今天是一个重要的里程碑！你的 App 第一次连接了互联网，获取了真实的数据。我们学了三个核心概念：
>
> 1. **API** 就像餐厅服务员，帮你的 App 和服务器之间传递请求和数据
> 2. **URL** 是请求的地址，包含基础地址和参数
> 3. **JSON** 是数据的格式，像嵌套的盒子需要一层层拆开
>
> 从下节课开始，我们要调用更强大的 API —— ChatGPT！让你的 App 拥有 AI 对话能力！
