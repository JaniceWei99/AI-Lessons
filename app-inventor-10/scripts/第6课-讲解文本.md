## 导入环节

> 同学们，前几节课我们的 AI 只能"读文字"。今天我们要教 AI 一个新本领——**看图片**！
>
> 我先演示一下（教师拿起手机拍一个物品）：拍一张照片，App 立刻告诉你这是什么东西。比如拍一朵花，AI 会说"这是一朵红色的玫瑰花"；拍一本书，AI 会识别出书名。这就是计算机视觉的魅力！

## 核心概念讲解

### 计算机是怎么"看"图片的？

> 我们人类看一张照片，一眼就知道这是一只猫还是一条狗。但计算机看到的是什么呢？
>
> 计算机看到的其实是一堆**数字**。一张图片由很多像素点组成，每个像素点有红、绿、蓝三个颜色值（0-255）。一张 224x224 的图片就有 224 x 224 x 3 = **超过 15 万个数字**！
>
> AI 要做的就是从这 15 万个数字中找出规律：
> 1. **像素** → 低级特征（边缘、颜色块）
> 2. **特征** → 中级特征（纹理、形状）
> 3. **中级特征** → 高级分类（"这是一只猫！"）
>
> 这就像你先看到毛茸茸的形状，再看到尖耳朵和胡须，最后大脑判断"这是猫"。

### 多模态模型

> 以前的 AI 只能处理文字，现在有了**多模态模型**——它既能读文字，又能看图片，有的还能听声音。
>
> 比如 Google 的 **Gemini** 和 OpenAI 的 **GPT-4V** 都是多模态模型。你可以给它一张图片加一句话"这是什么？"，它就能理解图片内容并用文字回答。
>
> 今天我们就用 **Gemini Vision API** 来做图像识别。

### Base64 编码——把图片变成"文字"

> API 传输数据用的是文字格式，但图片是二进制文件。怎么办呢？用 **Base64 编码**！
>
> 简单来说，Base64 就是把二进制数据转换成一串由字母和数字组成的很长的文字。就像把一幅画用密码本翻译成一封信——信可以通过网络传输，收到后再翻译回来。

## 操作指导

### 步骤一：搭建拍照识物 UI

> 新建项目 `PhotoID`。在 Designer 视图中按这个顺序添加组件：
>
> 1. 一个 **Label** 作为标题，Text 写"拍照识物"，字号 24，加粗，居中。
> 2. 一个 **Image** 组件（命名为 ImagePreview），Height 设 300px，Width 设 Fill parent——这是用来显示拍到的照片。
> 3. 一个 **Button**（Text 写"拍照识别"）。
> 4. 一个 **Label**（命名为 LabelStatus），Text 写"请拍一张照片"——显示当前状态。
> 5. 一个 **Label**（命名为 LabelResult），Text 先留空——显示识别结果。
> 6. 从 **Media** 类别拖入 **Camera** 组件（不可见组件）。
> 7. 从 **Connectivity** 类别拖入 **Web** 组件。

### 步骤二：实现拍照功能

> 切换到 Blocks 视图：
>
> 1. 拖出 `when BtnCapture.Click do`，里面放 `call Camera1.TakePicture`。
> 2. 拖出 `when Camera1.AfterPicture do`——这个事件在拍照完成后触发，参数 `image` 就是照片的文件路径。
> 3. 在 AfterPicture 里，用 `set ImagePreview.Picture to image` 把照片显示在界面上。
> 4. 然后更新状态标签：`set LabelStatus.Text to "正在识别中..."`

### 步骤三：调用 Gemini Vision API（重点，教师演示）

> 这一步比较复杂，我先完整演示，大家仔细看：
>
> 1. 拍照后需要把图片转为 Base64 格式。使用 File 组件的 `ReadFrom` 方法读取图片文件，再进行 Base64 编码。
> 2. 构造 Gemini API 的请求体，结构是：
>    ```
>    contents → parts → [text: "这张图片里是什么？", inline_data: {mime_type: "image/jpeg", data: Base64字符串}]
>    ```
> 3. 设置 Web 组件的 URL 为 Gemini API 地址（包含 API Key）。
> 4. 用 PostText 发送请求。
> 5. 在 `when Web1.GotText do` 中解析返回的 JSON，提取识别结果显示到 LabelResult。
>
> **大家先看我操作一遍，然后再跟着做。这部分积木块比较多，不要着急，一步步来。**

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| Camera 组件不工作 | 确认手机已授予 App 相机权限；在 AI Companion 中测试 |
| 图片太大上传失败 | 在拍照设置中降低分辨率；或在发送前压缩图片 |
| Base64 编码失败 | 确认图片文件路径正确；检查 File 组件是否已添加 |
| API 返回空结果 | 检查 JSON 请求体格式；确认 API Key 有效 |
| 识别结果不准 | 确保拍照清晰、光线充足；尝试换个角度拍摄 |

## 总结与回顾

> 今天我们让 AI 学会了"看东西"！关键知识点：
>
> 1. 计算机看图片看到的是**数字**（像素值），AI 从中提取特征进行分类
> 2. **多模态模型**（如 Gemini）可以同时理解文字和图片
> 3. **Base64 编码**把图片变成文字，方便通过 API 传输
> 4. Camera 组件负责拍照，Web 组件负责发送 API 请求
>
> 现在你的 App 既能"聊天"又能"看图"了！下节课我们再加一个超能力——让 App 能"听"和"说"！
