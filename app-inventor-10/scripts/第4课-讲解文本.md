## 核心概念讲解

### ChatGPT 网站 vs ChatGPT API

> 你平时用的 ChatGPT 网站（chat.openai.com）就像去餐厅堂食——餐厅给你准备好了桌子、菜单、服务员，你直接坐下来吃就行。
>
> 而 ChatGPT API 就像**外卖接口**——你自己的 App 可以给厨房下单，厨房做好了把菜送给你的 App，你可以自己决定怎么摆盘、怎么展示。API 给你的是"原材料"，你来决定用户体验。

### API Key——你的"入场券"

> 要使用 ChatGPT API，你需要一个 **API Key**。它就像一张入场券或者身份证，证明你有权使用这个服务。
>
> **安全警告：API Key 绝对不能分享给任何人！** 如果别人拿到你的 Key，就可以用你的账号花你的钱。就像银行卡密码一样，一定要保密。在 App Inventor 中，API Key 会写在积木块里，不要截图发到网上。

### 收费模式——Token 计费

> API 是按"Token"收费的。什么是 Token？大概来说，每个中文字算 1-2 个 Token，每个英文单词算 1 个 Token。
>
> 好消息是 **GPT-4o-mini** 模型非常便宜，1 块人民币大约能对话上千次，所以大家不用担心费用。我们课上用的量完全在免费额度内。

### GET vs POST 请求

> 上节课我们用的是 **GET 请求**，就像去图书馆**借书**——你只是去"拿"信息，告诉图书馆"我要这本书"就行。
>
> 而 ChatGPT 需要用 **POST 请求**，就像**寄信**——你不仅要告诉邮局你要寄给谁（地址），还要把信的内容（你的问题）放进信封里一起寄出去。
>
> 简单记：GET = 拿数据，POST = 送数据再拿回复。

## 操作指导

### 准备工作：获取 API Key

> 这一步比较重要，我来在屏幕上演示。打开 `platform.openai.com`，登录后进入 API Keys 页面，点击"Create new secret key"，给它起个名字比如"AppInventor"，然后复制生成的 Key。
>
> **一定要马上保存！这个 Key 只显示一次，关了就看不到了。** 建议先粘贴到一个文本文件里保存好。

### 步骤一：设计聊天 UI

> 新建项目 `ChatAI`。现在来设计一个聊天界面：
>
> 1. 拖入一个 **VerticalScrollArrangement**——这是一个可以滚动的垂直布局容器，用来放聊天记录。把 Height 设为 70%，Width 设为"Fill parent"。
> 2. 在里面放一个 **Label**（命名为 LabelChat），Text 先写"聊天记录将显示在这里"。
> 3. 在滚动区域下方放一个 **HorizontalArrangement**，里面放一个 **TextBox**（Width 设 80%，Hint 写"输入你的问题..."）和一个 **Button**（Text 写"发送"）。
> 4. 从 **Connectivity** 类别拖入一个 **Web** 组件。

### 步骤二：配置 Web 组件的请求头（重点！）

> 这一步和上节课不同，因为 ChatGPT API 需要 POST 请求，而且需要设置**请求头（Headers）**。
>
> 在 Blocks 视图中，当发送按钮被点击时：
>
> 1. 先设置 URL：`set Web1.Url to "https://api.openai.com/v1/chat/completions"`
> 2. 设置请求头：用 `call Web1.SetHeader` 分别设置：
>    - `Authorization` 的值为 `Bearer 你的API_KEY`（注意 Bearer 后面有个空格）
>    - `Content-Type` 的值为 `application/json`
> 3. 构造请求体（JSON 格式），包含 model 和 messages 两个字段。
> 4. 用 `call Web1.PostText` 发送请求。
>
> **大家注意看我屏幕上的积木块结构，特别是 JSON 字符串的拼接方式。这部分比较复杂，跟着我一步一步来。**

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| API Key 无效 | 检查是否复制完整；确认 Key 前面有 "Bearer " 前缀（含空格） |
| 返回 401 错误 | API Key 不正确或已过期；重新生成一个 Key |
| 返回 429 错误 | 请求太频繁，等一会儿再试；免费额度用完需充值 |
| JSON 拼接出错 | 注意引号的配对；字符串中的引号要正确嵌套 |
| 没收到回复 | 检查 `when Web1.GotText do` 积木块是否已添加 |

## 总结与回顾

> 今天我们迈出了连接 AI 的第一步！关键知识点：
>
> 1. **API Key** 是你的身份凭证，一定要保密
> 2. **GET** 拿数据，**POST** 送数据 —— ChatGPT 用 POST
> 3. POST 请求需要设置**请求头**（Authorization 和 Content-Type）
> 4. 请求体是 JSON 格式，包含模型名和消息内容
>
> 今天我们完成了 UI 搭建和第一次 API 调用。下节课我们会让聊天更智能——添加对话历史、角色设定、聊天气泡效果！
