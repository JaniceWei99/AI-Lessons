# 第11课：部署上线 — 发布到互联网

## 教学目标

- 理解"部署"的概念（本地文件 → 互联网可访问）
- 了解域名和 DNS 的基本原理
- 掌握 Git 基础操作：`init`、`add`、`commit`、`remote`、`push`
- 完成 GitHub 账号注册和仓库创建
- 学会使用 GitHub Pages 部署静态网站
- 了解 Vercel 作为替代部署方案
- 成功将自己的 AI 网站发布到互联网

## 课时建议

**总时长：90 分钟**

## 教学流程

### 一、导入环节（5 分钟）

> **教师讲稿：**
>
> 同学们，恭喜你们！经过 10 节课的学习，你们已经做出了很棒的 AI 网站——有聊天功能，有图像生成。但目前这些网页只存在于你们自己的电脑上。如果你想给朋友看，你得说"来我电脑上看"——这也太不方便了吧？
>
> 今天，我们要做一件非常激动人心的事——**把你的网站发布到互联网上**！发布之后，任何人只要输入一个网址，就能在手机、电脑上看到你的作品。这就是"部署"。

**教学提示：** 提前部署好一个示例网站，打开它的 URL，在手机和电脑上同时展示——让学生直观感受"部署"的意义。

### 二、核心概念讲解（25 分钟）

#### 2.1 什么是部署（3 分钟）

> 部署就是把你的代码从"本地"（你的电脑）放到"服务器"（互联网上一直开着的电脑）。当别人访问你的网址时，服务器就把你的网页文件发送给他们的浏览器。
>
> 简单说：**本地开发 → 上传到服务器 → 全世界可访问**

#### 2.2 域名与 DNS（3 分钟）

> 每个网站都有一个 IP 地址（如 `185.199.108.153`），但记数字太难了。域名（如 `github.com`）是给 IP 地址取的"别名"。DNS 就是"电话簿"——把域名翻译成 IP 地址。
>
> 今天我们用 GitHub Pages 免费部署，自动获得一个 `你的用户名.github.io` 的域名——不需要花钱买域名！

#### 2.3 Git 基础（12 分钟）

> **Git 是程序员的"时光机"** —— 它记录你代码的每一次变化，随时可以回退到之前的版本。而 **GitHub** 是存放 Git 仓库的"云端服务器"。
>
> 核心命令只有这几个：

```bash
# 1. 初始化 —— 在项目文件夹中创建 Git 仓库
git init

# 2. 暂存 —— 把改动的文件放入"待提交"区域
git add .                    # . 表示所有文件

# 3. 提交 —— 保存一个"快照"，附带说明信息
git commit -m "第一次提交：完成AI聊天网站"

# 4. 连接远程仓库 —— 告诉 Git 代码要推送到哪里
git remote add origin https://github.com/你的用户名/my-ai-website.git

# 5. 推送 —— 把代码上传到 GitHub
git push -u origin main
```

> 用比喻来理解：
> - `git add` = 把要寄的东西打包放进箱子
> - `git commit` = 在箱子上贴标签、封箱
> - `git push` = 把箱子寄出去

**教学提示：** 在终端中逐步演示每个命令，每执行一步都用 `git status` 查看当前状态。让学生看到文件从"红色（未暂存）"变成"绿色（已暂存）"再到"提交成功"的过程。

#### 2.4 GitHub Pages 部署（5 分钟）

> GitHub Pages 是 GitHub 提供的免费静态网站托管服务。部署步骤：
>
> 1. 在 GitHub 上创建一个新仓库
> 2. 把代码推送到仓库
> 3. 进入仓库 → **Settings** → **Pages**
> 4. Source 选择 **main** 分支，点 Save
> 5. 等待 1-2 分钟，你的网站就上线了！
>
> 网址格式：`https://你的用户名.github.io/仓库名/`
>
> 每次你 `git push` 更新代码，网站会自动更新——这叫"持续部署"。

#### 2.5 Vercel 替代方案（2 分钟）

> 除了 GitHub Pages，还有一个更现代的选择——**Vercel**：
> - 自动从 GitHub 拉取代码部署
> - 支持自定义域名
> - 内置 Serverless Function（可以安全地放 API Key！）
> - 部署速度更快
>
> 如果你以后要部署需要后端的项目，Vercel 是更好的选择。

### 三、编码指导 — 动手实践（45 分钟）

#### 3.1 教师带码 — 完整部署流程（20 分钟）

**教学提示：** 这节课的实操比较多，涉及终端操作和网页操作的切换。建议投影同时显示终端和浏览器。

**步骤一：检查项目结构（2 分钟）**
```bash
# 确保项目文件夹结构正确
my-ai-website/
├── index.html        # 首页（必须是这个名字！）
├── style.css
├── script.js
└── images/           # 图片文件夹（如果有的话）
```

> 重点：首页文件必须叫 `index.html`，GitHub Pages 会自动识别它。

**步骤二：Git 初始化与提交（5 分钟）**
```bash
cd my-ai-website
git init
git add .
git commit -m "初始提交：我的AI网站"
```

**步骤三：创建 GitHub 仓库（5 分钟）**
> 打开 github.com → 点右上角 "+" → New repository → 输入仓库名 → 选 Public → 不要勾选 README → Create

```bash
git remote add origin https://github.com/你的用户名/my-ai-website.git
git branch -M main
git push -u origin main
```

**步骤四：开启 GitHub Pages（3 分钟）**
> 仓库页面 → Settings → 左侧 Pages → Source 选 main → Save → 等待部署

**步骤五：验证（5 分钟）**
> 打开 `https://你的用户名.github.io/my-ai-website/`，用手机扫码访问。

#### 3.2 学生自主操作（25 分钟）

> 请按照以下步骤部署你的网站：
> 1. 检查项目结构，确保有 `index.html`
> 2. Git 初始化、添加、提交
> 3. 在 GitHub 创建仓库
> 4. 推送代码到 GitHub
> 5. 开启 GitHub Pages
> 6. 用手机访问你的网站！

**教学提示：** 这一步最容易出问题——Git 命令行操作对初学者来说比较陌生。建议巡视教室重点帮助。可以在黑板上写出完整的命令序列供学生参考。

### 四、常见问题与排错（5 分钟）

| 常见错误 | 原因 | 解决方法 |
|---------|------|---------|
| `git push` 需要登录 | 首次推送需要认证 | 使用 GitHub 个人访问令牌（Token）或 SSH |
| 网站打开是 404 | 首页不叫 `index.html` 或在子文件夹内 | 确保 `index.html` 在仓库根目录 |
| Pages 设置里没有 main 分支 | 没有 `git push` 成功 | 确认推送成功后刷新设置页 |
| CSS/JS 没加载 | 文件路径是绝对路径 | 改为相对路径（`./style.css` 而不是 `/style.css`） |
| 网站更新后没变化 | GitHub Pages 有缓存 | 等几分钟，或强制刷新（Ctrl+Shift+R） |

### 五、总结与回顾（5 分钟）

> 今天的成就感应该是整门课最强的——你的网站现在全世界都能访问了！
> 1. **Git** = 代码版本管理，`add` → `commit` → `push` 三步走
> 2. **GitHub Pages** = 免费静态网站托管
> 3. 首页必须叫 `index.html`
> 4. 每次更新只需 `git add . && git commit -m "更新" && git push`
>
> 请把你的网址分享给家人和朋友——你做出了一个真正的 AI 网站！

### 六、课后延伸（5 分钟）

- **基础作业：** 确保网站成功部署，用手机和电脑分别访问验证
- **进阶挑战：** 尝试在 Vercel 上也部署一份，对比两个平台
- **预习提示：** 下节课是 Demo Day！准备一个 4-5 分钟的展示——介绍你的网站、现场演示、讲讲你用到的技术
