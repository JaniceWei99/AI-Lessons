# 第7课：API 调用基础 — fetch() 的魔力

## 教学目标

- 理解 API 的概念及其在网络中的角色
- 掌握 JSON 数据格式（对象、数组、规则、解析与序列化）
- 学会使用 `fetch()` 发起网络请求
- 理解 Promise 和 `async/await` 的基本用法
- 了解常见 HTTP 状态码（200/404/500/401/429）
- 掌握 `try/catch` 错误处理
- 完成一个随机狗狗图片生成器

## 课时建议

**总时长：90 分钟**

## 教学流程

### 一、导入环节（8 分钟）

> **教师讲稿：**
>
> 同学们，到目前为止我们做的所有网页都是"自给自足"的——页面上的内容都是我们手动写在 HTML 中的。但你们有没有想过，微博是怎么显示最新消息的？天气预报网站是怎么获取实时天气的？答案就是——**API**！
>
> API 的全称是 Application Programming Interface（应用程序编程接口）。听起来很复杂，我用一个比喻来解释：想象你在餐厅吃饭。你（客户端/前端）想吃牛排，但你不能跑进厨房自己做。你需要通过**服务员（API）**把你的需求告诉**厨房（服务器）**，厨房做好后通过服务员把牛排端上来。
>
> API 就是这个"服务员"——它定义了"你可以点什么菜"以及"怎么点"。今天我们就来学习怎么"点菜"！

**教学提示：** 在浏览器地址栏直接打开 `https://dog.ceo/api/breeds/image/random`，让学生看到返回的 JSON 数据——这就是 API 的响应。

### 二、核心概念讲解（30 分钟）

#### 2.1 什么是 REST API（5 分钟）

> 互联网上大多数 API 遵循 REST 风格——简单来说就是通过 URL 来获取数据。就像你输入不同的网址会打开不同的网页，调用不同的 API 地址会获取不同的数据。
>
> 比如：
> - `https://dog.ceo/api/breeds/image/random` → 获取随机狗狗图片
> - `https://api.openweathermap.org/data/2.5/weather?q=Beijing` → 获取北京天气
>
> API 通常返回的数据格式不是 HTML，而是 **JSON**。

#### 2.2 JSON 格式（8 分钟）

```javascript
// JSON 对象（类似 JavaScript 的对象）
{
  "name": "小明",
  "age": 13,
  "isStudent": true,
  "hobbies": ["编程", "篮球", "阅读"],
  "address": {
    "city": "北京",
    "district": "海淀区"
  }
}
```

> JSON 有几个严格的规则：
> 1. 键名必须用**双引号**包裹（不能用单引号）
> 2. 字符串必须用**双引号**
> 3. 不能有注释
> 4. 不能有多余的逗号
>
> JavaScript 中处理 JSON 的两个方法：
> ```javascript
> // 把 JSON 字符串 → JavaScript 对象
> const obj = JSON.parse('{"name":"小明"}');
>
> // 把 JavaScript 对象 → JSON 字符串
> const str = JSON.stringify({ name: "小明" });
> ```

**教学提示：** 在控制台让学生实操 `JSON.parse` 和 `JSON.stringify`，建立对数据转换的直觉。

#### 2.3 fetch() 与 async/await（10 分钟）

> `fetch()` 是浏览器内置的"API 调用工具"。但注意，网络请求需要时间——不像 `1+1` 立刻就有结果。所以我们用 `async/await` 来"等待"结果：

```javascript
// async/await 方式（推荐！清晰易读）
async function getDogImage() {
  const response = await fetch("https://dog.ceo/api/breeds/image/random");
  const data = await response.json(); // 把响应转成 JSON 对象
  console.log(data);
  console.log(data.message); // 图片 URL
}

getDogImage();
```

> 逐行解读：
> 1. `async` 标记这个函数里面有"需要等待的操作"
> 2. `await fetch(url)` —— 发出请求，**等待**服务器响应
> 3. `await response.json()` —— 把响应数据**解析**成 JSON
> 4. 解析后就可以像普通对象一样用 `.` 来访问数据了

**现场演示：** 在控制台逐步执行，让学生看到 `response` 对象和 `data` 对象的内容。

#### 2.4 HTTP 状态码与错误处理（7 分钟）

> 服务器返回的不只是数据，还有一个"状态码"告诉你请求是否成功：
>
> | 状态码 | 含义 | 说明 |
> |-------|------|------|
> | 200 | OK | 成功 |
> | 404 | Not Found | 地址不存在 |
> | 500 | Server Error | 服务器出错 |
> | 401 | Unauthorized | 没有权限（缺少 API Key） |
> | 429 | Too Many Requests | 请求太频繁 |
>
> 如果网络出了问题或者 API 挂了，程序会崩溃。所以我们要用 `try/catch` 来"兜底"：

```javascript
async function getDogImage() {
  try {
    const response = await fetch("https://dog.ceo/api/breeds/image/random");

    if (!response.ok) {
      throw new Error(`请求失败：${response.status}`);
    }

    const data = await response.json();
    return data.message;
  } catch (error) {
    console.error("出错了：", error.message);
    alert("获取图片失败，请检查网络连接");
  }
}
```

> `try` 里放"可能出错"的代码，`catch` 里放"出错时该怎么办"。就像系安全带——平时不需要，关键时刻能救命。

### 三、编码指导 — 动手实践（35 分钟）

#### 3.1 教师带码 — 随机狗狗图片生成器（15 分钟）

**HTML：**
```html
<div class="dog-app">
  <h1>随机狗狗图片</h1>
  <button id="fetch-btn">换一张</button>
  <div id="loading" style="display:none;">加载中...</div>
  <div id="result">
    <img id="dog-image" src="" alt="随机狗狗" style="max-width:400px;">
  </div>
</div>
```

**JS（逐步构建）：**
```javascript
const fetchBtn = document.querySelector("#fetch-btn");
const dogImage = document.querySelector("#dog-image");
const loading = document.querySelector("#loading");

async function fetchDog() {
  try {
    // 显示加载状态
    loading.style.display = "block";
    fetchBtn.disabled = true;

    const response = await fetch("https://dog.ceo/api/breeds/image/random");
    if (!response.ok) throw new Error(`HTTP ${response.status}`);

    const data = await response.json();
    dogImage.src = data.message;

  } catch (error) {
    alert("获取失败：" + error.message);
  } finally {
    loading.style.display = "none";
    fetchBtn.disabled = false;
  }
}

fetchBtn.addEventListener("click", fetchDog);

// 页面加载时自动获取一张
fetchDog();
```

**教学提示：** 先不加 loading 状态，只做最基本的获取和显示。确认功能正常后，再加上 loading 和错误处理。在 DevTools 的 Network 面板中展示请求和响应的过程——让学生看到"数据是从网上拿来的"。

### 四、常见问题与排错（7 分钟）

| 常见错误 | 原因 | 解决方法 |
|---------|------|---------|
| `fetch is not defined` | 在非浏览器环境运行 | 确保代码在浏览器中执行 |
| `await` 报语法错误 | 没有在 `async` 函数内使用 `await` | 确保函数前面有 `async` 关键字 |
| 数据不是预期结构 | 没有认真查看 API 返回格式 | 先 `console.log(data)` 查看完整结构 |
| CORS 错误 | API 不允许浏览器直接调用 | 换一个支持 CORS 的 API，或后面学后端代理 |
| 图片不显示 | `src` 赋值了错误的数据字段 | 检查 JSON 中图片 URL 对应的 key |

**教学提示：** 重点教学生使用 DevTools 的 Network 面板——点击请求可以查看 Headers（请求头）、Response（响应内容）和 Status Code（状态码）。这是前端开发者最常用的调试工具之一。

### 五、总结与回顾（5 分钟）

> 今天的核心知识点：
> 1. **API** = 应用之间的"服务员"，负责传递请求和响应
> 2. **JSON** = API 数据的通用格式，键值对结构
> 3. **fetch + async/await** = 发起网络请求并等待结果
> 4. **try/catch** = 处理可能出现的错误
> 5. 状态码 200 表示成功，4xx 表示客户端错误，5xx 表示服务器错误
>
> 你们的网页终于不再是"孤岛"了——它可以和互联网上的任何 API 对话！下节课，我们要连接的可是世界上最先进的 AI！
