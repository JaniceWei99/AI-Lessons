# 第8课：接入 AI（上）— 搭建 AI 聊天界面

## 教学目标

- 了解主流 AI API 服务（OpenAI GPT-4o/4o-mini、Anthropic Claude）
- 理解 API Key 安全的重要性（绝不暴露在前端代码中）
- 了解 Token 计费和模型选择
- 搭建聊天 UI 界面（消息区域 + 输入框 + 发送按钮）
- 理解 OpenAI Chat Completions API 的请求格式
- 掌握消息角色概念：`system`、`user`、`assistant`
- 理解 `temperature` 参数的作用
- 完成基础的 AI 聊天流程

## 课时建议

**总时长：90 分钟**

### 二、核心概念讲解（30 分钟）

#### 2.1 AI API 服务概览（5 分钟）

> 目前最主流的 AI API 有：
> - **OpenAI**：GPT-4o（最强）、GPT-4o-mini（便宜又好用，推荐课程使用）
> - **Anthropic**：Claude 3.5 Sonnet / Claude 3 Haiku
>
> 它们都是通过 API 调用的——就像上节课我们调用狗狗图片 API 一样，只不过这次我们发送的是文字，收到的也是文字。

#### 2.2 API Key 安全（8 分钟）

> **这是今天最重要的知识点，没有之一！**
>
> API Key 就像你家的钥匙——有了它，任何人都能用你的账号调用 AI，花你的钱。所以有一条铁律：
>
> **永远、永远、永远不要把 API Key 放在前端代码中！**
>
> 为什么？因为浏览器中的所有代码用户都能看到——按 F12 打开 DevTools，你的 API Key 就暴露了。一旦泄露，别人可以用你的 Key 无限调用 AI，你可能会收到巨额账单。
>
> 正确的做法是：把 API Key 放在**后端服务器**或**无服务器函数（Serverless Function）**中。在这门课中，我们会使用一个课程提供的安全代理，或者在本地环境中演示。

**教学提示：** 强调这一点不要怕重复。可以举一个真实案例——有开发者不小心把 API Key 上传到 GitHub，几分钟内就被机器人扫描到并盗用。

#### 2.3 Token 与计费（3 分钟）

> AI API 按 **Token**（令牌）计费。Token 不等于字符数——大约每 4 个英文字符 = 1 个 Token，中文大约每 1-2 个字 = 1 个 Token。
>
> GPT-4o-mini 非常便宜：100 万输入 Token 只要约 0.15 美元。一次普通对话大约几百个 Token，所以成本很低。我们课程中完全不用担心费用。

#### 2.5 OpenAI Chat Completions API（10 分钟）

> 调用 OpenAI API 的核心是向 `https://api.openai.com/v1/chat/completions` 发送一个 POST 请求：

```javascript
async function sendMessage(userMessage) {
  const response = await fetch("https://api.openai.com/v1/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": "Bearer YOUR_API_KEY" // 实际项目中不要这样写！
    },
    body: JSON.stringify({
      model: "gpt-4o-mini",
      messages: [
        {
          role: "system",
          content: "你是一个友好的AI助手，用简洁的中文回答问题。"
        },
        {
          role: "user",
          content: userMessage
        }
      ],
      temperature: 0.7
    })
  });

  const data = await response.json();
  return data.choices[0].message.content;
}
```

> 关键概念：
> - **method: "POST"** —— 发送数据用 POST，获取数据用 GET
> - **messages 数组** —— 包含对话历史
>   - `system`：设定 AI 的角色和行为规则（"人设"）
>   - `user`：用户说的话
>   - `assistant`：AI 之前回复的内容
> - **temperature**：0-2 之间的值。0 = 最严谨确定，1 = 有创意，2 = 非常随机。推荐 0.7 作为通用值。

**现场演示：** 使用 Postman 或直接在控制台用 `fetch` 调用一次 API，让学生看到完整的请求和响应过程。

### 三、编码指导 — 动手实践（35 分钟）

#### 3.1 教师带码 — 基础聊天流程（18 分钟）

**教学提示：** 逐步构建，先做发送消息→显示在界面上，再做 API 调用→显示 AI 回复。

```javascript
const chatForm = document.querySelector("#chat-form");
const userInput = document.querySelector("#user-input");
const messagesDiv = document.querySelector("#messages");

// 添加消息到界面
function addMessage(content, role) {
  const msgDiv = document.createElement("div");
  msgDiv.classList.add("message", role);
  msgDiv.textContent = content;
  messagesDiv.appendChild(msgDiv);
  messagesDiv.scrollTop = messagesDiv.scrollHeight; // 自动滚动到底部
}

// 调用 AI API
async function getAIResponse(userMessage) {
  try {
    const response = await fetch("你的API代理地址", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        model: "gpt-4o-mini",
        messages: [
          { role: "system", content: "你是一个友好的AI助手。" },
          { role: "user", content: userMessage }
        ],
        temperature: 0.7
      })
    });

    if (!response.ok) throw new Error(`API 错误：${response.status}`);

    const data = await response.json();
    return data.choices[0].message.content;
  } catch (error) {
    console.error(error);
    return "抱歉，出现了错误，请稍后再试。";
  }
}

// 处理表单提交
chatForm.addEventListener("submit", async (e) => {
  e.preventDefault();

  const message = userInput.value.trim();
  if (!message) return;

  // 显示用户消息
  addMessage(message, "user");
  userInput.value = "";

  // 显示加载状态
  addMessage("思考中...", "assistant loading");

  // 调用 AI
  const aiReply = await getAIResponse(message);

  // 替换加载消息为实际回复
  const loadingMsg = messagesDiv.querySelector(".loading");
  if (loadingMsg) {
    loadingMsg.textContent = aiReply;
    loadingMsg.classList.remove("loading");
  }
});
```

**配套 CSS 要点：**
```css
.message.user { background: #007bff; color: white; align-self: flex-end; }
.message.assistant { background: #e9ecef; align-self: flex-start; }
.message.loading { opacity: 0.6; font-style: italic; }
```

### 四、常见问题与排错（7 分钟）

| 常见错误 | 原因 | 解决方法 |
|---------|------|---------|
| 401 Unauthorized | API Key 无效或未设置 | 检查 Key 是否正确，是否有余额 |
| 429 Too Many Requests | 调用频率太高 | 等一会再试，或升级 API 计划 |
| AI 回复为空 | 解析响应数据路径错误 | `console.log(data)` 检查完整响应结构 |
| CORS 错误 | 直接在前端调用 OpenAI | 使用后端代理或课程提供的代理 |
| 消息不滚动到底部 | 没有设置 `scrollTop` | 每次添加消息后设置 `scrollTop = scrollHeight` |

**教学提示：** 在 Network 面板查看 POST 请求的 Payload（发送的数据）和 Response（返回的数据），确保学生理解请求-响应的完整流程。

### 五、总结与回顾（5 分钟）

> 今天的核心知识点：
> 1. **API Key 绝不放前端** —— 安全第一！
> 2. AI API 用 **POST** 请求，发送 **JSON** 数据
> 3. 消息有三种角色：`system`（设定人设）、`user`（用户输入）、`assistant`（AI 回复）
> 4. `temperature` 控制回复的创意程度
> 5. 别忘了错误处理和加载状态
>
> 我们已经让网页能和 AI 对话了！但目前只有单轮对话——AI 不记得之前说过什么。下节课我们来解决这个问题！
