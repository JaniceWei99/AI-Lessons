# 第10课：AI 图像生成 — 用 API 在网站上生成图片

## 教学目标

- 了解主流文本生成图像模型（DALL-E、Stable Diffusion、Midjourney）
- 掌握图像 Prompt 编写公式（主体 + 风格 + 细节 + 氛围）
- 学会调用 OpenAI Image API（请求参数：prompt、n、size）
- 搭建图像生成器 UI（输入框 + 按钮 + 加载状态 + 结果展示）
- 完成一个 AI 图像生成应用

## 课时建议

**总时长：90 分钟**

## 教学流程

### 一、导入环节（8 分钟）

> **教师讲稿：**
>
> 同学们，过去两节课我们让 AI 能"说话"了。今天我们要让 AI 学会"画画"！
>
> 你们可能听说过 Midjourney、DALL-E 这些工具——你用文字描述一个场景，AI 就能帮你画出来。"一只戴墨镜的猫在沙滩上冲浪"，啪，几秒钟出一张图。今天我们要在自己的网页上实现这个功能！
>
> 而且，写好的"提示词"（Prompt）和写得不好的效果天差地别。今天我们不光学技术，还要学"Prompt 的艺术"。

**教学提示：** 准备 3-4 张用 AI 生成的图片，展示不同 Prompt 产出的效果差异（如简单 vs 详细 Prompt）。

### 二、核心概念讲解（25 分钟）

#### 2.1 文本生成图像技术概览（5 分钟）

> 目前主流的文本生成图像模型：
> - **DALL-E 3**（OpenAI）—— API 调用方便，和我们之前用的 ChatGPT API 是同一家
> - **Stable Diffusion** —— 开源，可以本地运行
> - **Midjourney** —— 效果最好，但只能在 Discord 使用
>
> 今天我们用 DALL-E，因为它的 API 和我们上节课学的 OpenAI API 很类似，上手最快。

#### 2.2 图像 Prompt 编写公式（8 分钟）

> 好的 Prompt 是好图片的关键。记住这个公式：
>
> **主体 + 风格 + 细节 + 氛围**
>
> | 要素 | 示例 |
> |------|------|
> | 主体 | 一只猫坐在窗边 |
> | 风格 | 水彩画风格 / 赛博朋克 / 吉卜力动画 |
> | 细节 | 阳光透过窗帘，旁边有一杯咖啡 |
> | 氛围 | 温暖的、宁静的午后 |
>
> 组合起来：**"一只橘色的猫坐在窗边，水彩画风格，阳光透过白色薄纱窗帘，旁边有一杯热咖啡冒着蒸汽，温暖宁静的午后氛围"**
>
> 对比：
> - 差的 Prompt："画一只猫" → 结果很随机
> - 好的 Prompt："一只戴着小礼帽的英国短毛猫，坐在维多利亚风格的扶手椅上，油画风格，暖色调光线" → 结果精确且富有艺术感

**教学提示：** 让学生先在纸上写 2-3 个 Prompt，互相分享讨论。好的 Prompt 是"沟通能力"的体现——你要把脑海中的画面准确地"翻译"成文字。

#### 2.3 OpenAI Image API（7 分钟）

```javascript
async function generateImage(prompt) {
  const response = await fetch("https://api.openai.com/v1/images/generations", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": "Bearer YOUR_API_KEY"  // 通过后端代理！
    },
    body: JSON.stringify({
      model: "dall-e-3",
      prompt: prompt,
      n: 1,                  // 生成几张图
      size: "1024x1024"      // 尺寸：256x256 / 512x512 / 1024x1024
    })
  });

  const data = await response.json();
  return data.data[0].url;   // 返回图片 URL
}
```

> 参数说明：
> - `model`：`dall-e-3`（最新最好）或 `dall-e-2`（便宜些）
> - `prompt`：图片描述
> - `n`：生成数量（DALL-E 3 只支持 1 张）
> - `size`：图片尺寸，越大越贵
>   - `256x256` —— 快速预览
>   - `512x512` —— 中等质量
>   - `1024x1024` —— 高质量（推荐）

#### 2.4 构建 UI 界面（5 分钟）

```html
<div class="image-generator">
  <h1>AI 图像生成器</h1>

  <div class="input-area">
    <textarea id="prompt-input" rows="3"
      placeholder="描述你想要的图片...例如：一只戴墨镜的柯基在太空漫步，像素艺术风格"></textarea>

    <div class="controls">
      <select id="size-select">
        <option value="1024x1024">1024 x 1024（高清）</option>
        <option value="512x512">512 x 512（标准）</option>
        <option value="256x256">256 x 256（快速）</option>
      </select>
      <button id="generate-btn">生成图片</button>
    </div>
  </div>

  <div id="loading" class="loading" style="display:none;">
    <p>AI 正在绘画中，请稍候（约 10-20 秒）...</p>
  </div>

  <div id="result" class="result"></div>
</div>
```

> 注意：图像生成比文字生成慢很多——通常需要 10-20 秒。所以加载状态提示非常重要，不然用户以为页面卡了。

### 三、编码指导 — 动手实践（40 分钟）

#### 3.1 教师带码 — AI 图像生成器（18 分钟）

**教学提示：** 分步构建，先做最简版（输入 + 按钮 + 显示图片），再添加尺寸选择和加载状态。

```javascript
const promptInput = document.querySelector("#prompt-input");
const sizeSelect = document.querySelector("#size-select");
const generateBtn = document.querySelector("#generate-btn");
const loading = document.querySelector("#loading");
const result = document.querySelector("#result");

async function generateImage() {
  const prompt = promptInput.value.trim();
  if (!prompt) {
    alert("请输入图片描述！");
    return;
  }

  try {
    // 显示加载状态
    loading.style.display = "block";
    generateBtn.disabled = true;
    generateBtn.textContent = "生成中...";
    result.innerHTML = "";

    const response = await fetch("你的API代理地址/v1/images/generations", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        model: "dall-e-3",
        prompt: prompt,
        n: 1,
        size: sizeSelect.value
      })
    });

    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(errorData.error?.message || `HTTP ${response.status}`);
    }

    const data = await response.json();
    const imageUrl = data.data[0].url;

    // 显示生成的图片
    const img = document.createElement("img");
    img.src = imageUrl;
    img.alt = prompt;
    img.style.maxWidth = "100%";

    const caption = document.createElement("p");
    caption.textContent = `Prompt: ${prompt}`;
    caption.classList.add("caption");

    result.appendChild(img);
    result.appendChild(caption);

  } catch (error) {
    result.innerHTML = `<p class="error">生成失败：${error.message}</p>`;
  } finally {
    loading.style.display = "none";
    generateBtn.disabled = false;
    generateBtn.textContent = "生成图片";
  }
}

generateBtn.addEventListener("click", generateImage);
```

**教学提示：** 第一次生成时让全班一起看效果——等待的十几秒可以让学生讨论他们的 Prompt。成功生成图片时学生通常会很兴奋，利用这个时刻！

#### 3.2 学生自主编码（22 分钟）

> 请完成你的 AI 图像生成器：
> 1. 搭建输入界面和结果展示区域
> 2. 实现图片生成功能（含加载状态和错误处理）

> - **挑战 1：** 添加尺寸选择下拉菜单

> - **挑战 3：** 添加"Prompt 建议"按钮，点击自动填入一个随机 Prompt 模板

### 四、常见问题与排错（7 分钟）

| 常见错误 | 原因 | 解决方法 |
|---------|------|---------|
| 生成超时或很慢 | 图像生成本身就需要时间 | 耐心等待，确保有清晰的加载提示 |
| 图片违规被拒绝 | Prompt 包含敏感内容 | 修改 Prompt，避免暴力/成人内容 |
| 返回的 URL 过期了 | DALL-E 返回的 URL 有有效期 | 及时下载保存，或用 localStorage 缓存 |
| 费用超出预期 | 1024x1024 的图片比较贵 | 测试时用 256x256，满意后再用高清 |
| `data.data[0]` undefined | API 返回了错误响应 | 先 `console.log(data)` 检查完整响应 |

**教学提示：** 在 Network 面板中查看 Image API 的请求——注意响应时间比文字 API 长很多。展示返回的 JSON 结构，让学生理解 `data.data[0].url` 的路径。

### 五、总结与回顾（5 分钟）

> 今天的核心知识点：
> 1. AI 图像生成的核心是**好的 Prompt**——主体 + 风格 + 细节 + 氛围
> 2. OpenAI Image API 和 Chat API 调用方式类似，都是 POST + JSON
> 3. 图像生成比文字生成慢得多，**加载状态**必不可少
> 4. 图片有尺寸和费用的权衡——测试用小图，成品用大图
>
> 现在你们的网页既能和 AI 聊天，又能让 AI 画画。下节课，我们要把这些作品发布到真正的互联网上！
