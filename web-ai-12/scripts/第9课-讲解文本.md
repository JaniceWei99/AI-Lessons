### 二、核心概念讲解（28 分钟）

#### 2.1 多轮对话（10 分钟）

> AI 本身是"无状态"的——每次请求对它来说都是全新的。要实现多轮对话，我们需要在每次请求中把**所有历史消息都发给 AI**：

```javascript
// 维护对话历史
const conversationHistory = [
  { role: "system", content: "你是一个友好的AI助手。" }
];

async function sendMessage(userMessage) {
  // 添加用户消息到历史
  conversationHistory.push({ role: "user", content: userMessage });

  const response = await fetch(API_URL, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "gpt-4o-mini",
      messages: conversationHistory,  // 发送完整历史
      temperature: 0.7
    })
  });

  const data = await response.json();
  const aiMessage = data.choices[0].message.content;

  // 添加 AI 回复到历史
  conversationHistory.push({ role: "assistant", content: aiMessage });

  return aiMessage;
}
```

> **但有一个限制——上下文窗口。** GPT-4o 最大支持 128K Token，Claude 支持 200K。听起来很多，但长时间对话仍然可能超限。所以我们需要一个"修剪"函数：

```javascript
function trimHistory(messages, maxMessages = 20) {
  if (messages.length <= maxMessages) return messages;
  // 保留 system 消息 + 最近的消息
  return [messages[0], ...messages.slice(-maxMessages + 1)];
}
```

> 每次发送前，用 `trimHistory` 确保消息数量不超限。`system` 消息永远保留——这是 AI 的"人设"。

#### 2.2 打字效果（8 分钟）

> **方式一：前端模拟（简单）** —— 收到完整回复后，逐字显示：

```javascript
async function typeEffect(element, text, speed = 30) {
  element.textContent = "";
  for (let i = 0; i < text.length; i++) {
    element.textContent += text[i];
    await new Promise(resolve => setTimeout(resolve, speed));
  }
}

// 使用
const msgDiv = document.createElement("div");
messagesDiv.appendChild(msgDiv);
await typeEffect(msgDiv, aiReply);
```

> **方式二：Streaming 流式输出（高级）** —— AI 一边生成一边发送，用户无需等待完整回复：

```javascript
const response = await fetch(API_URL, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "gpt-4o-mini",
    messages: conversationHistory,
    stream: true  // 开启流式输出
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value);
  // 解析并显示每个片段...
}
```

> 方式一适合入门，效果不错；方式二是 ChatGPT 实际使用的方式，体验更好但代码更复杂。今天我们先用方式一，有兴趣的同学可以课后尝试方式二。

#### 2.3 Markdown 渲染（5 分钟）

> AI 的回复经常包含 Markdown 格式——比如 `**粗体**`、`` `代码` ``、列表等。我们需要把它们转成 HTML 显示：

```javascript
// 简单版：用正则替换
function simpleMarkdown(text) {
  return text
    .replace(/\*\*(.*?)\*\*/g, '<strong>$1</strong>')
    .replace(/\*(.*?)\*/g, '<em>$1</em>')
    .replace(/`(.*?)`/g, '<code>$1</code>')
    .replace(/\n/g, '<br>');
}

// 生产版：使用 marked.js 库（推荐）
// <script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
// element.innerHTML = marked.parse(aiReply);
```

> 简单版能应付大部分情况，但对复杂 Markdown（如代码块、表格）支持不够。实际项目中推荐使用 `marked.js` 库。

**教学提示：** 对比展示——同一段包含 Markdown 的 AI 回复，用 `textContent`（原始文本）和 `innerHTML`（渲染后）的效果差异。

#### 2.4 UX 优化要点（5 分钟）

```javascript
// 1. 自动滚动到最新消息
messagesDiv.scrollTop = messagesDiv.scrollHeight;

// 2. 清空聊天
function clearChat() {
  messagesDiv.innerHTML = "";
  conversationHistory.length = 1; // 只保留 system 消息
}

// 3. 发送时禁用输入
function setLoading(isLoading) {
  userInput.disabled = isLoading;
  sendBtn.disabled = isLoading;
  sendBtn.textContent = isLoading ? "思考中..." : "发送";
}

// 4. 错误处理与提示
function showError(message) {
  addMessage(`⚠ ${message}`, "error");
}

// 5. 按 Enter 发送，Shift+Enter 换行
userInput.addEventListener("keydown", (e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    chatForm.dispatchEvent(new Event("submit"));
  }
});
```

> 好的产品和普通产品的差距，往往就在这些"小细节"上。加载状态、错误提示、快捷键——每一个都让用户感觉更舒服。

### 四、常见问题与排错（7 分钟）

| 常见错误 | 原因 | 解决方法 |
|---------|------|---------|
| AI 还是没有记忆 | `conversationHistory` 没有正确追加消息 | 确保 push 了 user 和 assistant 消息 |
| 打字效果卡顿 | speed 值太小或消息太长 | 调大 speed 值（如 20-50ms） |
| Markdown 渲染后样式混乱 | `innerHTML` 中有未转义的 HTML | 使用成熟的 marked.js 库处理 |
| 长对话后 API 报错 | Token 超出模型限制 | 使用 `trimHistory` 修剪历史消息 |
| 连续快速发送导致混乱 | 没有在等待回复时禁用输入 | 发送后立即 `disabled = true`，回复后恢复 |

**教学提示：** 在 Network 面板中查看每次请求的 Payload 大小——随着对话进行，请求体会越来越大。这直观展示了为什么需要 `trimHistory`。
