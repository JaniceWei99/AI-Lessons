# 第12课：Demo Day — 分享你的 AI 网站

## 教学目标

- 学会准备和进行技术项目展示（Demo）
- 掌握 Demo 的标准流程：介绍 → 现场演示 → 技术亮点 → 问答
- 完善项目 README 文档，打造个人技术作品集
- 了解后续学习路径和比赛机会
- 回顾整门课程 12 节课的知识体系
- 激发持续学习 Web 开发和 AI 的热情

## 课时建议

**总时长：90 分钟**

## 教学流程

### 一、导入环节（5 分钟）

> **教师讲稿：**
>
> 同学们，今天是我们 AI 网站工坊的最后一课——Demo Day！
>
> 在硅谷的科技公司和创业圈里，Demo Day 是非常重要的环节——创业者向投资人展示自己的产品，工程师向团队展示新功能。今天，你们就是"创业者"，向大家展示你花了 12 节课做出来的 AI 网站。
>
> 这不仅是展示技术，更是锻炼"表达力"。一个好的程序员不仅要会写代码，还要会"讲"代码——让别人理解你做了什么、为什么这么做、用了什么技术。

**教学提示：** 营造轻松正式的氛围。可以简单布置教室——设置投影、准备计时器。让学生知道这是一个展示成果而非考试的环节。

### 二、Demo 准备指导（15 分钟）

#### 2.1 Demo 标准流程（5 分钟）

> 每人 4-5 分钟，按这个结构来：
>
> 1. **自我介绍 + 项目介绍**（30 秒）
>    - "大家好，我是 XX，今天展示的是我做的 XX 网站"
>    - 一句话说明网站做什么
>
> 2. **现场演示**（2 分钟）
>    - 打开网站，展示核心功能
>    - 实际操作给大家看（聊天、生成图片等）
>
> 3. **技术亮点**（1 分钟）
>    - 讲 1-2 个你最自豪的技术点
>    - "这个打字效果我用了 async/await 实现"
>    - "这个响应式布局在手机上也很好看"
>
> 4. **遇到的挑战**（30 秒）
>    - 分享一个你遇到的困难和怎么解决的
>
> 5. **问答**（30 秒）
>    - 回答同学或老师的提问

#### 2.2 Demo 小技巧（5 分钟）

> **准备工作很重要！** 在演示 AI 功能时：
> - 提前准备好要输入的消息和 Prompt，不要现场临时想
> - 准备一段备用的录屏视频——万一网络出问题可以播放
> - 提前测试一遍，确保 API 能正常工作
> - 把浏览器字体调大一些，让后排同学看清
> - 关掉不相关的浏览器标签和通知
>
> **展示时的建议：**
> - 面对观众说话，不要一直盯着屏幕
> - 速度放慢，让大家跟上
> - 讲代码时挑重点，不要念代码
> - 犯错了也没关系——即兴应对也是一种能力

#### 2.3 准备时间（5 分钟）

> 现在给大家 5 分钟准备。检查以下清单：
> - [ ] 网站能正常访问？
> - [ ] 准备好要演示的操作步骤？
> - [ ] 想好要讲的技术亮点？
> - [ ] 准备好备用方案？

### 三、学生 Demo 展示（40 分钟）

**教学提示：** 根据班级人数安排时间。15 人的班级，每人约 2.5 分钟；10 人的班级，每人约 4 分钟。建议：
- 用计时器控制时间
- 每个展示后简短点评——先说优点，再提建议
- 鼓励其他学生提问
- 记录每个学生的亮点，课后可以作为评价参考

> **评价维度（供教师参考）：**
> - 功能完整度（聊天/图片生成是否正常工作）
> - 界面美观度（CSS 美化程度、响应式适配）
> - 代码质量（是否有错误处理、代码组织是否清晰）
> - 创意与个性化（独特的功能、UI 设计、AI 人设）
> - 展示表达力（是否清晰、自信、有条理）

### 四、完善 README 与作品集（10 分钟）

#### 4.1 README 模板

> 每个 GitHub 项目都应该有一个好的 README——这是别人了解你项目的"第一印象"。

```markdown
# 我的 AI 网站 🤖

## 关于项目
简短描述你的网站做什么。

## 功能特性
- ✅ AI 聊天对话（支持多轮对话）
- ✅ AI 图像生成
- ✅ 响应式设计（手机/电脑自适应）
- ✅ Markdown 渲染
- ✅ 打字机效果

## 使用技术
- HTML5 + CSS3 + JavaScript (ES6+)
- OpenAI GPT-4o-mini API
- DALL-E 3 Image API
- GitHub Pages 部署

## 在线预览
🔗 [点击访问](https://你的用户名.github.io/你的仓库名)

## 截图
（粘贴你网站的截图）

## 学到的知识
- HTML/CSS 网页搭建
- JavaScript DOM 操作和事件处理
- API 调用和异步编程
- AI 接口集成

## 作者
你的名字 - AI 网站工坊第 X 期学员
```

**教学提示：** 让学生现场更新 README 并推送到 GitHub。这是他们作品集的重要组成部分。

### 五、课程回顾与未来路径（15 分钟）

#### 5.1 课程知识地图（5 分钟）

> 让我们回顾一下这 12 节课走过的路：
>
> ```
> L1-L2: HTML 基础 ──── 网页的骨架（标签、链接、列表、表单）
>    ↓
> L3-L4: CSS 美化  ──── 网页的外观（颜色、字体、Flexbox、响应式）
>    ↓
> L5-L6: JavaScript ─── 网页的大脑（变量、函数、DOM、事件）
>    ↓
> L7:    API 基础   ──── 连接互联网（fetch、JSON、async/await）
>    ↓
> L8-L9: AI 聊天    ──── 接入人工智能（ChatGPT API、多轮对话）
>    ↓
> L10:   AI 图像    ──── AI 创作（DALL-E API、Prompt 工程）
>    ↓
> L11:   部署上线   ──── 发布到全世界（Git、GitHub Pages）
>    ↓
> L12:   Demo Day   ──── 展示与分享！
> ```
>
> 从零开始，12 节课，你们做出了一个能和 AI 对话、能让 AI 画画的网站，而且全世界都能访问。这真的很了不起！

#### 5.2 下一步学什么（5 分钟）

> 如果你想继续深入，这是推荐的学习路径：
>
> **前端进阶：**
> - **React** —— 当今最流行的前端框架，Facebook 出品
> - **Next.js** —— React 的"全家桶"框架，包含服务端渲染
> - **TypeScript** —— 给 JavaScript 加上类型系统
>
> **后端开发：**
> - **Node.js** —— 用 JavaScript 写服务器端代码
> - **数据库** —— MySQL、MongoDB 存储数据
> - **Express.js** —— Node.js 的 Web 框架
>
> **AI 方向：**
> - **Python** —— AI/ML 的主流语言
> - **Prompt Engineering** —— 更深入地学习 AI 提示词工程
> - **LangChain** —— AI 应用开发框架

#### 5.3 推荐资源与比赛（5 分钟）

> **免费学习资源：**
> - **freeCodeCamp**（freecodecamp.org）—— 免费完整的 Web 开发课程
> - **MDN Web Docs**（developer.mozilla.org）—— 最权威的 Web 开发文档
> - **Scrimba**（scrimba.com）—— 交互式编程学习平台
> - **The Odin Project** —— 免费全栈开发课程
>
> **比赛机会：**
> - **Google Code to Learn** —— Google 举办的编程比赛
> - **各类 Hackathon（黑客马拉松）**—— 24-48 小时团队编程比赛
> - **全国青少年信息学奥林匹克** —— 如果对算法感兴趣
> - **GitHub 上的开源项目** —— 给真实项目贡献代码

### 六、结语（5 分钟）

> **教师讲稿：**
>
> 同学们，12 节课的旅程到此结束了。但你们的编程之旅才刚刚开始。
>
> 记住几件事：
> 1. **代码是写出来的，不是看出来的** —— 一定要多动手实践
> 2. **遇到 bug 不要怕** —— 每一个 bug 都是一次学习机会。用 `console.log`、用 DevTools、用搜索引擎
> 3. **保持好奇心** —— 技术变化很快，但学习能力比任何具体技术都重要
> 4. **分享你的作品** —— 把网站链接放到你的社交媒体、简历上。你做出的东西比任何证书都有说服力
>
> 你们已经站在了 Web 开发和 AI 的起跑线上。继续前进，未来可期！

---

> **备课提醒：**
> - 提前确认教室的投影和网络环境，Demo Day 对网络的依赖很大
> - 准备一个备用热点以防 Wi-Fi 不稳定
> - 如果人数较多，可以分组展示或采用"展位"形式（学生在自己电脑前展示，其他人轮流参观）
> - 准备简单的奖项或证书（如"最佳创意奖"、"最佳 UI 设计"、"最佳技术实现"），增加仪式感
> - 课后收集学生的网站链接，整理成课程成果合集
