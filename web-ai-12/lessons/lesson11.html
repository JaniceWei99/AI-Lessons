部署上线 — 发布到互联网 Deployment — Publish to the Internet

让全世界都能访问你的网站！学习 Git、GitHub Pages 和 Vercel 部署。 Make your website accessible to the whole world! Learn Git, GitHub Pages, and Vercel deployment.

11.1 课程目标 11.1 Course Goals

到目前为止，你的网站只能在自己的电脑上运行。在本节课中，你将学会把网站部署到互联网上，让任何人都能通过一个网址访问它！ So far, your website only runs on your own computer. In this lesson, you'll learn to deploy it to the internet so anyone can visit it via a URL!

📦

学习 Git 基础Learn Git Basics

掌握版本控制工具 Git 的核心概念和基本命令。Master the core concepts and basic commands of the version control tool Git.

🐙

使用 GitHubUse GitHub

创建仓库、上传代码，学会用 GitHub 管理你的项目。Create repositories, upload code, and manage your projects with GitHub.

🚀

部署上线Deploy Live

通过 GitHub Pages 或 Vercel 将网站发布到互联网。Publish your website to the internet via GitHub Pages or Vercel.

11.2 什么是部署？ 11.2 What Is Deployment?

部署（Deployment）就是把你的网站文件放到一台连接互联网的服务器上，这样别人就能通过网址（URL）访问你的网站了。 Deployment means putting your website files on a server connected to the internet, so others can access your site through a URL.

💻

本地网站Local Website

文件在你的电脑上，只有你能看到。地址是 file:// 或 localhost。Files are on your computer, only you can see them. Address is file:// or localhost.

🌍

线上网站Live Website

文件在服务器上，全世界都能访问。地址是 https://yourname.github.io 等。Files are on a server, accessible worldwide. Address is https://yourname.github.io etc.

💡 科普：域名（Domain）是网站的地址，如 google.com。DNS 把域名翻译成服务器的 IP 地址。免费托管平台会给你一个子域名，如 yourname.github.io。 💡 Fun fact: A domain is a website's address, like google.com. DNS translates domains to server IP addresses. Free hosting platforms give you a subdomain like yourname.github.io.

11.3 Git 与 GitHub 基础 11.3 Git & GitHub Basics

Git 是一个版本控制工具——它帮你追踪代码的每一次修改，就像游戏中的"存档"功能。GitHub 是一个在线平台，让你把 Git 仓库存储到云端。 Git is a version control tool — it tracks every change to your code, like a "save game" feature. GitHub is an online platform that stores your Git repositories in the cloud.

第一步：创建 GitHub 账号Step 1: Create a GitHub Account

前往 github.com 注册一个免费账号。选择一个好记的用户名——它会出现在你的网站地址中！ Go to github.com and sign up for a free account. Choose a memorable username — it will appear in your website URL!

第二步：创建仓库并上传代码Step 2: Create a Repository and Upload Code

你可以通过 GitHub 网页直接上传文件，或使用 Git 命令行： You can upload files directly through GitHub's web interface, or use Git commands:

Bash 
# Initialize a Git repository in your project folder
git init

# Stage all files for commit
git add .

# Create your first commit
git commit -m "Initial commit - my AI website"

# Connect to your GitHub repository
git remote add origin https://github.com/YOUR_USERNAME/my-ai-website.git

# Push your code to GitHub
git push -u origin main

💡 不熟悉命令行？没关系！你可以在 GitHub 网站上点击 "Add file" → "Upload files"，直接拖拽上传你的项目文件夹中的所有文件。 💡 Not comfortable with the command line? No worries! On the GitHub website, click "Add file" → "Upload files" and drag-and-drop all files from your project folder.

11.4 GitHub Pages 部署 11.4 GitHub Pages Deployment

GitHub Pages 是 GitHub 提供的免费静态网站托管服务。只需几步设置，你的仓库就变成了一个活的网站！ GitHub Pages is a free static website hosting service from GitHub. With a few settings, your repository becomes a live website!

部署步骤Deployment Steps

1️⃣

进入仓库设置Go to Repository Settings

打开你的 GitHub 仓库，点击顶部的 Settings 选项卡。Open your GitHub repository and click the Settings tab at the top.

2️⃣

启用 PagesEnable Pages

在左侧菜单找到 Pages，在 Source 下选择 main 分支，文件夹选 / (root)，点击 Save。Find Pages in the left menu. Under Source, select the main branch, folder / (root), and click Save.

3️⃣

等待部署Wait for Deployment

GitHub 会自动构建并部署你的网站，通常需要 1-3 分钟。GitHub will automatically build and deploy your site, usually taking 1-3 minutes.

4️⃣

访问你的网站！Visit Your Website!

部署成功后，你的网站地址是：https://YOUR_USERNAME.github.io/REPO_NAME/Once deployed, your website URL is: https://YOUR_USERNAME.github.io/REPO_NAME/

11.5 Vercel 部署（备选方案） 11.5 Vercel Deployment (Alternative)

Vercel 是另一个优秀的免费托管平台，部署过程更加简单，而且支持自动部署——每次你推送代码到 GitHub，Vercel 会自动更新网站。 Vercel is another excellent free hosting platform with an even simpler deployment process. It supports automatic deployments — every time you push code to GitHub, Vercel updates your site automatically.

Steps 
1. Go to vercel.com and sign up with your GitHub account
2. Click "New Project"
3. Import your GitHub repository
4. Click "Deploy"
5. Done! Your site is live at https://your-project.vercel.app

💡 Vercel vs GitHub Pages：两者都免费且优秀。GitHub Pages 更简单，适合纯静态网站；Vercel 功能更强大，支持自定义域名和服务端功能。 💡 Vercel vs GitHub Pages: Both are free and excellent. GitHub Pages is simpler, great for pure static sites; Vercel is more powerful with custom domains and serverless functions.

课堂测验Lesson Quiz

1. "部署"一个网站意味着什么？1. What does "deploying" a website mean?

部署就是把网站文件放到互联网上的服务器中，这样任何人都能通过网址访问你的网站。Deployment means putting website files on an internet server so anyone can access your site via a URL.

2. Git 的主要功能是什么？2. What is the primary purpose of Git?

Git 是一个版本控制系统，它帮你记录代码的每次更改，可以随时回到之前的版本，就像游戏中的存档功能。Git is a version control system that records every code change. You can go back to any previous version, like a save-game feature.

3. 以下哪个命令用于将文件添加到 Git 暂存区？3. Which command adds files to the Git staging area?

git add 将文件添加到暂存区，准备提交。git commit 保存更改，git push 推送到远程仓库。git add stages files for commit. git commit saves changes, and git push uploads to the remote repository.

4. GitHub Pages 部署后，你的网站地址格式通常是什么？4. After deploying with GitHub Pages, what is the typical URL format?

GitHub Pages 的默认网址格式是 https://username.github.io/repo-name/，其中 username 是你的 GitHub 用户名。The default GitHub Pages URL format is https://username.github.io/repo-name/, where username is your GitHub username.

5. Vercel 相比 GitHub Pages 的一个优势是什么？5. What is one advantage of Vercel over GitHub Pages?

Vercel 支持自动部署：每次你把代码推送到 GitHub，Vercel 会自动检测变化并重新部署网站，非常方便。Vercel supports automatic deployments: every time you push code to GitHub, Vercel detects changes and redeploys your site automatically — very convenient.
