9.2 多轮对话9.2 Multi-Turn Conversation

在第 8 课中，我们每次只发送一条消息给 AI。但真正的聊天需要 AI 记住之前说过的话。要实现这一点，我们需要把整个对话历史发送给 API。 In Lesson 8, we sent only one message to AI each time. But real chat requires AI to remember what was said before. To achieve this, we need to send the entire conversation history to the API.

JavaScript 
// 维护对话历史数组
const conversationHistory = [
  {
    role: 'system',
    content: 'You are a helpful coding assistant.'
  }
];

async function sendMessage(userText) {
  // 1. 将用户消息加入历史
  conversationHistory.push({
    role: 'user',
    content: userText
  });

  // 2. 发送完整历史给 API
  const response = await fetch(API_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: conversationHistory,
      temperature: 0.7
    })
  });

  const data = await response.json();
  const aiReply = data.choices[0].message.content;

  // 3. 将 AI 回复也加入历史
  conversationHistory.push({
    role: 'assistant',
    content: aiReply
  });

  return aiReply;
}

上下文窗口与 Token 限制Context Window & Token Limits

每个模型都有上下文窗口（Context Window）限制——它一次能处理的最大 Token 数量。 Every model has a context window limit — the maximum number of tokens it can process at once.

  • GPT-4o-mini：128K tokens（约 10 万字）GPT-4o-mini: 128K tokens (~100,000 words)
  • GPT-4o：128K tokensGPT-4o: 128K tokens
  • Claude 3.5：200K tokensClaude 3.5: 200K tokens

当对话变得很长时，你可能需要裁剪历史，只保留最近的 N 条消息，以避免超出限制。 When conversations get very long, you may need to trim history, keeping only the last N messages, to avoid exceeding limits.

JavaScript 
// 裁剪对话历史，保留 system + 最近 20 条消息
function trimHistory(history, maxMessages = 20) {
  if (history.length <= maxMessages + 1) return history;

  const systemMsg = history[0]; // 保留 system prompt
  const recent = history.slice(-(maxMessages));
  return [systemMsg, ...recent];
}

9.3 打字效果9.3 Typing Effect

ChatGPT 和 Claude 的网页版都有逐字显示的效果，让用户感觉 AI 在"实时思考"。我们可以用两种方式实现：前端模拟或流式传输（Streaming）。 The web versions of ChatGPT and Claude both have a character-by-character display effect, making users feel AI is "thinking in real-time." We can implement this in two ways: frontend simulation or streaming.

方式一：前端模拟打字效果Method 1: Frontend Typing Simulation

JavaScript 
// 逐字显示文本
function typeText(element, text, speed = 20) {
  return new Promise((resolve) => {
    let index = 0;

    function type() {
      if (index < text.length) {
        element.textContent += text[index];
        index++;
        setTimeout(type, speed);
      } else {
        resolve(); // 打字完成
      }
    }

    type();
  });
}

// 使用示例
async function displayAIReply(text) {
  const msgDiv = document.createElement('div');
  msgDiv.classList.add('message', 'ai');
  chatMessages.appendChild(msgDiv);

  await typeText(msgDiv, text, 15);
  chatMessages.scrollTop = chatMessages.scrollHeight;
}

方式二：流式传输（Streaming）Method 2: Streaming

真正的流式传输是让 API 边生成边发送数据。通过设置 stream: true，服务器会用 Server-Sent Events（SSE）一块一块地返回内容。 True streaming lets the API send data as it generates. By setting stream: true, the server returns content chunk by chunk using Server-Sent Events (SSE).

JavaScript 
async function streamAIReply(messages) {
  const response = await fetch(API_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: messages,
      stream: true  // 开启流式传输！
    })
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let aiText = '';

  const msgDiv = document.createElement('div');
  msgDiv.classList.add('message', 'ai');
  chatMessages.appendChild(msgDiv);

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n');

    for (const line of lines) {
      if (line.startsWith('data: ') && line !== 'data: [DONE]') {
        const json = JSON.parse(line.slice(6));
        const content = json.choices[0]?.delta?.content || '';
        aiText += content;
        msgDiv.textContent = aiText;
      }
    }

    chatMessages.scrollTop = chatMessages.scrollHeight;
  }

  return aiText;
}

9.4 Markdown 渲染9.4 Markdown Rendering

AI 经常使用 Markdown 格式回复，包括粗体、斜体、代码块、列表等。我们需要把 Markdown 转换成 HTML 才能正确显示。 AI often responds in Markdown format, including bold, italic, code blocks, lists, and more. We need to convert Markdown to HTML for proper display.

JavaScript 
// 简易 Markdown → HTML 转换器
function markdownToHtml(md) {
  let html = md
    // 代码块 ```code```
    .replace(/```(\w*)\n([\s\S]*?)```/g,
      '<pre><code class="lang-$1">$2</code></pre>')
    // 行内代码 `code`
    .replace(/`([^`]+)`/g, '<code>$1</code>')
    // 粗体 **text**
    .replace(/\*\*(.+?)\*\*/g, '<strong>$1</strong>')
    // 斜体 *text*
    .replace(/\*(.+?)\*/g, '<em>$1</em>')
    // 标题 ### heading
    .replace(/^### (.+)$/gm, '<h4>$1</h4>')
    .replace(/^## (.+)$/gm, '<h3>$1</h3>')
    // 无序列表
    .replace(/^- (.+)$/gm, '<li>$1</li>')
    // 换行
    .replace(/\n/g, '<br>');

  // 包裹 li 标签
  html = html.replace(
    /(<li>.*?<\/li>(\s*<br>)?)+/g,
    '<ul>$&</ul>'
  );

  return html;
}

// 使用：设置 innerHTML 而不是 textContent
msgDiv.innerHTML = markdownToHtml(aiText);
💡 进阶提示：在生产环境中，推荐使用成熟的库来处理 Markdown 渲染，比如 marked.js 或 markdown-it。它们处理了更多的边界情况，更安全、更可靠。代码高亮可以用 highlight.js 或 Prism.js。 💡 Pro tip: In production, use mature libraries for Markdown rendering like marked.js or markdown-it. They handle many more edge cases and are safer and more reliable. For code highlighting, use highlight.js or Prism.js.

友好的错误消息Friendly Error Messages

JavaScript 
function handleError(error) {
  let errorMsg = 'Oops! Something went wrong.';

  if (error.message.includes('401')) {
    errorMsg = 'Invalid API key. Please check your key.';
  } else if (error.message.includes('429')) {
    errorMsg = 'Too many requests. Please wait a moment.';
  } else if (error.message.includes('Failed to fetch')) {
    errorMsg = 'Network error. Check your connection.';
  }

  addMessage(`⚠️ ${errorMsg}`, 'ai');
}

9.6 动手实践：升级你的聊天机器人9.6 Hands-On: Upgrade Your Chatbot

把以上所有功能整合在一起，升级你在第 8 课创建的聊天机器人。下面是完整的 JavaScript 主逻辑： Combine all the features above and upgrade the chatbot you created in Lesson 8. Here's the complete main JavaScript logic:

JavaScript 
// ========= 完整聊天机器人 =========
const chatMessages = document.getElementById('chatMessages');
const userInput = document.getElementById('userInput');
const sendBtn = document.getElementById('sendBtn');
const clearBtn = document.getElementById('clearBtn');

const conversationHistory = [
  { role: 'system', content: 'You are a helpful assistant.' }
];

// 添加消息到 UI
function addMessage(text, sender, useMarkdown = false) {
  const div = document.createElement('div');
  div.classList.add('message', sender);

  if (useMarkdown && sender === 'ai') {
    div.innerHTML = markdownToHtml(text);
  } else {
    div.textContent = text;
  }

  chatMessages.appendChild(div);
  scrollToBottom();
  return div;
}

// 发送消息
async function sendMessage() {
  const text = userInput.value.trim();
  if (!text) return;

  addMessage(text, 'user');
  userInput.value = '';
  setLoading(true);

  conversationHistory.push({ role: 'user', content: text });

  try {
    const trimmed = trimHistory(conversationHistory);
    const aiReply = await callAI(trimmed);

    conversationHistory.push({
      role: 'assistant', content: aiReply
    });

    // 用打字效果显示 AI 回复
    const msgDiv = addMessage('', 'ai');
    await typeText(msgDiv, aiReply, 12);
    msgDiv.innerHTML = markdownToHtml(aiReply);
  } catch (error) {
    handleError(error);
  } finally {
    setLoading(false);
    userInput.focus();
  }
}

// 事件监听
sendBtn.addEventListener('click', sendMessage);
userInput.addEventListener('keydown', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    sendMessage();
  }
});
clearBtn.addEventListener('click', clearChat);

// 初始欢迎消息
addMessage('Hello! I\'m your AI assistant. Ask me anything! 😊', 'ai');

课堂测验Lesson Quiz

1. 要实现多轮对话，我们需要在每次 API 请求中发送什么？ 1. What do we need to send in each API request to enable multi-turn conversation?

AI API 本身是无状态的（stateless）。要让 AI "记住"之前的对话，我们必须在每次请求中把完整的 messages 数组发过去。 The AI API is stateless. To make AI "remember" previous conversation, we must send the complete messages array in every request.

2. 当对话历史太长超出 Token 限制时，我们应该怎么做？ 2. What should we do when conversation history exceeds the token limit?

最常见的做法是裁剪对话历史：保留 system prompt 和最近的 N 条消息。这样既保持了上下文，又不会超出 Token 限制。 The most common approach is to trim conversation history: keep the system prompt and the most recent N messages. This maintains context without exceeding the token limit.

3. 要实现流式传输（Streaming），需要在请求中设置哪个参数？ 3. Which parameter enables streaming in the API request?

在请求体中设置 stream: true 可以启用流式传输。服务器会通过 Server-Sent Events 逐块返回生成的内容。 Setting stream: true in the request body enables streaming. The server returns generated content chunk by chunk via Server-Sent Events.

4. 为什么在移动端聊天输入框中设置 font-size: 16px？ 4. Why set font-size: 16px on the mobile chat input field?

iOS Safari 会在输入框字号小于 16px 时自动放大页面。将字号设为 16px 或更大可以防止这个行为，提供更好的移动端体验。 iOS Safari auto-zooms the page when input field font size is less than 16px. Setting it to 16px or larger prevents this behavior, providing a better mobile experience.

5. 在简易 Markdown 渲染中，**text** 应该转换成什么 HTML？ 5. In basic Markdown rendering, what HTML should **text** be converted to?

在 Markdown 中，**text**（双星号）表示粗体，应该转换成 <strong>text</strong>。单星号 *text* 是斜体，对应 <em>。 In Markdown, **text** (double asterisks) means bold and should be converted to <strong>text</strong>. Single asterisk *text* is italic, corresponding to <em>.
