8.2 AI API 介绍8.2 Introduction to AI APIs

现在最流行的 AI API 包括 OpenAI（GPT-4o）和 Anthropic（Claude）。它们背后是大型语言模型（LLM），经过海量文本训练，能够理解和生成自然语言。 The most popular AI APIs today include OpenAI (GPT-4o) and Anthropic (Claude). Behind them are Large Language Models (LLMs) trained on massive amounts of text that can understand and generate natural language.

LLM 是怎么工作的？（简化版）How Do LLMs Work? (Simplified)

  • 训练 — 模型阅读了互联网上数十亿的网页、书籍和文章Training — The model read billions of web pages, books, and articles from the internet
  • 预测 — 它学会了"给定前面的文字，下一个词最可能是什么"Prediction — It learned "given the preceding text, what word is most likely next"
  • 对话 — 通过特殊的消息格式（system/user/assistant），模型可以进行对话Conversation — Through a special message format (system/user/assistant), the model can have conversations

核心概念Key Concepts

  • API Key — 你的身份凭证，类似密码，用于验证你的请求API Key — Your credential, like a password, used to authenticate your requests
  • Rate Limits — API 对每分钟请求次数有限制，防止滥用Rate Limits — APIs limit requests per minute to prevent abuse
  • 定价 — 按 Token（大约 4 个字符 = 1 个 Token）计费，输入和输出分别计价Pricing — Charged per token (roughly 4 characters = 1 token), input and output priced separately
🔒 安全警告：永远不要把 API Key 直接写在前端代码中！任何人都能通过浏览器的开发者工具看到你的代码。正确做法是：使用后端代理（proxy server）来转发请求，API Key 只保存在服务器端。在学习阶段，我们会用一个简单的方式来演示，但真正上线时一定要用后端代理。 🔒 Security Warning: Never put your API key directly in frontend code! Anyone can see your code through browser dev tools. The correct approach is to use a backend proxy server to forward requests, keeping the API key on the server side only. For learning purposes, we'll use a simple approach, but for production always use a backend proxy.

8.4 连接 AI API8.4 Connecting to the AI API

OpenAI 的 Chat Completions API 是目前最流行的 AI 对话接口。让我们看看请求格式。 OpenAI's Chat Completions API is the most popular AI conversation interface today. Let's look at the request format.

请求格式Request Format

  • model — 使用的模型名称，如 "gpt-4o-mini"model — Model name, e.g. "gpt-4o-mini"
  • messages — 消息数组，包含对话历史messages — Array of messages containing conversation history
  • temperature — 创造性程度（0 = 精确，1 = 创造性）temperature — Creativity level (0 = precise, 1 = creative)
JavaScript 
async function sendToAI(userMessage) {
  const API_KEY = 'your-api-key-here'; // ⚠️ 学习用，上线要用后端代理！

  const response = await fetch(
    'https://api.openai.com/v1/chat/completions',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${API_KEY}`
      },
      body: JSON.stringify({
        model: 'gpt-4o-mini',
        messages: [
          {
            role: 'system',
            content: 'You are a helpful assistant.'
          },
          {
            role: 'user',
            content: userMessage
          }
        ],
        temperature: 0.7
      })
    }
  );

  const data = await response.json();
  return data.choices[0].message.content;
}

消息角色（Roles）说明Message Roles Explained

  • system — 设定 AI 的行为和性格（"你是一个友好的助手"）system — Sets the AI's behavior and personality ("You are a helpful assistant")
  • user — 用户发送的消息user — Messages sent by the user
  • assistant — AI 之前的回复（用于多轮对话）assistant — AI's previous replies (used for multi-turn conversations)

8.5 动手实践：完整的 AI 聊天页面8.5 Hands-On: Complete AI Chat Page

让我们把所有部分组合起来，实现一个完整的"发送消息 → AI 回复"的流程。 Let's combine everything to implement a complete "send message → AI replies" flow.

JavaScript 
const chatMessages = document.getElementById('chatMessages');
const userInput = document.getElementById('userInput');
const sendBtn = document.getElementById('sendBtn');

// 添加消息到聊天区域
function addMessage(text, sender) {
  const div = document.createElement('div');
  div.classList.add('message', sender);
  div.textContent = text;
  chatMessages.appendChild(div);
  chatMessages.scrollTop = chatMessages.scrollHeight;
}

// 发送消息给 AI
async function sendMessage() {
  const text = userInput.value.trim();
  if (!text) return;

  // 显示用户消息
  addMessage(text, 'user');
  userInput.value = '';
  sendBtn.disabled = true;

  try {
    const aiReply = await sendToAI(text);
    addMessage(aiReply, 'ai');
  } catch (error) {
    addMessage('Sorry, something went wrong.', 'ai');
    console.error(error);
  } finally {
    sendBtn.disabled = false;
    userInput.focus();
  }
}

// 点击发送
sendBtn.addEventListener('click', sendMessage);

// 按回车发送
userInput.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') sendMessage();
});
🔍 代码亮点：scrollTop = scrollHeight 让聊天区域自动滚动到最新消息。sendBtn.disabled = true 防止用户在等待 AI 回复时重复发送。按回车键也能发送消息，提升用户体验。 🔍 Code highlights: scrollTop = scrollHeight auto-scrolls the chat to the latest message. sendBtn.disabled = true prevents duplicate sends while waiting for AI. Pressing Enter also sends — improving UX.

8.6 挑战任务8.6 Challenges

挑战 A：添加"正在输入"指示器Challenge A: Add a "Typing..." Indicator

在等待 AI 回复时，显示一个跳动的"正在输入..."动画，让用户知道 AI 正在思考。 While waiting for AI to reply, show a bouncing "Typing..." animation so users know AI is thinking.

JavaScript 
// 提示：创建一个 typing indicator 元素
function showTyping() {
  const typing = document.createElement('div');
  typing.classList.add('message', 'ai', 'typing');
  typing.id = 'typingIndicator';
  typing.innerHTML = '<span>.</span><span>.</span><span>.</span>';
  chatMessages.appendChild(typing);
  chatMessages.scrollTop = chatMessages.scrollHeight;
}

function hideTyping() {
  const typing = document.getElementById('typingIndicator');
  if (typing) typing.remove();
}

课堂测验Lesson Quiz

1. LLM 的全称是什么？ 1. What does LLM stand for?

LLM 全称 Large Language Model（大型语言模型），如 GPT-4 和 Claude 都属于 LLM。 LLM stands for Large Language Model. GPT-4 and Claude are both LLMs.

2. 为什么不应该把 API Key 放在前端代码中？ 2. Why should you NOT put API keys in frontend code?

前端代码对所有人可见。任何人都可以通过浏览器开发者工具查看你的 JavaScript，从而窃取你的 API Key。应该使用后端代理来保护 API Key。 Frontend code is visible to everyone. Anyone can use browser dev tools to view your JavaScript and steal your API key. Use a backend proxy to protect API keys.

3. 在 Chat Completions API 中，system 角色的作用是什么？ 3. What is the role of system in the Chat Completions API?

system 消息用于设定 AI 的行为准则和性格特征。例如"你是一个友好的助手"或"你是一个编程老师"。 The system message sets the AI's behavior guidelines and personality. For example, "You are a helpful assistant" or "You are a coding teacher."

4. temperature 参数设为 0 时，AI 的回复会怎样？ 4. What happens when the temperature parameter is set to 0?

temperature 控制输出的随机性。0 表示最确定、最精确的回复；1 表示更随机、更有创造性。 temperature controls output randomness. 0 means the most deterministic and precise replies; 1 means more random and creative.

5. 以下哪个 HTTP 方法用于向 AI API 发送聊天请求？ 5. Which HTTP method is used to send chat requests to the AI API?

Chat Completions API 使用 POST 方法，因为我们需要在请求体（body）中发送消息数据。GET 请求不能携带请求体。 The Chat Completions API uses the POST method because we need to send message data in the request body. GET requests cannot carry a request body.
